# 回测引擎 (Backtest Engine)

> 创建日期: 2025-12-31

## 一、概述

### 1.1 做了什么

创建了 `BreakoutStrategy/backtest/` 模块，实现基于观察池系统的回测引擎。

| 组件 | 说明 |
|------|------|
| `BacktestConfig` | 回测配置（资金、止盈止损、仓位等） |
| `BacktestEngine` | 回测引擎核心类 |
| `BacktestResult` | 回测结果（交易记录、权益曲线、绩效） |
| `Trade` | 单笔交易记录 |

### 1.2 目的

**解决的问题**：验证观察池系统的买入信号在历史数据上的表现。

**与现有回测的关系**：
- `scripts/backtest/simple_backtest.py` - 简单回测，直接从突破点买入
- `BacktestEngine` (新) - 基于观察池，模拟真实的买入决策流程

```
简单回测:
  突破检测 → 立即买入 → 止盈止损

观察池回测:
  突破检测 → 加入观察池 → 每日评估 → 满足条件买入 → 止盈止损
                  ↓
            [双池管理]
            [四维评估]
            [信号生成]
```

### 1.3 核心价值

1. **验证评估器效果**：测试四维评估器（时间窗口、价格确认、成交量、风险过滤）的实际表现
2. **参数优化基础**：为评估器参数调优提供量化反馈
3. **策略对比**：与简单回测对比，衡量观察池系统的增益

## 二、使用方式

### 2.1 使用者类型

| 使用者 | 类型 | 说明 |
|--------|------|------|
| 策略研究员 | **用户直接使用** | 运行回测脚本验证策略 |
| 参数优化脚本 | 模块调用 | 批量运行不同参数组合 |

### 2.2 用户使用方式

#### 方式 1：运行入口脚本（推荐）

```bash
python scripts/backtest/realtime_pool_backtest.py
```

配置参数在脚本的 `main()` 函数中修改：

```python
def main():
    # ===== 配置参数 =====
    scan_result_path = 'outputs/scan_results.json'  # JSON 扫描结果
    data_dir = 'datasets/pkls'                       # PKL 数据目录
    start_date = '2024-01-01'                        # 回测起始
    end_date = '2024-06-30'                          # 回测结束

    config = BacktestConfig(
        initial_capital=100000,    # 初始资金
        stop_loss_pct=0.05,        # 止损 5%
        take_profit_pct=0.15,      # 止盈 15%
        max_holdings=10,           # 最多持有 10 只
        max_position_size_pct=0.10 # 单只最多 10% 仓位
    )
```

#### 方式 2：Python 代码调用

```python
from BreakoutStrategy.backtest import BacktestEngine, BacktestConfig

config = BacktestConfig(
    initial_capital=100000,
    stop_loss_pct=0.05,
    take_profit_pct=0.15,
)

engine = BacktestEngine(
    config=config,
    scan_result_path='outputs/scan_results.json',
    data_dir='datasets/pkls'
)

result = engine.run(start_date='2024-01-01', end_date='2024-06-30')

# 查看结果
print(result.summary())
print(f"交易次数: {len(result.trades)}")
print(f"胜率: {result.performance['win_rate']:.2%}")
```

### 2.3 输出说明

```

Winning Trades: 28
Win Rate: 62.22%
Total Return: 15.34%
Max Drawdown: 8.21%
Sharpe Ratio: 1.45

```

输出文件（如果启用 `save_results=True`）：
- `outputs/backtest/trades_2024-01-01_2024-06-30.csv` - 交易记录
- `outputs/backtest/equity_2024-01-01_2024-06-30.csv` - 权益曲线
- `outputs/backtest/performance_2024-01-01_2024-06-30.json` - 绩效统计

## 四、配置参数

### 4.1 BacktestConfig 详解

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `initial_capital` | 100000 | 初始资金 |
| `stop_loss_pct` | 0.05 | 止损比例 (5%) |
| `take_profit_pct` | 0.15 | 止盈比例 (15%) |
| `max_holdings` | 10 | 最大同时持仓数 |
| `max_position_size_pct` | 0.10 | 单只股票最大仓位比例 |
| `min_position_size` | 1000 | 最小持仓金额 |
| `commission_rate` | 0.001 | 手续费率 (0.1%) |
| `realtime_observation_days` | 1 | 实时池观察天数 |
| `daily_observation_days` | 30 | 日K池观察天数 |
| `min_quality_score` | 0.0 | 最低质量评分 |

### 4.2 绩效指标

| 指标 | 计算方式 |
|------|----------|
| `total_return` | 总盈亏 / 初始资金 |
| `win_rate` | 盈利交易数 / 总交易数 |
| `max_drawdown` | 最大回撤 = (峰值 - 谷值) / 峰值 |
| `sharpe_ratio` | 年化收益 / 年化波动率 × √252 |
| `profit_factor` | 总盈利 / 总亏损 |
| `avg_holding_days` | 平均持仓天数 |
