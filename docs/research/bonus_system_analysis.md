# BreakoutStrategy Bonus 体系综合评估报告

> 分析日期: 2026-01-12
> 分析目标: 评估现有 Bonus 体系的完整性、冗余性和缺口

---

## 一、执行摘要

本报告对 BreakoutStrategy 的 Bonus 评分体系进行了系统性审查，从冗余性、缺口、维度覆盖三个角度进行分析。

**核心发现**：
- 现有体系在**阻力位强度**和**波动率调整**维度覆盖完整
- **PBM 与 Overshoot 不冗余**：虽都与"涨势"相关，但衡量的是正交的信息维度
- **存在部分冗余**：`daily_return_vol_bonus` 与 `gap_vol_bonus` 存在双重计算风险
- **关键缺口**：趋势环境（MA）和 K 线形态未纳入评分

**总体完整度评分**: 80%

## 三、冗余性分析

### 3.1 PBM vs Overshoot：不冗余

用户观察到 PBM（突破前涨势）和 Overshoot（超涨惩罚）都在衡量"涨势"，但深入分析表明它们是**正交的**：

| 维度 | PBM Bonus | Overshoot Penalty |
|------|-----------|-------------------|
| **时间范围** | 突破**前** 5 天（不含突破日） | 突破日及**前** 4 天（共 5 天） |
| **衡量对象** | 价格走势的**效率和方向** | 绝对涨幅的**幅度** |
| **计算公式** | `(ΔP/P₀) × (ΔP/L) / N` | `gain_5d / (annual_vol / √50.4)` |
| **物理意义** | 涨势的"直接性"和"持续性" | 涨幅是否偏离正常波动范围 |
| **评分方向** | 正面**奖励**（涨势强=好） | 负面**惩罚**（涨太多=坏） |
| **单位** | 无量纲效率值 | σ（标准差倍数） |

**信息正交性示例**：

| 场景 | PBM | Overshoot | 解读 |
|------|-----|-----------|------|
| 理想突破 | 高 | 低 | 有动能积累，未超涨 |
| 追高风险 | 高 | 高 | 趋势好但涨幅已大 |
| 单日暴涨 | 低 | 高 | 突然暴涨，无前兆 |
| 低质突破 | 低 | 低 | 无明显趋势，也没超涨 |

**结论**：两者形成**逻辑互补**，不应合并。

---

### 3.2 Volume Bonus vs Peak Volume Bonus：需监控

| 维度 | Volume Bonus | Peak Volume Bonus |
|------|--------------|-------------------|
| **衡量时点** | **突破日**的成交量 | **峰值形成日**的成交量 |
| **技术意义** | 突破时的市场参与度 | 阻力形成时的市场博弈强度 |

**冗余程度**：中等 - 预计相关系数 0.4-0.6

**风险**：高成交量股票两者可能同时触发，导致量能维度被双重计算。

**建议**：
- 方案 A：合并为单一指标，取 `max(peak_vol, breakout_vol)`
- 方案 B：保留两者，但降低乘数避免过度放大

---

### 3.3 Daily Return Vol vs Gap Vol：部分冗余

| 维度 | Daily Return Vol Bonus | Gap Vol Bonus |
|------|------------------------|---------------|
| **分子** | `(close - open) / open` | `(open - prev_close) / prev_close` |
| **衡量对象** | 盘中涨幅强度 | 开盘跳空强度 |
| **默认阈值** | 2σ, 3σ | 1σ, 2σ |

**冗余性分析**：

两者衡量的是**同一天 K 线的不同部分**：

```
收盘价 ─────┐
            │ price_change_pct (Daily Return Vol 衡量)
开盘价 ─────┼───┐
            │   │ gap_up_pct (Gap Vol 衡量)
前收盘 ─────┘───┘
```

**问题**：
1. 跳空高开且收阳的 K 线会**双重奖励**
2. 日间总涨幅 = 跳空 + 盘中涨幅，信息有重叠

**建议**：合并为统一的 `day_strength_vol_bonus`：
```
ratio = (close - prev_close) / prev_close / daily_vol
```

---

### 3.4 冗余性判定汇总

| Bonus 对 | 冗余判定 | 理由 |
|----------|---------|------|
| PBM vs Overshoot | **不冗余** | 前者看突破前，后者看突破后；前者奖励，后者惩罚 |
| Volume vs Peak Volume | **需监控** | 前者看突破日，后者看峰值形成日；可能存在相关性 |
| Daily Return Vol vs Gap Vol | **部分冗余** | 同一天 K 线的不同部分，存在双重计算风险 |
| PBM vs Momentum | **不冗余** | 前者是价格动能，后者是事件频率 |
| PK Momentum vs Height | **不冗余** | 前者是形态（凹陷），后者是位置（高度） |

## 四、缺口分析

### 4.1 从第一性原理看高质量突破的特征

| 维度 | 特征 | 现有覆盖 |
|------|------|---------|
| **阻力位强度** | 存在时间长、被多次测试、位置较高、放量形成 | 完整 (4/4) |
| **突破确认** | 成交量放大、价格动量、连续突破、突破幅度 | 较完整 (4/6) |
| **市场环境** | 趋势方向、波动率环境、市场强度 | **严重不足** (1/4) |
| **风险因素** | 超涨、假突破、流动性、技术背离 | 不足 (1/4) |

### 4.2 识别的关键缺口

#### 缺口 1：趋势环境 Bonus（优先级：P0）

**现状**：系统有 `ma_period` 参数（默认 200），但**未转化为评分**。

**问题**：顺势突破的成功率显著高于逆势突破，当前同等对待。

**建议实现**：
```yaml
trend_bonus:
  logic:
    - price > MA200: 顺势突破 → 1.15x
    - price > MA200 且 MA50 > MA200: 强顺势 → 1.25x
    - price < MA50: 逆势突破 → 0.85x（惩罚）
```

**实现难度**：低（已有 MA 参数）

---

#### 缺口 2：K 线形态 Bonus（优先级：P0）

**现状**：代码中有 `breakout_type`（yang/yin/shadow）分类，但**未用于评分**。

**问题**：阳线突破的信号强度明显高于上影线突破。

**建议实现**：
```yaml
candle_pattern_bonus:
  logic:
    - 大阳线突破（涨幅 > 3%）→ 1.20x
    - 普通阳线突破 → 1.10x
    - 阴线/上影线突破 → 1.0x（无加成）
    - 长上影线（上影 > 实体）→ 0.85x（惩罚）
```

**实现难度**：低（已有 breakout_type）

---

#### 缺口 3：收盘位置 Bonus（优先级：P1）

**问题**：收于日内高位表明买盘强劲，收于低位表明卖压沉重。

**建议实现**：
```yaml
close_position_bonus:
  formula: position = (close - low) / (high - low)
  logic:
    - position >= 0.8（收于高位）→ 1.15x
    - position >= 0.6 → 1.05x
    - position < 0.3（收于低位）→ 0.90x
```

---

#### 缺口 4：阻力区密度 Bonus（优先级：P1）

**问题**：当前 `test_bonus` 只统计峰值数，未评估阻力区的"厚度"。

**建议实现**：
```yaml
resistance_density_bonus:
  logic: density = num_peaks / (price_range / avg_price)
  阈值: 高密度阻力区 → 1.15x
```

---

#### 缺口 5：成交量绝对水平（优先级：P2）

**问题**：当前只评估放量倍数（相对量），未过滤绝对流动性。

**建议实现**：
```yaml
liquidity_penalty:
  logic:
    - avg_volume < 50万股/日 → 0.85x
    - avg_volume < 20万股/日 → 0.70x
```

---

#### 缺口 6：相对强度 Bonus（优先级：P3）

**问题**：未考虑个股相对大盘/行业的强弱。

**建议实现**：
```yaml
relative_strength_bonus:
  logic: RS = stock_20d_return / SPY_20d_return
  阈值:
    - RS > 1.5 → 1.15x
    - RS < 0.5 → 0.90x
```

**实现难度**：高（需引入外部基准数据）

## 六、现有体系优势

1. **波动率标准化设计**：`daily_return_vol_bonus`, `gap_vol_bonus`, `overshoot_penalty` 使用 σ 单位，使不同波动率股票可比。

2. **乘法模型架构**：避免权重归一化问题，各维度独立可调。

3. **双阈值设计**：`exceed_threshold`（0.5%）用于敏感检测，`peak_supersede_threshold`（3%）用于保守移除。

4. **簇分组逻辑**：将价格相近的峰值分组为阻力簇，比简单的最大值更符合实际。

## 八、结论

BreakoutStrategy 的 Bonus 体系在**阻力位质量**和**波动率调整**两个核心维度上设计成熟，但存在以下可优化点：

1. **趋势维度缺失**：最关键的缺口，强烈建议新增 MA 趋势过滤
2. **K 线形态缺失**：已有 `breakout_type` 数据但未利用
3. **部分冗余**：`daily_return_vol_bonus` 与 `gap_vol_bonus` 建议合并
4. **需监控相关性**：`volume_bonus` 与 `peak_volume_bonus`

**整体评分**：

| 评估项 | 得分 | 说明 |
|--------|------|------|
| 维度覆盖 | 7/10 | 缺少趋势、K线形态、相对强度 |
| 指标冗余 | 7/10 | 波动率指标存在部分冗余 |
| 权重合理性 | 8/10 | 阻力位质量权重最高，符合策略逻辑 |
| 波动率处理 | 9/10 | 设计精巧，攻防平衡 |
| 可扩展性 | 9/10 | 乘法模型易于新增 Bonus |

**总分**：40/50 = **80%**

通过实现 P0 优先级的两个改进（trend_bonus + candle_pattern_bonus），可将完整度提升至 **90%+**。
