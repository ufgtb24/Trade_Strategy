# 新增 Bonus 因子必要性评估报告

> 基于 2026-02-15 统计数据（9791 个突破事件），对旧报告 `bonus_system_analysis.md` 建议的 6 个候选新增因子进行逐一审计。
>
> 核心立场：在 R^2=0.1051、现有 11 个因子中有 3-4 个无效/反向因子尚未修复的前提下，任何新增因子都必须通过极高的必要性门槛。

---

## 零、审计前提：当前系统的"技术债务"

在讨论是否新增因子之前，必须先量化当前系统的未修复问题。这些问题构成了新增因子讨论的基准背景。

### 0.1 现有因子效能速查

| 排名 | 因子 | Spearman r | RF 重要性 | 当前状态 | 紧急程度 |
|------|------|-----------|----------|---------|---------|
| 1 | Height | +0.3044 | 0.5810 | 启用但权重严重偏低（最大1.4x） | 需上调 |
| 2 | DayStr | +0.1590 | 0.0515 | 启用但权重偏低（最大1.2x） | 需上调 |
| 3 | Volume | +0.1320 | 0.1469 | 启用，权重合理（最大2.0x） | 正常 |
| 4 | PK-Mom | +0.0962 | 0.0106 | 启用，level 1 近乎空集 | 小问题 |
| 5 | Drought | +0.0918 | 0.0561 | 启用但线性递增应改非线性 | 需修改 |
| 6 | Overshoot | +0.0736 | 0.0132 | 惩罚方向与数据矛盾 | 需中性化 |
| 7 | PBM | +0.0396 | 0.0245 | 正常 | 正常 |
| 8 | PeakVol | +0.0239 | 0.0097 | 无效，level 1 < level 0 | 应禁用 |
| 9 | Tests | +0.0008 | 0.0233 | 无效（p=0.93） | 应降权 |
| 10 | Age | -0.0245 | 0.0119 | 无效/微弱负相关 | 应降权 |
| 11 | Streak | -0.1936 | 0.0712 | **方向完全错误**，正向加分（1.2x/1.4x） | **P0 紧急** |

### 0.2 量化"技术债务"

- **权重分配错误**：Age（12.7%）+ Tests（10.5%）+ Streak（10.5%）合计占乘法模型 ln 权重的 33.7%，但它们的 Spearman r 总和为 -0.2173（净负值）。超过三分之一的权重被分配给零贡献或负贡献因子。
- **Height 被严重低估**：Height 贡献了 RF 重要性的 58.1%，但其 ln 权重仅占 10.5%。
- **Streak 66.2% 样本被错误加分**：level 1（38.2%）和 level 2（28.0%）合计 66.2% 的突破事件，每一个都被错误地给予了正向加分。

**核心论点**：在系统有三分之一权重分配给错误方向的前提下，讨论新增第 12、13 个因子，如同在地基歪斜的建筑上加盖楼层。

## 一、候选 1：trend_bonus -- MA 趋势过滤

### 1.1 旧报告设计

```yaml
trend_bonus:
  - price > MA200: 顺势突破 -> 1.15x
  - price > MA200 且 MA50 > MA200: 强顺势 -> 1.25x
  - price < MA50: 逆势突破 -> 0.85x
```

旧报告标记优先级：P0（最高）

### 1.2 与现有因子的信息重叠度

**重叠因子分析**：

| 已有因子 | 重叠维度 | 重叠程度 | 说明 |
|---------|---------|---------|------|
| PBM | 近期涨势 | 中等（~40%） | PBM 衡量突破前 5 日的涨势效率。price > MA200 是更宏观的趋势判断，但两者都在捕获"上涨环境中的突破"。PBM 高的突破大概率也处于 price > MA200 的状态 |
| Height | 相对高度 | 低（~15%） | Height 衡量阻力位相对近期低点的幅度。高 Height 意味着近期有大幅回调后反弹，这与趋势方向有间接关联 |
| Streak | 突破频率 | 中（~30%） | 连续突破通常发生在上涨趋势中（price > MA200）。Streak 的高频突破现象本质上就是强趋势的表征 |
| Pattern: trend_continuation | 趋势标签 | 高（~60%） | trend_continuation 模式的定义就是 Streak >= 1，这直接对应趋势中的突破 |

**综合重叠度评估**：约 40-50%

### 1.3 增量信息评估

trend_bonus 可能提供的独立信息：

1. **逆势突破的惩罚**：当前系统对 price < MA50 的突破没有扣分机制。但 Streak level 0（无近期突破）在统计上已经是 median 最高的组（0.1956），而 Streak 高（频繁突破，对应强趋势）反而最差（0.1128）。这个反直觉的数据暗示：趋势信息在当前体系中已经被 Streak 的负向效应间接编码了。

2. **顺势突破的加成**：price > MA200 的突破是否显著优于 price < MA200？这需要数据验证。但从第一性原理看，突破本身就是价格上穿阻力位的事件，绝大多数突破发生时 price 已高于 MA200（否则阻力位就不在高位）。逆势突破的占比可能极低。

3. **关键的未知**：当前 9791 个突破事件中，多大比例的 price < MA200？如果 90%+ 的突破都发生在 price > MA200 的环境下，那 trend_bonus 的筛选价值就极为有限——它只是在确认一个大多数突破已经满足的条件。

### 1.4 第一性原理分析

**"什么预测突破后涨幅"的本质维度**：

突破后涨幅取决于：(a) 突破时蓄积的供需失衡程度（Height、Volume）；(b) 突破的动量质量（DayStr、PBM）；(c) 突破的稀缺性/信号价值（Drought、Streak反向）。

趋势环境（price vs MA）属于"市场背景"维度——它不直接反映突破本身的质量，而是反映突破发生的环境。在乘法模型中，每新增一个因子都增加一层乘法关系。趋势环境作为背景变量，其信息可能已经被现有因子的组合效应间接捕获：
- 强趋势环境 -> Streak 高 -> Streak 作为惩罚因子已降分
- 弱趋势环境 -> Drought 高 -> Drought 已加分
- 趋势与价格本身的关系在 Height（阻力位高度需要足够的上涨空间）中有隐含编码

### 1.5 成本收益分析

| 维度 | 评估 |
|------|------|
| 实现复杂度 | 中等。虽然 `indicators.py` 已有 MA50 计算，但 Breakout 对象中没有 MA200 字段。需要：(1) 在 FeatureCalculator 中计算 MA200 并传入 Breakout；(2) 在 scorer 中新增评分逻辑。此外 MA200 需要至少 200 天数据，部分数据窗口可能不足 |
| 过拟合风险 | 中等。MA200/MA50 是经典指标，但将其阈值化为离散 level 时，阈值选择（"金叉"、"死叉"）本身就是一种隐式数据拟合 |
| 维护成本 | 低。MA 计算稳定，无特殊边界情况 |
| 与现有体系的兼容性 | 中等。需修改 Breakout dataclass，增加字段 |
| 边际价值（R^2 提升）| 预期极小。在 R^2=0.1051 的模型上，趋势过滤器可能增加 0.002-0.005 的 R^2 |

### 1.6 最终判定

**判定：暂缓**

**理由**：
1. trend_bonus 的信息与 Streak（反向）、Drought、PBM 有 40-50% 重叠
2. 突破事件本身具有顺势偏向——大多数突破发生在 price > MA200 时，逆势突破占比可能极低，导致筛选价值有限
3. 在 Streak 尚未修正为惩罚项的前提下，趋势信息被错误编码，新增 trend_bonus 无法纠正这个方向错误
4. 实现需要扩展 Breakout dataclass，并确保 200 天数据窗口充足
5. **应先修复 Streak 方向，再评估残余的趋势信息是否仍有增量价值**

**如果一定要做**：建议先在统计管道中加入 `price_vs_ma200` 列，跑一轮统计验证，确认其独立于 Streak 和 Drought 之后再决定。

## 二、候选 2：candle_pattern_bonus -- K 线形态

### 2.1 旧报告设计

```yaml
candle_pattern_bonus:
  - 大阳线突破（涨幅 > 3%）-> 1.20x
  - 普通阳线突破 -> 1.10x
  - 阴线/上影线突破 -> 1.0x
  - 长上影线（上影 > 实体）-> 0.85x
```

旧报告标记优先级：P0（最高）

### 2.2 与现有因子的信息重叠度

这是 6 个候选中信息重叠问题最严重的一个。

**DayStr 与 candle_pattern 的核心重叠**：

DayStr（breakout_day_strength_bonus）的计算公式：
```
daily_vol = annual_volatility / sqrt(252)
idr_ratio = intraday_change_pct / daily_vol   # 日内涨幅 / 波动率
gap_ratio = gap_up_pct / daily_vol             # 跳空幅度 / 波动率
ratio = max(idr_ratio, gap_ratio)
```

candle_pattern 的核心判据是：
- 大阳线：`intraday_change_pct > 3%` -> 本质上是 idr_ratio 的未标准化版本
- 阳线 vs 阴线：`close > open` -> intraday_change_pct 的正负号
- 上影线长度：`(high - max(open,close)) / abs(close - open)`

**维度对比**：

| 信息维度 | DayStr 是否捕获 | candle_pattern 是否捕获 | 独立程度 |
|---------|---------------|----------------------|---------|
| 日内涨幅幅度 | 是（idr_ratio） | 是（>3% 判定） | **完全重叠** |
| 涨幅的波动率标准化 | 是（除以 daily_vol） | 否（绝对百分比） | DayStr 更优 |
| 跳空强度 | 是（gap_ratio） | 否 | DayStr 已覆盖 |
| 阳线/阴线分类 | 间接（idr_ratio > 0 隐含阳线） | 是（直接分类） | 极低 |
| **上影线长度** | **否** | **是** | **唯一独立信息** |

**关键结论**：candle_pattern 与 DayStr 的信息重叠度约为 **70-80%**。唯一的增量信息是"上影线长度"——即突破日收盘是否远离日内高点。

### 2.3 增量信息评估

**上影线长度的独立价值**：

上影线（(high - close) / (high - low)）确实提供了 DayStr 未覆盖的信息。长上影线意味着突破日虽然价格曾触及高位，但最终被打回，暗示高位存在抛压。

但需要考虑：
- 长上影线的突破对应 `breakout_type = 'shadow'`，这个数据已在 Breakout 对象中计算但未纳入评分
- shadow 类型在 9791 个样本中的占比未知。如果占比极低（<5%），其惩罚效应影响的样本太少，系统级价值有限
- **上影线信息可以通过在统计管道中验证 `breakout_type` 的收益差异来评估**，而不需要新增一个独立 bonus

### 2.4 第一性原理分析

candle_pattern_bonus 本质上在问："突破日这根 K 线长什么样？"

但这个问题的 80% 已经被 DayStr 回答了：DayStr ratio 高 = 日内涨幅（或跳空）相对波动率大 = 阳线/大阳线/跳空阳线。DayStr ratio 低 = 涨幅温和。

candle_pattern 新增的 20% 是"K 线的形状细节"（上影线、实体位置）。但这些细节在乘法模型中的权重应该很小。考虑到 DayStr（r=0.1590）已经是第二强因子，candle_pattern 作为 DayStr 的"子集信息"，其独立 r 值很可能低于 0.03，与 PeakVol（r=0.0239，已被建议禁用）同级。

### 2.5 成本收益分析

| 维度 | 评估 |
|------|------|
| 实现复杂度 | 低。breakout_type 已计算，仅需映射为乘数 |
| 过拟合风险 | 中高。涨幅 > 3% 的硬阈值不适用于不同波动率的股票（3% 对 TSLA 是小事，对 BRK.B 是大事），而 DayStr 的波动率标准化天然解决了这个问题 |
| 维护成本 | 低 |
| 双重计算风险 | **高**。大阳线 1.20x + DayStr level 2 的 1.20x = 1.44x，对同一现象（强势突破日）的双重奖励 |
| 边际价值 | 极小。DayStr 已覆盖核心信息 |

### 2.6 最终判定

**判定：拒绝**

**理由**：
1. 与 DayStr 信息重叠度 70-80%，构成严重的双重计算风险
2. 唯一的独立信息（上影线）可通过在统计管道中验证 breakout_type 收益差异来评估，无需新增 bonus
3. 大阳线判定使用绝对百分比（3%），不如 DayStr 的波动率标准化科学
4. 旧报告将其标记为 P0 是在 DayStr 尚未实现时的判断。现在 DayStr 已上线，candle_pattern 的价值被大幅压缩

**替代方案**：在统计管道中加入 `breakout_type` 列（旧缺口 GAP-07，P3），验证 yang/yin/shadow 的收益差异。如果 shadow 的惩罚效应显著，可考虑在 DayStr 中增加一个修正项（如 shadow 类型时 DayStr bonus 打折），而非新增独立 bonus。

## 四、候选 4：审视 volume_bonus vs peak_volume_bonus 相关性

### 4.1 最新统计数据已给出答案

| 因子 | Spearman r | RF 重要性 | level 0 median | level 1 median | level 2 median |
|------|-----------|----------|---------------|---------------|---------------|
| volume_bonus | +0.1320*** | 0.1469 | 0.1560 | 0.2365 | 0.2800 |
| peak_volume_bonus | +0.0239* | 0.0097 | 0.1607 | **0.1548** | 0.2222 |

PeakVol 的致命问题：
- level 1（3-5x 峰值放量）的 median（0.1548）**低于** level 0（0.1607）。这意味着中等程度的峰值放量甚至是一个微弱的负面信号。
- RF 重要性 0.0097，是所有 11 个因子中**最低的**。
- 仅 level 2（>5x）有提升（0.2222），但样本仅 353 个。

### 4.2 "审视相关性"已不必要

旧报告建议"审视两者相关性"的出发点是担心双重计算。但统计数据已经表明了更根本的问题：PeakVol 本身就近乎无效。在 Volume 存在的条件下，PeakVol 的增量信息约等于零。

### 4.3 最终判定

**判定：直接禁用 PeakVol**

**理由**：
1. PeakVol 的 RF 重要性是所有因子中最低的（0.0097），不到 Volume（0.1469）的 1/15
2. level 1 低于 level 0 的非单调性说明因子的阈值设计本身就有问题
3. PeakVol 不参与 Pattern 分类的任何判定逻辑，可安全禁用
4. 无需"审视相关性"——直接设 `enabled: false` 即可

**这已在缺口报告 GAP-05 中被确认为 P1 优先级。**

## 五、候选 5：close_position_bonus -- 收盘位置

### 5.1 旧报告设计

```yaml
close_position_bonus:
  formula: position = (close - low) / (high - low)
  - position >= 0.8（收于高位）-> 1.15x
  - position >= 0.6 -> 1.05x
  - position < 0.3（收于低位）-> 0.90x
```

旧报告标记优先级：P2

### 5.2 与现有因子的信息重叠度

**与 DayStr 的重叠**：

收盘位置 = (close - low) / (high - low)。这个指标衡量的是"收盘价在日内波动范围中的位置"。

DayStr 衡量的是 `(close - open) / daily_vol` 或 `(open - prev_close) / daily_vol`。

两者的关系：
- 收于高位（position >= 0.8）-> 通常意味着 close 远高于 open（阳线）-> idr_ratio 大 -> DayStr 高
- 收于低位（position < 0.3）-> 通常意味着 close 低于 open（阴线）或长上影线 -> idr_ratio 小或负 -> DayStr 低

重叠度估计：**约 55-65%**

**与 breakout_type 的重叠**：

- yang (阳线)：close > open -> 倾向于高位收盘
- yin (阴线)：close < open -> 倾向于低位收盘
- shadow (上影线)：close 接近 open 但日内有较大波动 -> 收盘位置不确定

重叠度：约 40-50%

### 5.3 增量信息评估

close_position 相对于 DayStr 的独立信息：

1. **日内波动幅度**：position 的分母是 (high - low)，它编码了日内波动的绝对范围。DayStr 用的是年化波动率标准化，两者的标准化基准不同。
2. **下影线信息**：position 低意味着日内曾到高点但收回，DayStr 的 idr_ratio 无法区分"虽然涨了但日内有大幅回撤"和"一路平稳上涨"的区别。

但这些增量信息的预测力有多大？从第一性原理看：
- 收盘位置高意味着"买盘在收盘前仍然占优"，这是动量信号
- 收盘位置低意味着"尽管突破了，但日内抛压沉重"，这是犹豫信号
- 然而，DayStr 的 idr_ratio 已经大致捕获了这个信号——idr_ratio 高 = 收盘远高于开盘 = 大概率收于高位

### 5.4 第一性原理分析

close_position 本质上是 DayStr 的一个变体，只是用日内波动范围（high - low）做标准化而非用年化波动率。从信息论角度，它不太可能提供超过 DayStr 已有信息的 20% 以上的增量。

更重要的是：在乘法模型中，如果 close_position 和 DayStr 同时存在，对于"大阳线收于高位"的突破，两者都会给出高分，产生 DayStr(1.2x) x ClosePos(1.15x) = 1.38x 的双重奖励，而它们奖励的本质上是同一个现象。

### 5.5 成本收益分析

| 维度 | 评估 |
|------|------|
| 实现复杂度 | 低。Breakout 对象虽然没有 close_position 字段，但 high/low/close 数据在 FeatureCalculator 中易取 |
| 过拟合风险 | 中。0.8/0.6/0.3 的阈值需要数据校准 |
| 双重计算风险 | **高**。与 DayStr 55-65% 重叠 |
| 边际价值 | 小。预期独立 r 值 < 0.05 |

### 5.6 最终判定

**判定：拒绝**

**理由**：
1. 与 DayStr 信息重叠度 55-65%，双重计算风险明显
2. 从 DayStr 的波动率标准化角度看，close_position 用日内范围做标准化不如 DayStr 的年化波动率科学——后者对不同波动率的股票具有更好的可比性
3. 在 R^2=0.1051 的模型上，增加一个与 DayStr 高度相关的因子，其边际 R^2 提升可能不到 0.001
4. 旧报告自己也将其标记为 P2，不是高优先级

## 六、候选 6：liquidity_filter -- 流动性门槛

### 6.1 旧报告设计

```yaml
liquidity_penalty:
  - avg_volume < 50 万股/日 -> 0.85x
  - avg_volume < 20 万股/日 -> 0.70x
```

旧报告标记优先级：P3

### 6.2 本质问题：这不是 Bonus 因子

这是本次审计中最重要的一个概念性纠正。

流动性是数据质量问题，不是信号质量问题。把它放在 Bonus 评分体系中是类别错误（category error）：

| 维度 | Bonus 因子的本质 | 流动性门槛的本质 |
|------|----------------|---------------|
| 作用对象 | 单次突破事件的质量 | 股票本身的可交易性 |
| 信息类型 | 技术面信号强度 | 基础面约束条件 |
| 变化频率 | 每次突破不同 | 股票长期属性，缓慢变化 |
| 决策层级 | 排序和筛选 | 准入门槛（pass/fail） |
| 正确位置 | 评分模型内 | 数据预处理阶段 |

**实例说明**：一只日均成交量 10 万股的小盘股，即使其突破事件在所有 Bonus 维度上都是满分，也不应该因为评分高就被推荐——因为你可能无法以合理价格建仓。这个约束应在数据进入评分流程之前就被过滤掉。

### 6.3 当前系统的现状

系统 **已经有** 流动性过滤能力。在 `current_state.md` 中记录：

> Scan Configuration 股票筛选条件 — `min_volume`：最小平均成交量限制

这个筛选发生在 UI 扫描配置层面，在突破检测和评分之前。这正是流动性过滤应处的正确位置。

### 6.4 最终判定

**判定：拒绝（作为 Bonus 因子）**

**理由**：
1. 流动性是准入条件，不是评分因子。将其放入 Bonus 体系是类别错误。
2. 系统已有 `min_volume` 扫描配置，流动性过滤功能已存在于正确的层级。
3. 如果需要更精细的流动性处理，应增强 `min_volume` 的配置选项，而非在 Bonus 模型中新增因子。

## 八、候选 8：resistance_density_bonus -- 阻力区密度

### 8.1 旧报告设计

旧报告标记优先级：P1

### 8.2 与现有因子的信息重叠度

**与 Tests 的重叠**：

resistance_density 的分子是 `num_peaks`，这与 test_bonus 的核心输入（测试次数/峰值数量）高度相关。density 只是在 num_peaks 的基础上除以了一个价格范围标准化因子。

但 Tests（r=+0.0008, p=0.93）已被统计确认为**完全无效**。

如果 Tests 无效（峰值数量多少不影响收益），那么 resistance_density（峰值数量除以价格范围）也不太可能有效——因为分子本身就没有预测力，除以一个标准化因子不会凭空创造预测力。

### 8.3 最终判定

**判定：拒绝**

**理由**：
1. 与已确认无效的 Tests 因子高度相关
2. Tests 的 Spearman r = 0.0008（p=0.93），在分子无效的前提下，除以价格范围标准化不改变其无效性
3. 增加模型复杂度但几乎没有信息增量

## 十、对旧报告结论的修正

### 10.1 旧报告 `bonus_system_analysis.md` 的核心结论

旧报告给出的结论是：
- "当前覆盖率 80%，趋势维度和 K 线形态维度缺失"
- "通过实现 P0 优先级的两个改进（trend_bonus + candle_pattern_bonus），可将完整度提升至 90%+"

### 10.2 修正后的评估

旧报告的分析框架存在几个根本性问题：

1. **"维度覆盖率"是虚假指标**。旧报告列出了"阻力位质量、成交量、动量/涨势、波动率调整、趋势过滤、K 线形态、相对强度"7 个维度，然后按覆盖数量算覆盖率。但维度的选择本身就是主观的——你可以任意划分更多维度来人为降低覆盖率，或合并维度来提高覆盖率。这不是基于数据的分析。

2. **旧报告写于 DayStr 和 Height 实现之前**。当时 daily_return_vol_bonus 和 gap_vol_bonus 尚未合并，Height 尚未启用。在 DayStr 已实现的今天，candle_pattern 的价值被大幅压缩。

3. **旧报告未考虑 R^2 的约束**。R^2=0.1051 意味着 11 个因子合计只解释 10.5% 的收益方差。在这个基础上新增因子的边际 R^2 可能不到 0.005。旧报告忽略了这个"天花板效应"。

4. **旧报告未考虑因子间的信息冗余**。旧报告独立评估每个"缺失维度"的价值，但未分析它们与现有因子的重叠。在乘法模型中，重叠因子会产生双重计算，不是简单的"覆盖率提升"。

### 10.3 修正后的完整度评估

| 评估项 | 旧报告评分 | 修正后评分 | 理由 |
|--------|----------|----------|------|
| 维度覆盖 | 7/10 | 8/10 | DayStr 和 Height 启用后，"K 线形态"和部分"趋势"信息已被间接覆盖 |
| 指标冗余 | 7/10 | 5/10 | DayStr 合并改善了波动率维度，但 Age/Tests/PeakVol 的无效性是更严重的"冗余" |
| 权重合理性 | 8/10 | **3/10** | 统计数据揭示了严重的权重失衡：Height 被低估 3 倍，Streak 方向错误，33.7% 权重分配给负贡献因子 |
| 波动率处理 | 9/10 | 9/10 | DayStr 的设计确实优秀 |
| 可扩展性 | 9/10 | 9/10 | 乘法模型架构健全 |
| **总分** | **40/50 (80%)** | **34/50 (68%)** | 权重失衡是被低估的严重问题 |

**修正后的结论**：系统的核心问题不是"缺少维度"，而是**"现有维度的权重分配严重错误"**。将完整度从 68% 提升到 85% 的最有效路径不是新增维度，而是修正权重。
