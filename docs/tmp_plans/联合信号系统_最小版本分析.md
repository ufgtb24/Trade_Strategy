# 联合信号系统 — 最小版本分析

> 创建日期：2026-02-06

## 一、对 v2 计划的核心问题

v2 已经比 v1 好很多，但仍然存在以下问题：

| 问题 | 具体表现 |
|------|---------|
| **凭直觉定参数** | 为什么 strength 权重 0.30？为什么 decay lambda = 0.07？为什么 cap = 6.0？ |
| **凭直觉定分数** | 为什么 D→B = +25 分、B→V = +15 分、B→D = -10 分？ |
| **5 个维度 + 加权求和** | 本质上就是 PRD 反对的"多维评分系统" |
| **19 个参数** | 对于一个"简单信号计数"哲学的系统来说太多了 |

**核心矛盾**：PRD 明确说 "简单信号计数 > 多维评分"、"评分系统依赖人为权重，容易过拟合"，但 v2 正在构建一个 5 维加权评分系统。

## 二、最小版本路径（由简到难）

从第一性原理出发，当前系统唯一的排序依据是 `signal_count`。最小的改进应该是：**用信号自身已有的内在属性替换等权计数，不引入新参数。**

### Level 0（当前）：`sort by count`

```
B + D + V + Y = 4
B + B + V + Y = 4    ← 与上面相同，但实际上 2 个突破更有意义
```

### Level 1（最小改进）：`sort by weighted_sum` — 使用信号已有的内在属性

```python
def signal_weight(signal):
    if signal.signal_type == "B":
        return signal.details["pk_num"]      # 穿越几层阻力就是几
    elif signal.signal_type == "D":
        return signal.details["tr_num"]      # 几次底部确认就是几
    else:  # V, Y
        return 1.0                            # 保持简单
```

**新增参数：0 个。** pk_num=3 就是穿越了 3 层阻力，这不是人为赋权，是物理事实。

排序变化：
```
B(3) + D(2) = 5  >  B(1) + D(1) + V + Y = 4
```

这立刻就有意义了，而且完全没有主观判断。

### Level 2（加一点）：V/Y 也用内在属性

```python
# V: volume_ratio / base_threshold，超过触发阈值多少就贡献多少
# Y: sigma / base_threshold，同理
# base 使用检测器自身的触发阈值，不是新参数
```

新增参数：0 个（复用检测器已有的阈值）。但需要设 cap 防极端值，所以实际是 **2 个 cap 参数**。

### Level 3（可选）：超涨过滤

新增 **2 个参数**（threshold + lookback）。这是 v2 中最有价值的新功能之一。

核心逻辑：
- 42 天内涨幅超过 60% 的股票，其 B/V/Y 信号标记为 overextended，不计入 weighted_sum
- D 信号免疫（D 本身意味着已深度回调）

### Level 4（可选）：顺序标签（纯展示，不参与排序）

生成 `D(2) → B(3) → V` 这样的可读标签，帮助理解但**不影响排序**。**0 个参数。**

### Level 5（后续）：时间衰减、序列评分等

逐步引入更复杂的维度，但每一步都需要回测数据支撑，而非凭直觉设定。

## 四、核心原则

**让数据自己说话，不要替数据做判断。**

- pk_num=3 比 pk_num=1 强 — 这是事实（穿越了 3 层阻力位）
- tr_num=2 比 tr_num=1 可靠 — 这是事实（2 次底部确认）
- D→B 值 25 分而 B→V 值 15 分 — 这是主观判断，应避免

先实现事实部分，主观部分等有回测数据后再考虑。

## 六、建议的起步方案

| 方面 | 建议 |
|------|------|
| 起步版本 | Level 1：仅用 pk_num/tr_num 替代等权计数，0 新参数 |
| 第二步 | Level 1 + 顺序标签（纯展示）+ 可选的超涨过滤 |
| 避免的东西 | 多维加权评分、凭直觉定的分数/权重、ABCD 分级 |
| 排序方式 | `weighted_sum` 替代 `signal_count`，一维排序 |
