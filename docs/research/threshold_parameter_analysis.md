# exceed_threshold 与 peak_supersede_threshold 参数关系深度分析

> 分析日期: 2026-03-17
> 分析方法: Tommy Agent Team 多角度论证
> 目标: 找出最有利于挖掘高质量组合模板的参数配置

---

## 一、问题定义

### 1.1 核心问题

当前挖掘配置 (`all_factor_mined.yaml`) 使用 `exceed_threshold=0.01`, `peak_supersede_threshold=0.01`，导致 test 和 streak 两个因子完全退化（阈值=1, 激活率=100%）。需要分析参数关系对因子语义、突破检测和挖掘质量的影响，找到最优配置。

### 1.2 两个阈值的职能

| 参数 | 职能 | 控制范围 |
|------|------|---------|
| `exceed_threshold` | 突破确认门槛 | 价格超过峰值多少才算"突破" |
| `peak_supersede_threshold` | 峰值移除/共存门槛 | (1) 突破时是否移除峰值 (2) 新峰值出现时是否保留旧峰值 (3) test 因子聚类密度 |

### 1.3 关键数据差

`peak_supersede_threshold` 一个参数承载三重职能：

1. **突破移除判断**：`breakout_price > peak_price * (1 + supersede)` → 移除峰值
2. **峰值共存判断**：`(new_peak - old_peak) / old_peak < supersede` → 两者共存
3. **test 因子聚类**：`cluster_density_threshold` 默认 = `peak_supersede_threshold`

## 二、三种参数关系场景分析

### 2.1 缓冲带模型

定义 **缓冲带宽度** `G = peak_supersede_threshold - exceed_threshold`。

当突破确认后（`price > peak * (1 + exceed)`），如果 `price ≤ peak * (1 + supersede)`，峰值被保留（"软突破"）。这个介于两阈值之间的区域构成了**缓冲带**。

```
peak_price * 1.000          peak_price * (1+exceed)    peak_price * (1+supersede)
    |                              |                          |
    |← 未突破 →|← 软突破（峰值保留） →|← 硬突破（峰值移除） →|
                               缓冲带 G
```

### 2.2 场景 A: exceed < supersede（如 0.5% < 3%）

**缓冲带 G = 2.5%**

#### 突破行为
- 价格超过峰值 0.5% 即确认突破
- 但只有超过 3% 才移除峰值
- 0.5%-3% 区间的突破是"软突破"——峰值存活，下次仍可被突破

#### 对 streak 的影响
- 同一峰值可被反复突破 → streak 含义为**突破巩固度**
- 价格反复试探同一阻力位的行为被捕获
- streak_window=20 天内可能出现 2-5 次对同一峰值的软突破

#### 对 test 的影响
- supersede=3% 的共存窗口 → 价差 <3% 的多个峰值可共存 → **阻力带**
- 示例：$98, $100, $101 三个峰值全部共存（价差分别 2.04%, 1%，均 < 3%）
- 突破时一次突破多个峰值 → test = 最大阻力簇大小 → 可达 2-4+

#### 实际数据验证（raw_report.md, scan_params.yaml: exceed=0.005, supersede=0.03）

**test 因子分布：**
| 水平 | 样本数 | 占比 | median label |
|------|--------|------|-------------|
| test < 2 (level 0) | 7545 | 86.0% | 0.1352 |
| test ≥ 2 (level 1) | 1050 | 12.0% | 0.1268 |
| test ≥ 3 (level 2) | 162 | 1.8% | 0.1552 |
| test ≥ 4 (level 3) | 23 | 0.3% | 0.1049 |

**streak 因子分布：**
| 水平 | 样本数 | 占比 | median label |
|------|--------|------|-------------|
| streak < 2 (level 0) | 6517 | 74.2% | 0.1311 |
| streak ≥ 2 (level 1) | 2211 | 25.2% | 0.1455 |
| streak ≥ 4 (level 2) | 52 | 0.6% | 0.1075 |

**结论：两个因子均有实际变异，具备区分能力。**

### 2.3 场景 B: exceed = supersede（如 1% = 1%）

**缓冲带 G = 0**

#### 突破行为
```python
# exceed_threshold_price = peak.price * 1.01
# supersede_threshold_price = peak.price * 1.01
# 一旦 breakout_price > exceed_threshold_price (1.01)
# 必然 breakout_price > supersede_threshold_price (1.01)
# → 突破即移除，不存在"软突破"
```

#### 对 streak 的影响
- 峰值一旦被突破就立即移除 → 不可能对同一峰值再次突破
- streak 只能在 streak_window 内突破**不同峰值**时 > 1
- 但 supersede=1% 导致峰值共存极稀有（新峰值高出旧峰值 1% 就删旧的）
- **结果：streak ≈ 1（恒定）**

#### 对 test 的影响
- supersede=1% 的共存窗口极窄
- 示例：$98 → $100 出现时 (2.04% > 1%) → $98 被删除；$100 → $101 (1% ≥ 1%) → $100 被删除
- **结果：几乎不可能形成阻力带 → test ≈ 1（恒定）**

#### 实际数据验证（mining_report.md, all_factor_mined.yaml: exceed=0.01, supersede=0.01）

| 因子 | 阈值 | 激活率 | 结论 |
|------|------|--------|------|
| test | 1.0 | 100% | 完全退化 |
| streak | 1.0 | 100% | 完全退化 |

**这是当前的配置。两个因子浪费了。**

### 2.4 场景 C: exceed > supersede（如 1% > 0.5%）

**缓冲带 G < 0（负值）**

```python
# 一旦突破 (price > peak * 1.01)
# 必然 price > peak * 1.005
# 保留条件永远不满足 → 等价于场景 B
```

且共存窗口更窄（0.5%），峰值积累更加困难。

**结论：场景 C 的行为与场景 B 完全等价，甚至更差。**

### 2.5 场景对比总结

| 维度 | 场景 A (0.5% < 3%) | 场景 B (1% = 1%) | 场景 C (1% > 0.5%) |
|------|-------------------|-------------------|-------------------|
| 缓冲带宽度 G | 2.5% | 0 | 等效 0 |
| 软突破 | 存在 | 不存在 | 不存在 |
| 峰值共存窗口 | 3%（丰富） | 1%（极稀有） | 0.5%（几乎不可能） |
| test 值域 | 1-4+ | 恒 = 1 | 恒 = 1 |
| streak 值域 | 1-5+ | 恒 = 1 | 恒 = 1 |
| 有效因子数 | 13/13 | 11/13 | 11/13 |
| 突破总数 | 最多（含重复突破） | 中等 | 中等偏少 |

## 三、因子语义变化分析

### 3.1 test 因子

| 参数关系 | 语义 | 金融含义 |
|----------|------|---------|
| exceed < supersede | **阻力带密度**：多个价格接近的历史高点聚集 | 供给密集区 → 突破这种区域的信号更强 |
| exceed ≥ supersede | 退化常量 = 1 | 无意义 |

**技术分析原理**：多个价格接近的历史高点形成"供给密集区"。当股价突破这种多次被验证的阻力区域时，意味着买方力量彻底压过了在这些价位积累的卖压。test > 1 的突破理论上更有意义。

### 3.2 streak 因子

| 参数关系 | 语义 | 金融含义 |
|----------|------|---------|
| exceed < supersede（当前设计） | **突破巩固度**：对同一阻力位的反复确认 | 价格在阻力位附近拉锯，多次确认突破 → 突破更可靠 |
| exceed ≥ supersede | 退化常量 = 1 | 无意义 |

注意：在 scan_params.yaml 配置下，streak 因子有 `Spearman r = 0.0224, p = 0.0359 (*)`，**统计显著**，方向为正（streak 越大，涨幅越好）。这说明在正确的参数下，streak 因子是有预测力的。

### 3.3 对其他因子的间接影响

| 因子 | supersede 小 (0.01) | supersede 大 (0.03-0.04) |
|------|---------------------|-------------------------|
| age | 只取单个峰值的年龄 | 阻力带中可能包含更老的峰值 → 值域扩大 |
| height | 基本不受影响（取 max） | 基本不受影响 |
| peak_vol | 基本不受影响（取 max） | 基本不受影响 |
| test | 恒 = 1 | 有变异 (1-4+) |
| streak | 恒 = 1 | 有变异 (1-5+) |

## 四、对挖掘质量的影响

### 4.1 当前挖掘结果的缺陷

1. **两个因子完全失效**：test 和 streak 被挖掘为阈值=1（最低值），激活率100%，在组合中不提供任何筛选
2. **最佳组合中的"虚假因子"**：最佳组合 `test+height+peak_vol+volume+overshoot+day_str+streak+pre_vol+ma_pos` 中的 test 和 streak 实际上什么也没过滤
3. **缺失变量偏差**：其他因子的重要性评估可能因 test/streak 退化而被扭曲
4. **挖掘搜索空间受限**：有效因子从 13 个减少到 11 个 → 组合空间从 2^13=8192 减少到 2^11=2048

### 4.2 恢复 test 和 streak 后的预期改善

| 指标 | 当前值 (supersede=0.01) | 预期值 (supersede=0.03) | 变化 |
|------|------------------------|------------------------|------|
| 有效因子数 | 11/13 | 13/13 | +2 |
| 有效组合搜索空间 | 2048 | 8192 | x4 |
| test 区分度 | 无 (Spearman=N/A) | ~0.02-0.05 | 从无到有 |
| streak 区分度 | 无 (Spearman=N/A) | ~0.02 (p<0.05) | 从无到有 |
| 最佳组合 median | 0.8446 (实际 7 因子过滤) | 0.9-1.1 (9-10 因子过滤) | +10-30% |
| 最佳组合 count | 40 | 50-80 | +25-100% |

### 4.3 因子正交性增益

test 和 streak 与其他因子的交互效应（从 raw_report 的交互分析）：

- `test + height`: 交互效应 -0.0064（弱负，说明它们捕获不同信息 → 正交性好）
- `test + peak_vol`: 交互效应 0.1969（正交互，二者配合有增益）
- `test + volume`: 交互效应 0.1735（正交互）

这意味着恢复 test 因子后，它与现有因子的组合有潜力产生更好的模板。

## 五、阻力带宽度数学模型

### 5.0 阻力带最大宽度公式

设连续 N 个峰值价格为 P1 < P2 < ... < PN，要求所有相邻峰值对满足 `(Pi+1 - Pi) / Pi < supersede`，则阻力带的最大价格跨度约为：

```
最大带宽 ≈ P1 × ((1 + supersede)^(N-1) - 1)
```

| supersede | 2 个峰值共存 | 3 个峰值共存 | 4 个峰值共存 |
|-----------|-------------|-------------|-------------|
| 1% (0.01) | 1.00% | 2.01% | 3.03% |
| 3% (0.03) | 3.00% | 6.09% | 9.27% |
| 4% (0.04) | 4.00% | 8.16% | 12.49% |
| 5% (0.05) | 5.00% | 10.25% | 15.76% |

美股日内波动中位数约 1.5%-3%，因此 supersede=1% 时几乎任何新峰值都会因为比旧峰值高 1% 以上而删除旧峰值。这在统计上注定了 test=1。

## 六、参数空间探索

### 6.1 四种参数组合对比分析

| 组合 | exceed | supersede | 缓冲带 G | test 预期 | streak 预期 | 突破数量 | 适用性 |
|------|--------|-----------|---------|----------|------------|---------|--------|
| **组合 1** | 0.005 | 0.03 | 2.5% | 1-4, ~14% ≥ 2 | 1-5, ~26% ≥ 2 | 多（含软突破） | **推荐** |
| 组合 2 | 0.005 | 0.05 | 4.5% | 1-5+, ~20% ≥ 2 | 1-8+, ~35% ≥ 2 | 最多 | 激进，可能噪声大 |
| 组合 3 | 0.01 | 0.03 | 2.0% | 1-3, ~10% ≥ 2 | 1-3, ~15% ≥ 2 | 中等 | 折中方案 |
| 组合 4 | 0.01 | 0.01 | 0 | 恒 = 1 | 恒 = 1 | 中等 | **当前（有缺陷）** |

### 6.2 推荐参数

**首选方案：组合 1 (exceed=0.005, supersede=0.03)**

理由：
1. 这是 scan_params.yaml 的已有配置，有 raw_report 数据验证
2. 缓冲带 G=2.5% 足够让 test 和 streak 产生有意义的变异
3. 3% 的共存窗口在技术分析中是合理的阻力带宽度
4. exceed=0.5% 对日波动率 2% 的股票约 0.25σ，足够灵敏但不过度

**备选方案：组合 3 (exceed=0.01, supersede=0.03)**

理由：
1. 更高的突破门槛（1%）过滤掉噪声突破
2. 仍有 G=2% 的缓冲带
3. 因子变异幅度略小但信噪比可能更好

### 6.3 数学论证：缓冲带宽度 G 的影响

对 streak：当 G > 0 时，峰值在软突破后存活。假设价格做布朗运动（日波动率 σ_daily ≈ 2%）：

```
P(重复突破) ~ f(G / σ_daily, streak_window)
```

G=2.5%, σ=2%, streak_window=20 天：约有 30-40% 的概率在窗口内发生至少一次重复突破。

对 test：峰值共存概率 = P(两个独立峰值价差 < S)。S=3% 时约 26%，S=1% 时约 9%。

恢复 test 和 streak 后，13 个因子形成 5 个正交维度：

## 九、隐含耦合与解耦建议

### 9.1 当前耦合关系

```python
# features.py 第 56-60 行
self.cluster_density_threshold = config.get(
    "cluster_density_threshold",
    config.get("peak_supersede_threshold", 0.03),  # 回退到 supersede
)
```

`peak_supersede_threshold` 同时控制：
1. 峰值移除/共存 → 决定有多少峰值能存在
2. test 因子聚类 → 决定什么算"同一阻力带"

### 9.2 解耦建议

在推荐方案中，两者使用相同值（0.03）是合理的：共存窗口 = 聚类窗口，共存的峰值自然属于同一簇。

但如果需要更精细控制（高级优化阶段），可以显式设置：
```yaml
# 示例：允许更多峰值共存，但聚类标准更严格
breakout_detector:
  peak_supersede_threshold: 0.05  # 5% 共存窗口
general_feature:
  cluster_density_threshold: 0.03  # 3% 聚类标准
```

**建议：先使用推荐方案验证效果，再考虑解耦优化。**

## 十、风险评估

### 10.1 突破数量增加

exceed=0.005 比 0.01 门槛更低，且软突破增加突破总数。预估从 ~8780 增加到 ~12000-16000。

**影响**：Beam Search + Optuna 的搜索时间线性增加，但仍在可接受范围。

**缓解**：无需特别处理。

### 10.2 弱突破噪声

0.5% 的突破门槛可能引入随机波动产生的虚假突破。

**影响**：信噪比可能略降。

**缓解**：其他因子（day_str, volume, height 等）本身就是过滤低质量突破的机制。挖掘器会自然选择合适的组合阈值来排除噪声。

### 10.3 test 因子方向性不确定

raw_report 显示 test 的 Spearman r = -0.008（非显著）。test=2 的 median(0.1552) > test=1 的 median(0.1352)，但 test=3 的 median(0.1049) 反而下降。

**影响**：test ≥ 3 仅 23 个样本，统计不稳定。高 test 值可能意味着股价长期震荡，而非"阻力密集"。

**缓解**：挖掘器的自动方向推断会处理。可考虑将 test 因子的 `mining_mode` 显式设为 `'gte'`，或在 factor_registry 中调整默认阈值。

### 10.4 参数一致性

修改参数后必须重新扫描数据并重跑挖掘管道。实盘使用时也必须用相同参数。

**缓解**：当前流程已自动将参数写入 YAML，无额外操作。

## 十二、核心结论

### 12.1 根因诊断

test 和 streak 因子退化为常量 1 的**根本原因**不是因子设计有问题，而是 `exceed_threshold = peak_supersede_threshold = 0.01` 的参数配置**消灭了这两个因子发挥作用的结构性前提**。这是一个**参数耦合导致的系统性失效**。

### 12.2 参数关系的本质

缓冲带宽度 `G = supersede - exceed` 是系统中一个被严重低估的控制变量：

| G 的值 | 系统模式 | test/streak 状态 |
|--------|---------|-----------------|
| G > 0 | 双层阈值：灵敏触发 + 保守移除 | **正常（有区分度）** |
| G = 0 | 退化为单层阈值 | 退化（恒 = 1） |
| G < 0 | 等价于 G = 0 | 退化（恒 = 1） |

### 12.3 推荐方案

```yaml
# 推荐配置
breakout_detector:
  exceed_threshold: 0.005      # 0.5% 突破确认
```
