# UI Development Rules
## 对话框元素的排列规范
- 对话框应该自适应其包含的组件的大小，避免不必要的空白区域，也不要产生遮挡

## 分析模式与浏览模式的代码复用

分析模式（Analysis Mode）和浏览模式（Browse Mode）必须尽量保持代码复用：

1. **共享组件**：两种模式的通用 UI 组件应提取到公共模块
2. **数据标准化**：两种模式的数据流不一致，可以使用适配器（Adapter）将分析模式和浏览模式的原始数据转换为统一的内部格式
3. **高层级复用**：应从可能抽取的最高层级开始复用数据处理代码和底层算法
4. **UI解耦**：UI组件只消费统一格式的数据，不应感知数据来源的具体差异
5. **条件渲染优于重复实现**：优先用条件判断控制差异，而非复制相似代码
6. **模式切换**：切换模式时应复用已加载的数据和状态，避免重复请求

在修改任一模式的代码时，检查：
- 这个改动是否也适用于另一个模式？
- 是否可以抽取为共享逻辑？
