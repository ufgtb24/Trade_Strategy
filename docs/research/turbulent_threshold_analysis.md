# Turbulent Filter 阈值分析：统一 80% Amplitude 对不同价位股票的合理性

> 创建日期：2026-02-06
> 分析范围：datasets/pkls/ 中 3000 只随机抽样股票，lookback = 42 交易日

---

## Executive Summary

**结论：当前统一 80% amplitude 阈值确实对低价股存在系统性偏差，但推荐的解决方案不是调整阈值，而是利用已有的 `min_price` 筛选机制排除低价股。**

核心发现：
1. 低价股 (<$5) 被标记为 turbulent 的比率（46.5%）是中价股 ($20-50) 的 22 倍
2. 但这种"不公平"实际上是合理的 -- 低价股的高振幅反映的是真实的高风险
3. 引入价格分段阈值会增加系统复杂度且不符合 PRD 原则
4. 最优方案是**保持 80% 阈值不变，通过 `min_price >= 5` 从源头排除低价股**

---

## 一、事实验证：不同价位的振幅分布

对 1704 只有效股票（42 日 lookback 窗口）的振幅统计：

| $5-10 | 144 | 0.641 | 0.284 | 0.581 | 1.276 | 2.609 | 18.1% | 29.2% |
| $10-20 | 262 | 0.313 | 0.164 | 0.296 | 0.623 | 0.917 | 6.1% | 12.6% |

**关键观察**：
- 价格与振幅的 Spearman 相关系数为 **-0.363**，log-log Pearson 为 **-0.512**，即显著负相关
- $50 以上的股票，振幅分布几乎相同（mean 0.18-0.20，median 0.13-0.16），说明 80% 阈值对它们是一致的
- $5 以下的股票是真正的异类：median 振幅已经达到 42%-107%

### 1.2 直觉验证

**"低价股天然波动更大"这个直觉完全正确，且差异是数量级的。**

但需要区分两个问题：
- Q1: 低价股的振幅是否确实更大？ -- **是的，差异巨大**
- Q2: 这种更大的振幅是否意味着 80% 阈值"太严格"？ -- **不，这恰恰是合理的**

## 二、问题本质：80% Amplitude 对不同价位意味着什么

### 2.1 具体数字

| 股票价格 | 80% amplitude 含义 | 价格范围 | 美元波动 |
|---------|-------------------|---------|---------|
| $3 | $3 到 $5.4 | $2.4 | 常见 |
| $10 | $10 到 $18 | $8 | 显著但不罕见 |
| $50 | $50 到 $90 | $40 | 强势行情 |
| $200 | $200 到 $360 | $160 | 极端行情 |

### 2.2 关键洞察：amplitude 是百分比，已经是自标准化的

amplitude 公式 `(max_high - min_low) / min_low` 本身就是以价格为基准的百分比。一只 $3 的股票 42 天内从 $3 涨到 $5.4（80%），和一只 $200 的股票从 $200 涨到 $360（80%），**在百分比上是等价的**。

问题不在于"80% 对低价股太严格"，而在于：**低价股天然就更容易达到 80% 的百分比波动**。这不是阈值的问题，而是低价股的特性。

### 2.3 为什么低价股振幅更大？

这不是统计噪音，而是金融市场的结构性特征：

1. **微结构效应**：$0.01 的 tick 对 $2 的股票是 0.5%，对 $200 的股票是 0.005%。相同的 tick 跳动产生截然不同的百分比影响。

2. **流动性稀薄**：低价股通常成交量较低（虽然数据显示 liquid 和 illiquid 的低价股都有高振幅），少量资金就能引发大幅波动。

3. **信息不对称**：低价股多为小型/微型公司，信息透明度低，价格发现效率低，容易出现剧烈重估。

4. **幸存者偏差的反面**：许多低价股曾经是高价股，跌到低价本身就意味着它经历过极端行情。

## 三、解决方案深度分析

### 方案 A：按价格分段设置阈值

```python
def tiered_threshold(price):
    if price < 5:   return 1.5   # 150%
    elif price < 10: return 1.2  # 120%
    elif price < 50: return 1.0  # 100%
    else:            return 0.8  # 80%
```

**各价格区间被标记比例对比：**

| 区间 | 固定 80% | 分段阈值 | 变化 |
|------|---------|---------|------|
| < $2 | 58.3% | 41.7% | -16.6pp |
| $2-5 | 28.4% | 11.8% | -16.6pp |
| $5-10 | 18.1% | 11.1% | -7.0pp |
| $10-20 | 6.1% | 4.6% | -1.5pp |
| $20-50 | 2.1% | 1.5% | -0.6pp |
| $50+ | ~1% | ~1% | 不变 |

**优点**：
- 低价股不再被过度标记
- 实现简单，容易理解

**缺点**：
- 引入 4+ 个新参数（每个分段的边界和阈值），全是人为设定
- 分段边界怎么选？为什么 $5/$10/$50 而不是 $8/$15/$40？
- 直接违反 PRD 原则："不依赖人为参数"
- **核心问题**："拯救"下来的低价股（amplitude 80%-150%）真的值得买入吗？

### 方案 B：波动率标准化

```python
# amplitude / (daily_vol * sqrt(lookback))
# 如果标准化后的值 > k（如 3.0），则标记为 turbulent
```

**各价格区间被标记比例（k=3.0）：**

| 区间 | 固定 80% | VolNorm (k=3) |
|------|---------|---------------|
| < $2 | 58.3% | 46.8% |
| $2-5 | 28.4% | 14.7% |
| $5-10 | 18.1% | 7.6% |
| $10-20 | 6.1% | 5.7% |
| $20-50 | 2.1% | 3.9% |
| $50-100 | 1.0% | 3.5% |
| $100-200 | 0.0% | 2.3% |
| $200+ | 1.1% | 2.2% |

**优点**：
- 理论上最"公平"的方案，按统计异常度判断
- 自适应，不需要人工分段

**缺点**：
- 实际上**增加了高价股的误标记率**（$50-100 从 1.0% 升到 3.5%）
- 这是因为波动率标准化的哲学与 turbulent filter 的目标不一致
- Turbulent filter 的目标是**捕捉极端价格走势**，不是**捕捉统计异常值**
- 一只 $200 的稳定股突然波动 35% 在 VolNorm 下是"异常"的，但 35% 并非 turbulent 的目标场景
- 引入新的计算依赖（daily_vol），增加了复杂度
- k 值的选取同样是人为参数

### 方案 C：保持阈值 + min_price 筛选

```python
# 保持现有 AMPLITUDE_THRESHOLD = 0.8 不变
# 在 scan 配置中设置 min_price >= 5（已有功能）
```

**效果**：
- 低价股在扫描阶段就被排除，turbulent filter 只面对 $5+ 的股票
- $5+ 的股票中，80% 阈值的标记率在 0%-18% 之间，合理分布

**优点**：
- **零代码修改**，利用已有的 min_price 筛选
- 符合 PRD 原则：不增加参数
- min_price 本身是有独立意义的筛选条件（低价股流动性差、操纵风险高）
- turbulent 和价格筛选是正交的两个维度，各管各的

**缺点**：
- 不是"优化" turbulent filter 本身，而是绕开了问题
- 如果用户不设 min_price，问题仍在
- 但这实际上是一个特性，不是缺点（见下文分析）

### 方案 D：保持现状不改

**论据**：
- 80% 是 42 天内的振幅，年化约 576%，确实是极端行情
- 低价股被高比例标记 turbulent 是**合理的**，因为它们确实处于极端行情
- turbulent 不是"惩罚"，是事实描述：这只股票正在经历剧烈价格变动
- turbulent 股票的 D 信号仍然计入，不是完全放弃

## 四、与系统设计原则的一致性分析

### 4.1 PRD 核心原则回顾

1. "不依赖人为参数"
2. "简单信号计数 > 多维评分"
3. "基于价量事实 > 人为参数阈值"

### 4.2 当前 AMPLITUDE_THRESHOLD = 0.8 的定位

IMPL 文档中 D3 设计决策已经说明：

> "42 个交易日内振幅 80% = 年化约 576%。正常强势突破（30%）远不触发，只有真正的极端行情才会达到。"

这是一个合理的经验值，但确实是"人为参数"。**这是整个系统中少数几个允许存在的人为参数之一**，因为：
- amplitude 检测的是极端走势，阈值定义了"什么是极端"的边界
- 这类判断天然需要一个阈值，无法完全消除人为因素
- 与 V 检测的 volume_ratio >= 3 或 Y 检测的 2.5 sigma 类似，是系统必要的边界定义

### 4.3 引入价格分段是否违反简洁性？

**是的，显著违反。** 原因：

1. **参数爆炸**：从 1 个参数（0.8）变为 4+ 个参数（分段边界 + 每段阈值）
2. **维度耦合**：turbulent 原本只关心价格振幅，现在还要关心绝对价位，引入了不必要的维度
3. **可解释性下降**："amplitude >= 80% 为极端行情"简单直观；"$5 以下用 150%，$5-10 用 120%..." 需要更多解释
4. **无数据支撑**：分段边界和对应阈值是凭直觉设定的，没有回测依据

## 五、深层分析：被"错误标记"的低价股真的值得买入吗？

这是决定方案选择的核心问题。

### 5.1 $2-5 区间，amplitude 80%-150% 的股票样本

从数据中发现 17 只这样的股票。它们的典型特征：

```
LLNW     price=$2.46  amplitude=129.8%  (dollar range: ~$1.4)
SFUN     price=$2.80  amplitude= 80.7%  (dollar range: ~$1.3)
JELD     price=$2.46  amplitude= 95.9%  (dollar range: ~$1.2)
GALT     price=$3.88  amplitude=102.0%  (dollar range: ~$2.0)
```

这些股票在 42 天内经历了 80%-150% 的价格波动。问题是：**即使放宽阈值让它们"通过"turbulent filter，这些股票是否是好的买入候选？**

答案大概率是否定的：
- 42 天内翻倍或接近翻倍的低价股，通常意味着重大事件驱动（FDA 批准/拒绝、财报暴雷/暴涨、被做空/轧空）
- 这类行情后的走势高度不确定，B/V/Y 信号的可靠性大幅下降
- 对于一个"选股"系统来说，这类股票的风险收益比很差

### 5.2 turbulent filter 的哲学回顾

IMPL 文档明确指出 turbulent filter 的目的：

> "过滤超涨/暴涨暴跌的股票，避免在极端行情后买入"

这个目的不区分价位。不管是 $3 的股票从 $3 涨到 $5.4，还是 $200 的股票从 $200 涨到 $360，如果这种涨幅发生在 42 天内，**都代表极端行情**。低价股更容易达到 80% 不是"不公平"，而是**低价股本来就更经常处于极端行情**。

## 六、最终建议

### 推荐方案：C+D 混合 -- 保持现状 + 文档化 min_price 的建议用法

**具体行动：**

1. **AMPLITUDE_THRESHOLD = 0.8 保持不变**
   - 理由：阈值本身是合理的，低价股的高标记率不是阈值的问题，是低价股的特性

2. **在文档/UI 中建议用户设置 `min_price >= 5`**
   - 理由：这是已有功能，且有独立的投资逻辑支撑（低价股风险高、流动性差）
   - 这不是为了"修正" turbulent filter，而是基于投资的基本面考虑

3. **不引入价格分段阈值**
   - 理由：违反简洁性原则，增加人为参数，没有回测数据支撑

4. **如果未来需要细化，优先考虑的方向不是调整阈值，而是：**
   - 将 turbulent 从二值判断改为连续值（如 amplitude/threshold 的比率），实现"渐进式降权"而非"悬崖式切断"
   - 这能解决 IMPL 文档中已识别的局限 #5（"amplitude=0.79 和 0.81 的处理差距很大"）

### 为什么不是方案 B（波动率标准化）？

波动率标准化虽然理论上优雅，但：
- 它改变了 turbulent filter 的语义：从"检测极端价格走势"变成了"检测统计异常波动"
- 对高价股的误标记率显著上升（$50-100 从 1.0% 到 3.5%）
- 引入了新的计算依赖和参数（k 值）
- 解决了一个不存在的问题（低价股的高振幅不是误标记，是事实）

### 关于"渐进式降权"的预留方向

如果后续 Layer 2 需要优化 turbulent 的处理方式，可以考虑：

```
# 替代当前的 binary turbulent
# turbulent_ratio = amplitude / threshold
# effective_weight = max(0, 1 - (turbulent_ratio - 1))  当 amplitude > threshold 时逐步降权
# 这样 amplitude=0.81 的降权很小，amplitude=1.6 的降权为 0
```

但这属于 Layer 2 的范畴，当前 Layer 1 的二值判断是合理的最小版本。
