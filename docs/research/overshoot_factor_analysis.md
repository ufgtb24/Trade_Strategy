# Overshoot 因子分析：数据表现与直觉不一致的原因

## 问题

用户直觉认为 overshoot 应该是非单调因子：
- **低值** = 上涨惯性，突破温和 → 好事
- **高值** = 已经超涨，不适合买入 → 坏事

但全局 Spearman r=+0.0331（正相关），与 `mining_mode='lte'`（越低越好）矛盾。为什么？

## 计算公式回顾

```
overshoot = gain_5d / (annual_volatility / sqrt(50.4))
```

- `gain_5d`：突破日收盘价相对 5 天前收盘价的涨幅
- 分母：年化波动率缩放到 5 日周期的标准差
- **语义**：过去 5 日涨幅是该股正常 5 日波动的多少个 sigma

值域分布极度右偏（skewness=33.5），绝大部分样本集中在 0~3σ：

| 范围 | 占比 |
|------|------|
| 0~1σ | 39.1% |
| 1~2σ | 37.8% |
| 2~3σ | 14.2% |
| 3~5σ | 6.7% |
| 5σ+ | 2.1% |

## 核心发现：不是非单调，是"高方差博弈"

### 数据否定了倒 U 形假设

十分位分段分析显示，**mean label 从 D0 到 D9 单调递增**：

| 分位 | Overshoot 范围 | Mean Label | Median Label | 正 label 比例 | Label Std |
|------|---------------|-----------|-------------|--------------|-----------|
| D0 | 0.00~0.43σ | +0.271 | +0.114 | 88.4% | ~0.57 |
| D1 | 0.43~0.65σ | +0.237 | +0.100 | 88.1% | - |
| ... | ... | ... | ... | ... | - |
| D8 | 2.14~2.85σ | +0.345 | +0.133 | 86.1% | - |
| D9 | 2.85~127.6σ | +0.410 | +0.153 | 80.0% | ~1.10 |

去掉极端值后相关性依然稳健（去 top 5%: r=+0.0285），二次拟合的二次项为正（凹形而非凸形），均否定倒 U 形。

### 真实模式：期望收益上升 + 风险同步恶化

| 维度 | 低 overshoot (D0) | 高 overshoot (D9) | 变化 |
|------|-------------------|-------------------|------|
| Mean label | +0.271 | +0.410 | +51% |
| 正 label 比例 | 88.4% | 80.0% | -8.4pp |
| 大涨 (label>1.0) 比例 | 4.9% | 10.1% | +5.2pp |
| 亏损比例 | 10.5% | 19.6% | +9.1pp |
| Label std | ~0.57 | ~1.10 | ~2x |

**高 overshoot 是一个高方差博弈区域**——大涨概率翻倍（4.9%→10.1%），但亏损概率也翻倍（10.5%→19.6%）。少数极端大涨（D9 top10 的 label 达 6~21）拉高了均值，但对大多数样本而言，高 overshoot 意味着更大的不确定性。

## 为什么直觉"部分正确"

用户直觉与数据的关系：

| 直觉 | 数据验证 | 结论 |
|------|---------|------|
| 高 overshoot = 超涨 = 不适合买入 | 亏损概率确实更高（19.6% vs 10.5%） | **胜率维度正确** |
| 高 overshoot = 期望收益差 | 期望收益反而更高（+0.410 vs +0.271） | **期望值维度不正确** |
| 低 overshoot = 稳健 = 好事 | 低 overshoot 胜率更高、方差更小 | **风险维度正确** |

直觉基于"胜率/风险"视角，数据的全局 Spearman 基于"期望值"视角，两者度量的维度不同。

## mining_mode='lte' 的合理性评估

当前 `mining_mode='lte'` 硬编码的效果：
- **符合**胜率优化目标（低 overshoot 胜率更高）
- **违背**期望收益优化目标（低 overshoot 期望收益更低）
- 与全局 Spearman（r=+0.0331）方向矛盾

评分系统中 values=(0.80, 0.60) 只惩罚 >=4σ 的极端值，这个处理本身是合理的——极端超涨确实风险更大。但 mining 管道中 `lte` 模式会在整个值域上按"越低越好"搜索阈值，这在 0~3σ 的主体区间内是反直觉且反数据的。

## 建议

1. **移除 `mining_mode='lte'` 硬编码**：让 `diagnose_direction` 基于数据自动推断方向。当前数据表明全局方向应该是 `gte`（正相关）。

2. **如果想要同时捕获两端效应**（第二阶段实施时），可以将 overshoot 拆为：
   - `overshoot_high`（mining_mode='lte'）：惩罚极端超涨（>4σ），保留当前的惩罚逻辑
   - `overshoot_low`（mining_mode='gte'）：奖励适度动量，让 TPE 在 0~3σ 区间找到正向阈值

3. **当前阶段**：先观察移除硬编码后 diagnose_direction 的自动推断结果，评估是否需要拆分。
