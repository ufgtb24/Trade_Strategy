# 突破评分系统：模式识别 vs 统一评分 — 综合分析报告

**日期**: 2026-02-10
**分析团队**: math-analyst, pattern-analyst, architect

---

## Executive Summary

当前评分系统采用 Bonus 乘法模型（11 个乘数因子，基准分 50），将所有突破行为和阻力属性统一相乘产出总分。经三位分析师从数学性质、模式识别和架构决策三个角度深入分析后，核心结论如下：

**推荐方案：统一评分修复 + 模式标签（方案 C+）**
- 保持现有乘法评分框架，修复 3 个高优先级数学问题
- 新增轻量级模式分类标签，作为评分的后处理
- 标签不影响评分核心，但为下游策略选择和回测分组提供关键信息

## 一、当前系统的核心问题

### 1.1 已确认的数学问题

| 问题 | 严重程度 | 描述 |
|------|---------|------|
| gap_vol 与 idr_vol 双重计分 | **高** | 跳空高开通常日内涨幅也大，两者高度正相关（共现概率 60-80%），导致同一现象被计分两次，联合加成达 1.21~1.38x |
| overshoot penalty 过严 | **高** | Level 2 的 0.40x 可将优质突破分数压到基准分以下（示例：100分突破 → 40分），且与 volume/streak 等正向 bonus 形成几乎完全的抵消 |
| streak bonus 被低估 | **高** | 仅一级 1.10x（生产配置为 1.20x），且连续突破几乎必然触发 overshoot penalty（净效果 1.20×0.70=0.84x，反而惩罚） |
| pk_momentum 与 age 结构性互斥 | **中** | 单 peak 场景完全互斥，但这是特征本身的性质，不是评分 bug |
| 阶梯阈值不连续性 | **中** | 20天→21天差1天产生15%分数变化，21天→62天差41天产生0%变化 |

### 1.2 "通才溢价"问题的澄清

数学分析证明：**乘法模型在对数空间等价于加法模型**，它不天然偏向"全面但平庸"。真正的问题在于：
- 不同模式能触发的 bonus 数量确实不同
- 各 bonus 的最大乘数设计不对称（Age 1.50 vs Streak 1.10/1.20）
- 这导致历史阻力突破模式（可触发 5-6 个 bonus）天然比趋势延续模式（仅 1-2 个）得分高

## 二、识别的 5 种突破模式

### 模式总览

| 模式 | 核心 Bonus | 市场含义 | 典型评分范围 | 当前被低估？ |
|------|-----------|---------|-------------|------------|
| A: 势能突破 | PK-Mom, PBM, Volume | V型反弹突破近期peak，买方力量强 | 69-170 | 是（PK-Mom 上限仅1.25x） |
| B: 历史阻力突破 | Age, Tests, Height | 突破远期阻力，长期格局转变 | 71.5-200+ | 否（当前系统最优势） |
| C: 放量爆发突破 | Volume, IDR-Vol, Gap-Vol | 催化剂驱动的强力突破 | 82.5-138 | 是（被overshoot严重压制） |
| D: 密集测试突破 | Tests, PeakVol | 多次测试后突破，假突破概率低 | 71.9-109.2 | 否（评分合理） |
| E: 趋势延续突破 | Streak, PBM | 阶梯式连续突破，强趋势 | 40-71.5 | **严重**（streak被overshoot完全抵消） |

### 模式间关系

**互斥关系**：
- A（势能）vs E（趋势延续）— 强互斥（凹陷 vs 连续上涨）
- 跳空 vs 渐进 — 突破方式互斥

**叠加关系（高价值混合模式）**：
- A+B "深蹲远射"：凹陷反弹 + 远期阻力，评分潜力最高
- B+D "磨穿防线"：远期阻力 + 多次测试，最稳健
- B+C "炸穿天花板"：远期阻力 + 放量爆发，但易被overshoot惩罚

### 用户提出的两种特殊信号

| 信号 | 当前状态 | 建议 |
|------|---------|------|
| 连续 B 信号（streak） | streak_bonus 仅一级，被 overshoot 抵消 | 扩展层级 + 调整overshoot交互 |
| 久旱逢甘霖（drought） | **完全未被捕捉** | 可作为未来新 bonus 探索 |

## 三、方案评估与推荐

### 3.1 三种方案对比

| 维度 | A: 模式识别+分别评分 | B: 统一评分现状改良 | **C+: 统一评分修复+模式标签** |
|------|-------|-------|-------|
| 实现复杂度 | 高（~500-800行） | 低（~50-100行） | **中低（~150-250行）** |
| 可维护性 | 差（5套评分参数） | 好 | **好（评分与分类解耦）** |
| 与 Simple Pool 衔接 | 需大改（5种分数适配） | 无缝 | **轻微改造（标签可选用）** |
| 回测验证难度 | 极高（每套独立回测） | 低 | **中** |
| 信息丰富度 | 最高 | 低（仅一个数字） | **高（数字+模式标签）** |
| 代码改动量 | ~500-800行 | ~50-100行 | **~150-250行** |

### 3.2 推荐：方案 C+（统一评分修复 + 模式标签）

**理由**：

1. **第一性原理**：评分（排序）和策略选择（模式）是两个独立需求，应由两个机制分别满足
2. **奥卡姆剃刀**：不增加评分参数，只增加一个轻量分类层
3. **项目阶段**：MVP Phase 3 → Phase 4，应"增加观察维度"而非"增加模型复杂度"
4. **渐进演化**：如果回测证明模式间确实需要差异化策略，可从 C+ 渐进演化到 A

### 3.3 为什么不选方案 A

1. **数据不足**：~100个突破样本分到5模式后每个仅~20个，不足以支撑独立参数调优
2. **参数爆炸**：5套×5因子×2参数 = 50个参数，远超样本量
3. **分类容错性低**：错误分类 + 错误评分标准 = 双重误差
4. **与 MVP 理念冲突**：Simple Pool 从 29 参数降到 4 参数的理念

### Phase 2：模式标签（1-2 天）

**改动文件**：`breakout_scorer.py`（主要）

1. 在 `ScoreBreakdown` 中添加 `pattern_label: Optional[str] = None`
2. 新增 `_classify_pattern(bonuses)` 方法，基于主导 bonus 分类
3. 在 `get_breakout_score_breakdown_bonus()` 末尾调用分类

分类逻辑伪代码：
```python
def _classify_pattern(self, bonuses: List[BonusDetail]) -> str:
    bonuses_dict = {b.name: b for b in bonuses}
    pk_mom = bonuses_dict["PK-Mom"]
    age = bonuses_dict["Age"]
    tests = bonuses_dict["Tests"]
    vol = bonuses_dict["Volume"]
    streak = bonuses_dict["Streak"]

    # 混合模式优先判定
    if pk_mom.level >= 1 and age.level >= 2:
        return "deep_rebound"       # A+B 深蹲远射
    if vol.level >= 1 and age.level >= 2:
        return "power_historical"   # B+C 炸穿天花板
    if age.level >= 2 and tests.level >= 1:
        return "grind_through"      # B+D 磨穿防线

    # 单一模式判定
    if pk_mom.level >= 1:
        return "momentum"           # A 势能突破
    if age.level >= 2:
        return "historical"         # B 历史阻力突破
    if vol.level >= 1:
        return "volume_surge"       # C 放量爆发
    if tests.level >= 2:
        return "dense_test"         # D 密集测试
    if streak.level >= 1:
        return "trend_continuation" # E 趋势延续

    return "basic"                  # F 基础突破
```

## 附录：评分系统数学性质摘要

- **公式**：`总分 = 50 × B1 × B2 × ... × B11`
- **对数空间等价**：`log(总分) = log(50) + Σlog(Bi)`，本质是加法模型
- **理论最高分（不含overshoot）**：~573（所有bonus满级，实际不可达）
- **实际高分区间**：100-200（优质突破），200+（极罕见的多模式叠加）
- **pk_momentum vs age 互斥性**：单peak完全互斥，多peak有限共存
- **gap_vol vs idr_vol 共现率**：60-80%（需合并）
- **streak vs overshoot 对冲**：净效果通常为负（需修复）
