# 持仓管理策略分析

> 创建日期: 2024-12-28
> 状态: 分析阶段

## 一、背景

当前观察池系统的职责边界：
- **负责**: 突破检测 → 买点观察 → 生成买入信号
- **不负责**: 买入后的持仓追踪、止损止盈管理

## 二、设计思路

### 核心理念：轻量级持仓管理

```
┌─────────────────────────────────────────────────────────────┐
│  我们的系统                   券商基础设施                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  发现机会 ──► 买入信号 ──►  Trailing Stop 监控              │
│  (核心能力)                  (7x24, 高可靠)                  │
│                                                             │
│  判断重入 ◄── 闭仓通知 ◄──  止损/止盈触发                   │
│  (核心能力)                                                 │
└─────────────────────────────────────────────────────────────┘
```

**设计原则：**
1. 买入后附加券商的 Trailing Stop（跟踪止损）
2. 由券商负责实时监控，不需要我们实现复杂的持仓追踪
3. 闭仓后重新纳入我们的系统，判断是否还有操作空间

## 三、优点分析

| 优点 | 说明 |
|------|------|
| **可靠性** | 券商止损单 7x24 监控，不怕程序崩溃、网络中断 |
| **专注核心** | 系统专注"发现机会"，不需要实现复杂的实时持仓监控 |
| **简化架构** | 避免维护一个独立的持仓追踪模块 |
| **降低延迟风险** | 券商端执行止损，避免网络延迟导致的滑点 |

## 四、需要补充的细节

### 4.1 止损 vs 止盈的区分处理

```
止损闭仓:
  ├─ 快速止损 (持有 < 3天) → 可能是假突破，冷却期更长
  ├─ 正常止损 (trailing触发) → 趋势可能结束，谨慎观察
  └─ 重入条件: 需要新的突破信号，不是简单回调

止盈闭仓:
  ├─ 趋势可能仍在 → 观察回调买入机会
  └─ 重入条件: 回调到支撑位 + 企稳信号
```

### 4.2 重新纳入观察的条件

不是所有闭仓都应该重新观察：

| 场景 | 是否重新观察 | 理由 |
|------|-------------|------|
| Trailing stop 止盈后回调 | ✅ 是 | 趋势可能延续，回调是买入机会 |
| 小幅止损 (<5%) | ✅ 是 | 可能只是震荡洗盘 |
| 大幅止损 (>10%) | ❌ 否 | 趋势已反转，不再追踪 |
| 短期内多次止损 | ❌ 否 | 避免来回被割，设置冷却期 |

### 4.3 需要记录的数据结构

```python
@dataclass
class ClosedPosition:
    """闭仓记录 - 用于判断是否重新观察"""
    symbol: str
    entry_date: date
    entry_price: float
    exit_date: date
    exit_price: float
    exit_reason: str  # 'trailing_stop' | 'stop_loss' | 'take_profit'
    holding_days: int
    pnl_pct: float

    # 用于判断重入
    can_reenter: bool = True
    cooldown_until: Optional[date] = None  # 冷却期结束日期
    reentry_count: int = 0  # 重入次数，用于限制最大重入次数
```

## 五、潜在风险与应对

### 5.1 Trailing Stop 的局限性

| 局限 | 说明 | 应对方案 |
|------|------|----------|
| 固定百分比 | 券商通常只支持固定比例 trailing | 使用 ATR 计算合理幅度 |
| 无法动态调整 | 不能根据支撑位调整止损 | 接受这个限制，或使用条件单 |
| 高波动被洗 | 剧烈波动可能触发假止损 | 设置合理的 trailing 幅度 |

### 5.2 同一股票多次进出的风险

- 需要记录累计交易次数
- 设置单只股票的最大重入次数（如 3 次）
- 设置冷却期，避免频繁进出

   - 建议限制为 2-3 次，避免无限追踪

   - 回测时如何模拟券商的 trailing stop？
