# UI 观察池集成 (UI Pool Integration)

> 创建日期: 2025-12-31

## 一、概述

### 1.1 做了什么

在交互式 UI 中添加了观察池集成功能：

| 新增内容 | 位置 | 说明 |
|----------|------|------|
| "Add to Pool" 按钮 | 参数面板 | 点击将当前突破添加到观察池 |
| `_pool_mgr` 属性 | `InteractiveUI` | 观察池管理器（懒加载） |
| 观察池方法 | `InteractiveUI` | 添加、查看、清空观察池 |

### 1.2 目的

**解决的问题**：用户在 UI 中浏览突破后，无法将感兴趣的突破纳入观察池进行跟踪。

**使用场景**：
1. 用户在 UI 中浏览扫描结果
2. 发现某只股票的突破形态良好
3. 点击 "Add to Pool" 将其加入观察池
4. 后续可通过回测引擎或实盘系统跟踪买入信号

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户工作流                                │
│                                                                 │
│   浏览 K 线图  →  发现好的突破  →  Add to Pool  →  后续跟踪     │
│       │                │                │              │        │
│       ▼                ▼                ▼              ▼        │
│   ┌────────┐      ┌────────┐      ┌────────┐     ┌────────┐    │
│   │  UI    │      │ 人工   │      │ 观察池 │     │ 回测/  │    │
│   │ 显示   │      │ 筛选   │      │ 系统   │     │ 实盘   │    │
│   └────────┘      └────────┘      └────────┘     └────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 设计理念

**懒加载**：观察池管理器仅在首次使用时初始化，避免 UI 启动时的不必要开销。

```python
def _get_or_create_pool_manager(self):
    if self._pool_mgr is None:
        self._pool_mgr = create_backtest_pool_manager(date.today())
    return self._pool_mgr
```

## 二、使用方式

### 2.1 使用者类型

| 使用者 | 类型 | 说明 |
|--------|------|------|
| 终端用户 | **用户直接使用** | 通过 UI 按钮操作 |

### 2.2 用户操作步骤

#### 步骤 1：启动 UI

```bash
python -m BreakoutStrategy.UI
# 或
python scripts/visualization/interactive_ui.py
```

#### 步骤 2：加载扫描结果

点击 "Load Scan Results" → 选择 JSON 文件

#### 步骤 3：浏览突破

在左侧股票列表中点击股票，右侧显示 K 线图和突破点

#### 步骤 4：添加到观察池

找到感兴趣的突破后，点击顶部的 **"Add to Pool"** 按钮

状态栏显示：`Added 3 breakouts from AAPL to pool`

## 四、新增方法说明

### 4.1 `InteractiveUI` 类新增方法

| 方法 | 功能 | 返回值 |
|------|------|--------|
| `_get_or_create_pool_manager()` | 获取或创建观察池管理器 | `PoolManager` |
| `add_to_observation_pool()` | 将当前突破添加到观察池 | `None` |
| `show_pool_status()` | 显示观察池统计信息 | `None` |
| `clear_observation_pool()` | 清空观察池 | `None` |

### 4.2 `ParameterPanel` 类修改

| 修改 | 说明 |
|------|------|
| 新增参数 `on_add_to_pool_callback` | 构造函数新增回调参数 |
| 新增按钮 `add_to_pool_btn` | "Add to Pool" 按钮 |
| 新增方法 `_on_add_to_pool_clicked()` | 按钮点击处理 |

## 五、状态反馈

| 场景 | 状态栏显示 | 颜色 |
|------|-----------|------|
| 成功添加 | `Added 3 breakouts from AAPL to pool` | 绿色 |
| 无突破可添加 | `No breakouts to add` | 橙色 |
| 已存在（未添加新的） | `No new breakouts added (may already exist)` | 橙色 |
| 查看状态 | `Pool Status: Realtime=5, Daily=12, Total=17` | 蓝色 |
| 清空成功 | `Cleared 17 entries from pool` | 绿色 |
