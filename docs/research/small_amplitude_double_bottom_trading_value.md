# 小幅度双底 (D) 信号的交易价值分析

> 研究日期: 2026-02-06

---

## 一、执行摘要

本报告从第一性原理出发，分析 Bounce_ATR 在 1.0-2.5 倍范围内的"小幅度双底"信号是否具有交易价值，以及在低价股（$1-$5）背景下应如何设置参数。

**核心结论**：

1. **小幅度双底（1-2.5 ATR bounce）在低价股中是有意义的信号**，但其性质与大幅度双底不同 -- 它更多代表"早期积累"而非"确认反转"。
2. **不建议简单的 pass/fail 阈值**，而应采用**分层评级方案**，将 D 信号按反弹强度分为不同等级。
3. **推荐的默认参数**：Bounce_ATR=1.5, Depth_ATR=0.8（宽松捕获），配合三级评级系统进行信号质量标注。
4. **1.0 ATR 是绝对下界**：低于此值的反弹在统计上无法与日间噪声区分。

## 二、第一性原理分析：双底形态的本质

### 2.1 双底反转的物理学类比

双底形态的核心逻辑可以类比为弹性碰撞：

```
价格
 ^
 |        弹性势能 = bounce_height
 |       ↗    ↘
 |  TR1 ↗      ↘  TR2
 |  (底)          (次底, TR2 > TR1)
 +--------------------------------→ 时间
```

- **TR1**：卖压达到极致，价格触底。类比球体撞击地面。
- **反弹（Bounce）**：买方力量首次介入，消化部分卖压。类比球体弹起的高度 -- 高度反映了"地面"（支撑位）的弹性。
- **TR2**：价格再次回落但不破前低，卖压衰竭确认。类比第二次落地 -- 弹性未减意味着支撑有效。

**关键洞察**：反弹高度反映的是**买方力量相对于卖压的强度**。小反弹不意味着"无效"，而是买方力量刚开始积累。

### 2.2 反弹幅度的信息含量光谱

反弹幅度并非二元（有效/无效），而是一个信息含量的连续光谱：

| ATR 倍数 | 占价格比例 ($3 股, ATR ~$0.25) | 性质 | 信息含量 |
|----------|-------------------------------|------|----------|
| < 0.5 | < 4.2% | 纯噪声 | 几乎为零 -- 日内波动即可产生 |
| 0.5 - 1.0 | 4.2% - 8.3% | 噪声/微弱信号边界 | 极低 -- 1-2天正常波动 |
| 1.0 - 2.0 | 8.3% - 16.7% | 弱但有意义 | 中低 -- 需要持续数日买盘 |
| 2.0 - 3.0 | 16.7% - 25.0% | 中等强度 | 中 -- 明确的买方参与 |
| 3.0 - 5.0 | 25.0% - 41.7% | 强势 | 高 -- 大资金明确介入 |
| > 5.0 | > 41.7% | 极强 | 极高 -- 机构级别买入 |

### 2.3 为什么"小反弹后大涨"是合理的

用户观察到左侧小幅度 D 信号之后价格出现显著上涨，这在市场微观结构理论中有坚实基础：

**Wyckoff 积累模型（Accumulation Phase）**：

```
阶段 1: Selling Climax (SC) = TR1
阶段 2: Automatic Rally (AR) = 首次反弹（可能小幅）
阶段 3: Secondary Test (ST) = TR2
阶段 4: Sign of Strength (SOS) = 突破反弹高点
阶段 5: Markup = 主升浪
```

在 Wyckoff 模型中，**Automatic Rally 不需要很大**。它只需证明卖压暂时耗尽即可。真正的力量在 SOS 阶段才展现。

这意味着：小幅度双底可能恰恰是 Wyckoff 积累早期的信号 -- 它捕捉的是"积累开始"而非"积累完成"。

## 三、低价股背景下的波动率特征分析

### 3.1 低价股的 ATR 特性

对于 $1-$5 区间的低价股，典型的 ATR(14) 特征：

| 价格 | 典型日波动率 | 估计 ATR(14) | ATR/Price | 1 ATR 等效价格变动 |
|------|-------------|-------------|-----------|-------------------|
| $1.00 | 5-10% | $0.06 - $0.10 | 6-10% | $0.06 - $0.10 |
| $2.00 | 4-8% | $0.10 - $0.16 | 5-8% | $0.10 - $0.16 |
| $3.00 | 3-7% | $0.12 - $0.21 | 4-7% | $0.12 - $0.21 |
| $5.00 | 2-5% | $0.15 - $0.25 | 3-5% | $0.15 - $0.25 |

**关键特性**：低价股的 ATR/Price 比率天然偏高（5-10%），这意味着 1 ATR 的价格变动已经相当"显著"。

### 3.2 噪声下界的推导

从统计角度，"噪声"的定义是：在无方向性信息的情况下，纯随机波动能产生的价格变动。

对于低价股，单日高低差（日内范围）通常为 1.0-1.5 ATR。这意味着：

- **1 日随机波动**：可产生 ~1 ATR 的假反弹
- **2-3 日连续同方向随机波动**：可产生 ~1.5-2 ATR 的假反弹（概率约 12.5%-25%）
- **5 日连续同方向**：可产生 ~2.5-3 ATR 的假反弹（概率约 3%）

因此：
- **< 1.0 ATR**：高概率是噪声（>50%的日子单日就能产生这个幅度）
- **1.0 - 2.0 ATR**：灰色地带，可能是噪声也可能是弱信号
- **> 2.0 ATR**：较难仅靠随机波动产生，包含方向性信息的概率显著增加

### 3.3 低价股的特殊考量

低价股与中高价股有几个关键差异：

1. **流动性差**：买卖价差大，小额资金即可造成较大价格波动
2. **机构参与度低**：价格行为更多受散户和算法交易驱动
3. **波动率聚类效应强**：安静期和剧烈期交替出现
4. **反转前通常伴随长期下跌**：TR1 往往出现在深度下跌的末端

这些特性意味着：
- **ATR 标准化是正确的方向**（不同股票波动率差异大）
- **但 ATR 本身在低价股中波动也大**（波动率的波动率高）
- **需要更宽容的阈值**，因为真实信号也可能幅度较小

## 四、信噪比权衡的定量分析

### 4.1 参数空间的三个区域

```
信号捕获率
100% |***                           (噪声区)
     |   ****
     |       ****                    (灰色区)
     |           *****
     |                ******         (高置信区)
     |                      ********
  0% +----+----+----+----+----+---→ Bounce_ATR
     0    1    2    3    4    5
```

| 区域 | Bounce_ATR | 特性 | 适用场景 |
|------|-----------|------|----------|
| 噪声区 | 0 - 1.0 | 高捕获率，极低信噪比 | 不推荐用于交易 |
| 灰色区 | 1.0 - 2.5 | 中等捕获率，中等信噪比 | 早期发现，需配合其他验证 |
| 高置信区 | 2.5 - 5.0+ | 低捕获率，高信噪比 | 高置信度信号，适合直接行动 |

### 4.2 信噪比的量化估计

基于统计推理（假设价格日变动近似正态分布，标准差约为 ATR/1.5）：

| Bounce_ATR 阈值 | 纯随机产生概率 (5日窗口) | 估计信噪比 | 每100个信号中的真信号 |
|----------------|----------------------|-----------|---------------------|
| 0.5 | ~80% | 0.25:1 | ~20 |
| 1.0 | ~45% | 1.2:1 | ~55 |
| 1.5 | ~20% | 4:1 | ~80 |
| 2.0 | ~8% | 12:1 | ~92 |
| 2.5 | ~3% | 33:1 | ~97 |
| 3.0 | ~1% | 99:1 | ~99 |
| 4.0 | ~0.1% | ~999:1 | ~99.9 |

注意：这些是理论估计值。实际市场中由于波动率聚类、趋势延续等因素，真实信噪比可能偏离这些值。但相对关系是可靠的。

### 4.3 当前默认参数（4.0 ATR）的评估

当前配置 `first_bounce_atr: 4.0` 的含义：

- 对于 $3 股（ATR ~$0.18），要求反弹 $0.72，即 24%
- 对于 $1 股（ATR ~$0.08），要求反弹 $0.32，即 32%
- 对于 $5 股（ATR ~$0.22），要求反弹 $0.88，即 18%

**评估**：对于低价股而言，4.0 ATR 过于严格。它本质上要求 TR1 之后出现一次"迷你牛市"（20-30% 反弹），这在低价股的积累阶段可能尚未发生。用户观察到的"左侧被过滤的信号后来大涨"正是这个问题的体现。

### 4.4 用户测试参数的评估

**图4（宽松）：Bounce_ATR=1.0, Depth_ATR=0.5**
- 信噪比估计：~1.2:1（接近噪声边界）
- 优点：能捕获早期积累信号
- 风险：大量假阳性

**图5（较严）：Bounce_ATR=2.5, Depth_ATR=1.2**
- 信噪比估计：~33:1（高置信度）
- 优点：信号可靠性高
- 风险：错过早期机会

## 五、核心问题回答

### 5.1 小幅度双底在技术分析中是否有效？

**结论：有效，但性质不同。**

传统技术分析（如 Bulkowski 的 "Encyclopedia of Chart Patterns"）对双底的定义通常要求中间反弹高度为底部深度的 10-20%。但这个标准针对的是中大盘股（$20+）。

对于低价股的小幅度双底（1-2.5 ATR），它的技术分析意义是：

1. **早期积累信号**：与 Wyckoff 的 Automatic Rally 对应，标志着卖压第一次耗尽
2. **潜在支撑确认**：即使反弹幅度小，TR2 > TR1 仍然确认了支撑位
3. **前瞻性指标**：它可能出现在大幅反弹之前，提供早期入场机会

与大幅反弹（>2.5 ATR）形成的双底相比：
- **预测力更弱**：单独看，小幅度双底的反转成功率低于大幅度双底
- **期望收益可能更高**：如果确实反转，从更低/更早的位置进入，收益空间更大
- **需要更多验证**：不适合作为独立的交易信号，应配合其他信号（B、V、Y）

### 5.2 低价股中 1-2 ATR 的反弹是噪声还是信号？

**结论：处于灰色地带，需根据上下文判断。**

纯粹从统计角度：
- 1 ATR 反弹有约 45% 的概率是噪声
- 2 ATR 反弹仅有约 8% 的概率是噪声

但关键的上下文因素可以大幅改变这个判断：
- **反弹是否伴随成交量增加？**（如果是，噪声概率大幅降低）
- **反弹后是否形成了明确的 TR2 形态？**（TroughDetector 的形状确认已提供此验证）
- **是否处于长期下跌后的底部区域？**（126 日最低点的约束已保证这一点）

当前系统已有的保护机制：
1. TR1 必须是 126 日最低点 -- 排除了下跌中途的假底
2. TR2 必须通过 TroughDetector 形状确认 -- 排除了无形态的价格波动
3. TR2 > TR1 -- 确认了底部抬升

这些约束已经在一定程度上过滤了噪声。在此基础上，1.5+ ATR 的反弹具有足够的信息含量。

### 5.3 信噪比权衡的建议

**核心观点：在绝对信号系统中，不应该用参数阈值承担"评级"职能。**

当前系统的设计哲学是：
- 信号是事件（发生/未发生）
- 排序靠信号数量
- 没有评分系统

但这不意味着所有 D 信号必须被同等对待。合理的做法是：

1. **用较宽松的阈值捕获信号**（确保不遗漏）
2. **用 details 中的 bounce_atr、depth_atr 数值提供额外信息**
3. **在 UI 展示和人工分析时参考这些数值**

这符合"绝对信号"的哲学 -- 信号的"存在"由宽松阈值决定，信号的"质量"由量化指标反映。

## 六、参数建议

### 6.1 推荐的默认参数

| 参数 | 当前值 | 推荐值 | 理由 |
|------|--------|--------|------|
| `first_bounce_atr` | 4.0 | **1.5** | 在噪声下界（1.0）之上，但能捕获灰色区信号 |
| `min_tr2_depth_atr` | 2.0 | **0.8** | 与 bounce 成比例降低；TR2 形状确认已提供质量保证 |
| `min_recovery_atr` | 0.1 | **0.1** | 保持不变，仅排除 TR2 几乎等于 TR1 的退化情况 |
| `atr_period` | 14 | **14** | 标准值，无需调整 |

**推荐值的关键推导**：

`first_bounce_atr = 1.5` 的选择逻辑：
- **下界 1.0**：低于此值，纯随机噪声能产生（单日高低差 ~1 ATR）
- **安全边际 +0.5**：要求反弹超过单日波动，说明持续了至少 2-3 天
- **上界 2.5 是灰色区终点**：超过此值进入高置信区，作为"捕获"阈值过严

`min_tr2_depth_atr = 0.8` 的选择逻辑：
- TR2 深度的核心要求是"明确的回调"，不是简单的横盘
- 0.8 ATR 约等于 1 天的正常波幅上限，确保 TR2 是一个真正的下跌
- 更重要的是，TroughDetector 的形状确认（window=5, min_side_bars=2）已经保证了 TR2 的形态有效性

### 6.2 分层评级方案（推荐实施）

在保持绝对信号系统"无评分"哲学的前提下，利用 `details` 中已有的 `bounce_atr` 和 `depth_atr` 字段，定义三个质量等级：

```
D 信号质量分层：

Tier 1 - "Early" (早期信号)
  条件: 1.5 <= bounce_atr < 2.5
  含义: 潜在积累开始，需配合其他验证
  UI 标注: "D" (普通显示)

Tier 2 - "Confirmed" (确认信号)
  条件: 2.5 <= bounce_atr < 4.0
  含义: 中等强度反转信号，独立可参考
  UI 标注: "D" (加粗/不同颜色)

Tier 3 - "Strong" (强信号)
  条件: bounce_atr >= 4.0
  含义: 高置信度反转，大资金明确参与
  UI 标注: "D" (高亮)
```

**分层边界的理论基础**：

| 边界 | 值 | 理由 |
|------|-----|------|
| 噪声/信号 | 1.0 ATR | 单日波动上限，低于此为噪声 |
| Early/Confirmed | 2.5 ATR | ~3% 随机产生概率，统计显著性拐点 |
| Confirmed/Strong | 4.0 ATR | ~0.1% 随机产生概率，几乎确定非随机 |

### 6.3 实现方式建议

分层评级**不需要修改核心检测逻辑**。两种实现路径：

**路径 A（轻量级 -- 推荐）**：仅在 `details` 中增加 `tier` 字段

在 `detect()` 方法的信号生成部分，根据 `bounce_atr_ratio` 添加：

```python
# 在 details 字典中增加
"tier": (
    "strong" if bounce_atr_ratio >= 4.0
    else "confirmed" if bounce_atr_ratio >= 2.5
    else "early"
),
```

UI 展示时读取 `tier` 字段进行差异化显示。

**路径 B（完整版）**：在配置文件中定义分层阈值

```yaml
double_trough:
  first_bounce_atr: 1.5        # 捕获阈值（最低质量门槛）
  tiers:                        # 质量分层
    early_max: 2.5              # early 的上限
    confirmed_max: 4.0          # confirmed 的上限
    # strong: >= confirmed_max
```

### 6.4 Depth_ATR 的联动考量

`min_tr2_depth_atr` 与 `first_bounce_atr` 应当保持合理比例关系。逻辑上：

```
depth_atr 应 <= bounce_atr
```

因为 TR2 的回调深度不可能超过之前的反弹高度（否则 TR2 < TR1，信号本身不成立）。

推荐的比例关系：`depth_atr ≈ 0.4 ~ 0.6 * bounce_atr`

| Bounce_ATR | 推荐 Depth_ATR 范围 | 建议默认 |
|-----------|-------------------|---------|
| 1.5 | 0.6 - 0.9 | 0.8 |
| 2.5 | 1.0 - 1.5 | 1.2 |
| 4.0 | 1.6 - 2.4 | 2.0 |

当前配置 (bounce=4.0, depth=2.0) 的比例为 0.5，处于推荐范围内。
推荐配置 (bounce=1.5, depth=0.8) 的比例为 0.53，同样合理。

## 七、与现有系统的兼容性分析

### 7.1 与信号排序系统的兼容

当前系统按信号数量降序排列。降低 Bounce_ATR 阈值会增加 D 信号的总数量。

影响评估：
- **信号数量膨胀**：更多股票会获得 D 信号，可能稀释排序效果
- **缓解措施**：分层评级允许在排序时加权（如 strong D = 1.0 信号，early D = 0.5 信号）
- **但注意**：当前系统明确不使用评分/加权，因此分层信息仅供人工参考

### 7.2 与支撑分析的兼容

SupportAnalyzer 使用 D 信号的 TR2 作为支撑位基准。较小的 bounce 产生的 D 信号，其 TR2 位置仍然是有效的支撑位（因为 126 日最低点 + 形状确认的逻辑不变）。因此，支撑分析功能不受影响。

### 7.3 与 UI 展示的兼容

当前 UI 使用 "D" 标记在 K 线图上标注信号位置。分层评级可以通过：
- 不同颜色（如灰色 early、蓝色 confirmed、红色 strong）
- 不同透明度
- 标注文字（如 "D1"、"D2"、"D3"）

来区分信号等级，不需要修改核心图表渲染逻辑。

## 八、风险与注意事项

### 8.1 降低阈值的风险

1. **过拟合风险**：如果基于有限案例（如左侧信号后来大涨）来调整参数，可能是 survivorship bias
2. **信号稀释**：更多低质量 D 信号可能降低用户对 D 信号的信任度
3. **计算成本**：更多信号意味着更多的支撑分析计算

### 8.2 缓解策略

1. **分层评级**：区分信号质量，避免信任度稀释
2. **回测验证**：在实施参数调整后，应进行历史回测比较新旧参数的表现
3. **渐进式调整**：建议先从 2.5 降到 1.5，观察效果，而非直接降到 1.0

### 8.3 不建议低于 1.0 ATR 的理由

| 因素 | 说明 |
|------|------|
| 统计 | 单日高低差约 1.0-1.5 ATR，低于 1.0 ATR 的"反弹"可以在单根 K 线内完成 |
| 形态 | 如此小的反弹难以形成 TroughDetector 可识别的 TR2 形态 |
| 语义 | 双底形态的核心是"两次试探支撑后反弹"，微不足道的反弹不构成"试探" |
| 实践 | 交易成本（买卖价差 + 滑点）在低价股中可达 1-2%，小于 1 ATR 的反弹可能被交易成本吞噬 |

## 九、最终建议总结

### 9.1 参数设置

```yaml
# 推荐的低价股优化参数
double_trough:
  first_bounce_atr: 1.5    # 从 4.0 降至 1.5（捕获灰色区信号）
  min_tr2_depth_atr: 0.8   # 从 2.0 降至 0.8（与 bounce 比例协调）
  min_recovery_atr: 0.1    # 不变
  atr_period: 14            # 不变
```

### 9.2 分层方案

```
D 信号输出增加 tier 字段：
  bounce_atr < 2.5  → tier = "early"
  bounce_atr < 4.0  → tier = "confirmed"
  bounce_atr >= 4.0 → tier = "strong"
```

### 9.3 实施优先级

1. **P0（立即）**：将 `first_bounce_atr` 从 4.0 降至 1.5，将 `min_tr2_depth_atr` 从 2.0 降至 0.8
2. **P1（短期）**：在 `details` 中添加 `tier` 字段
3. **P2（中期）**：UI 差异化展示不同 tier 的 D 信号
4. **P3（后续）**：利用历史数据回测验证不同 tier 的实际反转成功率

### 9.4 回答用户的四个核心问题

**Q1：小幅度双底是否有效？**
A：有效，但性质是"早期积累信号"而非"确认反转信号"。在 Wyckoff 积累模型中，小幅度反弹对应 Automatic Rally 阶段。

**Q2：低价股中 1-2 ATR 反弹是噪声还是信号？**
A：灰色地带。1 ATR 是噪声下界，2 ATR 以上大概率包含方向性信息。1.5 ATR 作为阈值提供了合理的安全边际。

**Q3：宽松 vs 严格参数如何权衡？**
A：不应该用 pass/fail 二元思维。应降低阈值宽松捕获，用分层评级区分质量。这样既不遗漏早期信号，也能让用户识别高置信度信号。

**Q4：应该设什么参数？**
A：Bounce_ATR=1.5, Depth_ATR=0.8 作为默认值，配合三级评级系统（early/confirmed/strong）。这是对低价股波动特征的系统性适配，而非简单的"调参"。
