# Impact 聚合校准路径选择：认识论分析报告

> 2026-03-26 | Tommy 深度分析

## 问题

在 news_sentiment 模块中，用 LLM 评估的 impact 等级替代原先的 confidence 时，应该采用哪种校准路径？

**路径 A：标注驱动（先有标准，再设计公式）**
1. 用 LLM 双 agent 对合成场景做标注，得到 ground truth（sentiment + sentiment_score）
2. 基于 ground truth 做参数搜索 / 回归拟合，让公式输出逼近标注值
3. 公式设计受标注结果约束

**路径 B：公式驱动（先确定公式，再总结结果含义）**
1. 基于第一性原理和金融经验直接设计公式（损失厌恶、impact 非线性权重等）
2. 在大量真实新闻数据上运行公式
3. 观察公式输出分布，找到分层阈值（如 >0.3 算 positive，<-0.3 算 negative）
4. 逆推 sentiment_score 各区间的实际含义

本质区别：前者先有标准再设计公式，后者先确定公式再根据结果总结含义。

---

## Executive Summary

**结论：推荐路径 B（公式驱动），以路径 A 的标注结果作为"事后审计工具"而非"事前优化目标"。**

这不是两条路径的妥协折中，而是基于对问题认识论本质的重新理解。核心发现：`impact_labeling_prompt.md` 中的标注原则与 `analyzer.py` 中的公式结构编码了同一套知识——用一个的输出来校准另一个在认识论上是循环论证。已有的 43 个标注结果仍有价值，但应从"MAD 最小化目标"降级为"异常检测审计工具"。

---

## 一、认识论层面分析

### 1.1 路径 A 的知识论基础：经验主义回归

路径 A 的逻辑链路：

```
人类设计标注原则 → LLM 执行标注原则 → 参数搜索逼近 LLM 输出 → 公式参数
```

隐含核心假设：

**假设 A1：LLM 双 agent 标注是有效的 ground truth。**

从 `impact_baseline.json` 可以验证：LLM 标注的内部一致性极强（场景 1/6/37 标注值几乎相同，准确执行了"neutral 不稀释"规则），损失厌恶倍率被机械执行（场景 1 的 +0.28 × 1.43 ≈ 场景 2 的 0.40，reasoning 中明确写出了乘法计算）。

问题在于：这些一致性恰恰说明 LLM 只是 prompt 规则的忠实执行者，不存在超出规则的"隐性知识"。标注的信息量上限就是标注原则本身的信息量。

**假设 A2：标注值比公式更精确地捕捉了"正确答案"。**

在传统 ML 中成立（真人标注包含无法形式化的隐性判断），但在此场景下不成立——LLM 标注完全可被 prompt 中的 6 条原则解释。

**推论：路径 A 实际上是用一种编码方式（prompt 规则 → LLM 执行 → 数值）来校准另一种编码方式（prompt 规则 → 公式参数 → 数值），两者编码的是同一套知识。**

### 1.2 路径 B 的知识论基础：先验演绎

路径 B 的逻辑链路：

```
金融第一性原理（损失厌恶、信息非线性价值）→ 公式结构 + IMPACT_MAP → 参数推导 → 真实数据验证
```

隐含核心假设：

**假设 B1：IMPACT_MAP + 公式结构已足够捕捉新闻聚合的核心逻辑。**

当前映射的非线性已经内嵌了合理假设（negligible→low 是 4 倍跳跃，越往高端边际递减）。

**假设 B2：sentiment_score 只需排序正确和区分度够用，不需要绝对精确。**

这是任务需求的关键约束——score 是选股因子之一，用于正面筛选和负面排除，0.35 和 0.42 的差异对决策无影响。

### 1.3 认识论对比

| 维度 | 路径 A | 路径 B |
|------|--------|--------|
| 知识来源 | prompt 规则 → LLM 执行 → 数值 | 金融原理 → 公式 → 数值 |
| 信息瓶颈 | prompt 中的标注原则 | IMPACT_MAP + 公式参数 |
| 适用前提 | 标注者拥有公式无法编码的隐性知识 | 领域规则可显式形式化 |
| 此场景实际 | 隐性知识几乎为零 | 规则可直接编码 |

## 二、实践可行性分析

### 2.1 路径 A：标注的 ground truth 可靠性

三个观察：

1. **标注者过度依赖评分锚点表**：场景 1(+0.28)、场景 6(+0.30)、场景 37(+0.30) 几乎相同，是规则执行而非直觉判断。真实分析师面对"1条利好"vs"1条利好+14条背景新闻"可能有不同感受，但这种微妙差异被规则抹平。

2. **损失厌恶倍率的机械计算**：场景 1/2 的 reasoning 明确写出乘法，不是"直觉"而是"计算"。

3. **边界场景标注存疑**：场景 27（5P(low)+2N(medium)+3U(low)）标注为 negative(-0.05)。从权重看 w_p=1.0, w_n=1.0，标注为 neutral 也完全合理。这类边界场景的标注值对参数搜索的引导作用有限。

**结论：LLM 标注的可靠性上限 = prompt 规则的质量。它不提供超出规则的独立判断。**

### 2.2 路径 B：操作关键问题

**问题 1：参数初值从哪来？**

开发计划已完成理论推导：
- `_K` ≈ 0.49（基于 impact evidence 均值约 0.33 vs confidence 均值约 0.70 的比例缩放）
- 其他参数（`_GAMMA`, `_BETA` 等）操作在比率空间，理论上不受绝对值域影响，保持现值

**问题 2：如何判断输出"合理"？**

开发计划定义了 5 个理论约束替代 ground truth：
1. 跨样本量一致性（同比例 cold vs hot 的 confidence 差 < 0.08）
2. 单调性（P 比例增加 → score 增加）
3. 不对称性（mirror 场景中 |neg| > |pos|）
4. 饱和行为（指数曲线自然控制上限）
5. 冲突 vs 共识区分

这些约束是公式的结构性质，可在任何场景集上验证。

## 三、过拟合风险分析

### 3.1 路径 A：高且隐蔽

43 个场景拟合 3-5 个参数看似充裕，但合成场景存在系统性分布偏差：

- **impact 等级分布偏高端**：为测试边界行为大量使用 high/extreme，真实数据中 negligible+low 占约 60%
- **场景大小分布不均**：1-20 条均有，真实使用集中在 5-15 条
- **neutral 主导场景偏少**：真实中最常见的"大量 neutral + 少量方向性"场景覆盖不足

**核心风险：在合成场景上 MAD < 0.05 不保证真实数据上的表现。**

### 3.2 路径 B：低但需验证

参数由理论推导确定，不依赖特定场景集，无传统过拟合。风险仅在于理论假设不成立（如 impact 分布估计错误），可通过真实数据运行直接验证。

## 四、工程成本对比

| 维度 | 路径 A | 路径 B |
|------|--------|--------|
| 场景设计 | 已完成 | 不需要专用场景 |
| 标注工作 | 已完成 | 不需要 |
| 参数搜索 | 网格搜索 + 精细搜索 ~1h | 理论推导 + 敏感性扫描 ~1h |
| 验证 | MAD + 匹配率（自动化） | 5 个理论约束 + 真实数据分布 |
| **迭代成本** | **高：改原则 → 重标注 → 重搜索** | **低：改参数 → 重跑验证** |
| 真实数据验证 | 两条路径都需要 | 两条路径都需要 |

**关键差异在迭代成本。** 路径 A 修正链路长且重，路径 B 可直接修改参数并重新验证。

## 五、推荐方案：路径 B + 标注审计

### 5.1 核心思路

以路径 B 为主干（公式驱动），以路径 A 的标注结果作为审计工具。

### 5.2 标注数据的正确使用方式

将 43 个标注场景按约束强度分三级：

**A 类：硬约束（~30 个）** — 公式输出方向必须一致
- 纯同方向场景（场景 16: 4N medium → 必须 negative）
- 完美冲突（场景 22: 5P+5N → 必须 neutral）
- 极端主导（场景 2: 单条 extreme negative → 必须 negative）
- 方向不一致 = 公式有 bug

**B 类：软约束（~10 个）** — 方向可参考，幅度仅供参考
- 不对称冲突场景（场景 13: 2P(medium) + 1N(high)）
- 边界场景（场景 27: 5P(low) + 2N(medium)）
- "正确答案"本身有争议空间

**C 类：信息性（~3 个）** — 观察但不约束
- 极端值域场景（场景 42: 3x extreme negative vs 大量 positive）
- 真实中极罕见，公式行为不"疯"即可

### 5.3 执行步骤

```
Step 1: 理论推导 _K 初值 → 已完成，约 0.49
Step 2: 敏感性扫描确认 _K ∈ [0.30, 0.70]，步长 0.05
        验证 5 个理论约束，选满足约束的最优 K
Step 3: 标注审计 — 公式输出 vs impact_baseline.json
        关注方向不一致（严重异常）和幅度偏差 > 0.15（系统性分歧）
Step 4: 如方向不一致 > 2 个，调查是公式问题还是标注问题
Step 5: 真实数据验证（10-20 只股票，最近 30 天新闻）
Step 6: 基于真实数据分布最终确认参数
```

### 5.4 三条理由

1. **认识论正当性**：标注和公式编码同一套知识，用一个校准另一个是循环论证。公式驱动从第一性原理出发，知识链路更短更透明。

2. **泛化安全性**：理论推导的参数不对 43 个合成场景的特异性过拟合。

3. **迭代效率**：真实数据验证发现问题时可直接修改参数重跑，不需要回溯到标注环节。探索阶段的模块需要这种灵活性。

## 六、对路径 A 的深层回应

### 6.1 可能被低估的优势：参数交互效应

路径 A 的参数搜索天然捕捉参数间的交互效应（K 的变化影响 sufficiency，进而影响 BETA 的有效作用域）。但实际需要调整的参数仅 1 个（K），1 维搜索无交互效应问题。

### 6.2 可能被低估的优势：定量验收标准

"MAD < 0.05, sentiment_match = 100%" 比 "5 个理论约束全满足" 看起来更精确。但精确地逼近一个不精确的 ground truth，并不比粗略地满足一组结构性约束更好。

### 6.3 路径 A 唯一成立的场景

如果标注者是真人金融分析师团队（3 人以上独立标注 + 一致性检验），路径 A 的认识论基础会显著加强——因为真人具有 prompt 规则无法编码的隐性知识。但在当前 LLM 标注的设定下，这个前提不成立。
