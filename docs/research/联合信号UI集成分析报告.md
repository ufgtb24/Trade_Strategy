> **目标**：在 scan_date 找到适合买入的股票
> **日期**：2026-02-07
> **分析方法**：多 Tommy 代理并行深度分析

## Executive Summary

### 核心发现

**问题诊断**：当前系统优化的是"信号排序"，但用户真正需要的是"找到买入机会"。这是两个不同的问题：

| 维度 | 信号最多的股票 | 最值得买入的股票 |
|------|---------------|-----------------|
| 本质 | 过去 N 天发生了多少事件 | 现在是否是好的入场时机 |
| 时态 | 历史回顾 | 当下决策 |
| 可操作性 | 排序筛选 | 决策执行 |

**关键结论**：
1. **信号多 ≠ 买入机会好**：D→B 序列的 2 个信号 > V→Y→B 序列的 3 个信号
2. **时效性是缺失的关键维度**：42 天前的信号和今天的信号被同等对待是不合理的
3. **序列模式决定买入质量**：D 开头的序列大概率是好机会，V→Y→B 是追高陷阱
4. **turbulent 需要分情况**：高位危险，但低位 + D 信号可能是机会

### 推荐方案

**实施路径**：方案 A（增强列表）+ 方案 C（自动高亮），工作量约 1-2 人天

```
Phase 1: 数据增强 (0.5 天)
├── JSON 扩展：weighted_sum, sequence_label, amplitude, turbulent
└── 列表新增列 + 配置

Phase 2: 视觉高亮 (0.5 天)
├── turbulent 行红色背景
├── 高 weighted_sum 行绿色背景
└── "Hide Turbulent" 过滤器

Phase 3 (可选): 买入评分模型
└── 6 因子评分 → S/A/B/C 分层展示
```

## 二、信号序列价值分析

### 2.1 序列模式与买入价值

| 序列模式 | 市场含义 | 买入价值 | 理由 |
|----------|---------|----------|------|
| `D → B` | 底部确认后突破 | ★★★★★ | D 提供下行保护，B 确认上行方向 |
| `D (纯)` | 底部已确认，尚未启动 | ★★★★☆ | 风险最小，适合入池观察 |
| `D → B → V` | 底部突破 + 资金确认 | ★★★★★ | 形态 + 趋势 + 资金三确认 |
| `B → V → Y` | 突破后放量大涨 | ★★★☆☆ | 取决于 B 后天数 |
| `V → Y → B` | 放量大涨后突破 | ★★☆☆☆ | **追高陷阱**，大概率已错过 |

### 2.2 序列语义的量化规则

```
高价值序列:
  - D 先于 B: "底部确认"加分
  - 最后一个信号是 B 或 V: "动能新鲜"加分

低价值序列:
  - 纯 V/Y 组合无 B/D: "无趋势确认"减分
  - B 在窗口早期，后续无新信号: "动能衰减"减分
  - V → Y → B 模式: "追高陷阱"警告
```

## 三、时效性分析

### 3.1 信号时效等级

| 时效等级 | 时间范围 | 买入相关性 | 处理建议 |
|----------|----------|-----------|----------|
| **新鲜 (Fresh)** | 0-5 交易日 | 极高 | 全权重，优先关注 |
| **有效 (Valid)** | 6-21 交易日 | 中等 | 部分权重 |
| **过期 (Stale)** | 22-42 交易日 | 低 | 仅作为背景信息 |

### 3.2 各信号类型的时效敏感度

| 信号 | 时效敏感度 | 理由 |
|------|-----------|------|
| Y (Big Yang) | 极高 | 情绪信号，次日即可能反转 |
| V (High Volume) | 高 | 资金行为，3-5 天内有效 |
| B (Breakout) | 中 | 突破后回踩是常态，7-10 天内有效 |
| D (Double Trough) | 较低 | 底部形态，有效期较长（14-21 天） |

### 3.3 scan_date 当天信号的特殊价值

- **即时行动机会**：可以在次日开盘介入
- **信息新鲜**：市场刚刚发出信号
- **风险可控**：知道信号触发时的精确价位

**建议**：对 scan_date 当天的信号给予特殊加分（+5 分）

## 四、风险过滤逻辑

### 4.1 当前 turbulent 机制的局限

```python
amplitude = (max(High) - min(Low)) / min(Low)
turbulent = amplitude >= 0.8  # 80%
```

**问题**：
- 阈值硬编码，不区分高波动/低波动股票
- 二值判断，79% 和 81% 处理差距巨大
- 方向盲区，不区分"暴涨后高位"和"暴跌后低位"

### 4.2 补充风险指标建议

#### 价格位置风险 (Position Risk)
```
position_risk = (current_price - window_low) / (window_high - window_low)

position_risk > 0.8 → 高位风险警告
position_risk < 0.3 → 低位，D 信号有价值
```

**关键洞察**：turbulent + position_risk < 0.3 + D 信号 = 可能是好机会（深度回调后反转）

### 4.3 综合风险标签

| 风险等级 | 触发条件 | 处理方式 |
|----------|----------|----------|
| **Safe** | position_risk < 0.5 | 正常计入 |
| **Caution** | position_risk 0.5-0.7 | 权重衰减 20% |
| **Warning** | position_risk 0.7-0.85 | 权重衰减 50% |
| **Danger** | position_risk > 0.85 或 turbulent 高位 | 仅 D 信号计入 |

## 五、买入机会评分模型

### 5.1 6 因子加法模型 (0-100 分)

```
Buy Opportunity Score = Base + Freshness + Sequence + Diversity + Position + Momentum
                        (30)    (25)        (20)       (10)        (10)        (5)
```

| 因子 | 权重 | 计算方式 | 逻辑 |
|------|------|----------|------|
| **Base** | 30% | min(weighted_sum / 6.0, 1.0) × 30 | 信号强度基础 |
| **Freshness** | 25% | 加权新鲜度 + 当天信号 bonus | 时效性是买入时机核心 |
| **Sequence** | 20% | 模式匹配 (D→B=20, D纯=15, 追高=-10) | 序列决定买入质量 |
| **Diversity** | 10% | unique_types / 4 × 10 | 多种信号共振 |
| **Position** | 10% | (1 - position_risk) × 10 | 低位优于高位 |
| **Momentum** | 5% | 阳线+放量+站上均线 | 辅助确认 |

### 5.2 分层展示

| 档次 | 分数范围 | 标签 | 行动建议 |
|------|----------|------|----------|
| **S** | >= 75 | Strong Buy | 立即关注，考虑次日介入 |
| **A** | 60-74 | Buy | 值得分析，条件满足可入池 |
| **B** | 45-59 | Watch | 加入观察池，等待更好时机 |
| **C** | < 45 | Hold | 可能已错过或信号弱 |
