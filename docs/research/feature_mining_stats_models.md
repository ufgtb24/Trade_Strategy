## 0. 摘要（Executive Summary）

**结论：推荐 Beta-Binomial 共轭后验作为基础累积统计量，配合 Jeffreys prior + counter-evidence 整合 + 时间衰减/挑剔抽样修正项。**

### 0.1 当前累积体系的两个根本缺陷

1. **N-K 沉默样本无惩罚**：当前 `obs += K`，N-K 个未支持样本不影响信号。结果：稀有信号经过时间累积必然升级，违反统计直觉。
2. **K/N 比例信息丢失**：2/10 (+2 obs) > 1/1 (+1 obs)，但 1/1 的统计意义本应不弱于 2/10（前者后验 [50%, 100%]，后者后验 [3%, 56%]）。

### 0.2 候选模型对比矩阵（速查）

| 候选模型 | 数学严谨度 | 累加便利性 | counter 整合 | 退化能力 | 计算成本 | 解释力 | 推荐档位 |
|---|---|---|---|---|---|---|---|
| **A. Beta-Binomial 后验** | 高（贝叶斯共轭） | 极高（α/β 加法） | 自然（β 增量） | 通过衰减 | O(1) | 极高 | ★★★★★ **首选** |
| **B. 累积 LLR** | 高（Neyman-Pearson） | 极高（log 加性） | 自然（负向 LLR） | 通过衰减 | O(1) | 中（需 H0/H1 假设） | ★★★★☆ 备选 |
| **C. Wilson 区间下界** | 高（频率派） | 中（需保留 K, N） | 较弱（只能算入 N） | 受限 | O(1) | 极高（直接是概率下界） | ★★★☆☆ 备选 |
| **D. SPRT** | 高（决策理论） | 极高（log 加性） | 自然 | 弱（停时即决策） | O(1) | 中 | ★★☆☆☆ 不推荐 |
| **E. PMI** | 中（信息论） | 中（需全局边际） | 弱 | 弱 | O(N) | 高 | ★★☆☆☆ 不推荐 |

### 0.3 一句话推荐

**Beta(α, β) 共轭后验**：α 累积 K（支持），β 累积 (N-K) + γ·counter（沉默 + 加权反对），用 Beta 分布的下分位数（如 P5）作为统一信号强度。Rule 阈值 = 信号 ≥ θ_rule（如 0.4）；Hypothesis = 0 < 信号 < θ_rule；遗忘 = 信号 < θ_forget。

## §1 问题形式化

### 1.1 数据模型

每次 skill 调用产生一个 **观察事件** `e = (K, N, C, t)`：
- `K`：支持本特征的样本数（Inducer 报告）
- `N`：本轮 batch 总样本数
- `C`：counter 数（DeepSeek L1 精判中明确判 no 的样本数；硬反对证据，质量高于沉默样本）
- `t`：事件时间戳

约束：`K + C ≤ N`（K 为支持，C 为反对，N-K-C 为既未明确支持也未明确反对的"沉默样本"）。

特殊情形：
- **Batch 模式**：N 较大（如 12-30），K, C 均为整数 ≥ 0
- **Incremental 模式**：N = 1，K ∈ {0, 1}，C ∈ {0, 1}（互斥），等价于 batch size 为 1
- **统一性要求**：incremental 必须是 batch 的特例，单个公式覆盖两种情形

### 1.2 我们想要什么样的统计量 S(features)

设特征 `f` 的累积观察序列为 `{e_1, e_2, ..., e_T}`，定义信号强度 `S(f)` 应满足：

| 性质 | 形式化 |
|---|---|
| **P1 — 累加性** | `S(e_1, ..., e_T) = update(S(e_1, ..., e_{T-1}), e_T)`，O(1) 增量更新 |
| **P2 — 低 K/N 惩罚** | 给定 K 不变，N 增大时 S 不增；2/10 < 1/1 |
| **P3 — Counter 整合** | C 增大时 S 严格下降；C 的边际惩罚 ≥ N-K-C 沉默样本 |
| **P4 — 可退化** | 长期无新事件时 S 缓慢衰减；持续负证据时 S 严格下降 |
| **P5 — 数学严谨** | 对应可证明的统计量（后验、似然比、置信区间），不是凑出来的启发式 |
| **P6 — Batch/Incremental 统一** | 单条公式同时处理两种 N 的情形 |
| **P7 — 用户挑剔抽样的偏差缓解** | 允许通过修正项处理"用户故意选高质量样本"的偏差 |

### 1.3 设计自由度

我们有**两个自由度**可以选择：
1. **基础统计量**（A/B/C/D/E）：决定 P1, P2, P5, P6 怎么实现
2. **附加机制**：counter 加权 γ、时间衰减 λ、抽样修正 ρ —— 决定 P3, P4, P7 怎么实现

**重要洞察**：附加机制（counter 加权、时间衰减、抽样修正）在所有候选模型上**通用**——它们都可以折算成"等效观察数"再喂给基础统计量。所以核心选型是基础统计量。

## §2 候选模型 A — Beta-Binomial 共轭后验

### 2.1 形式定义

**先验**：Beta(α₀, β₀)
- 推荐 Jeffreys prior：α₀ = β₀ = 0.5（无信息但不退化）
- 备选：均匀先验 α₀ = β₀ = 1（更保守）

**似然**（每个事件 e = (K, N, C)）：
- 把每个支持样本看作伯努利成功 → α 增量 = K
- 把每个沉默样本看作伯努利失败（弱反对） → β 增量 = N - K - C
- 把每个 counter 样本看作伯努利失败（强反对，加权 γ ≥ 1） → β 增量 = γ · C

**后验**（共轭）：

```
α_T = α₀ + Σᵢ Kᵢ
β_T = β₀ + Σᵢ (Nᵢ - Kᵢ - Cᵢ) + γ · Σᵢ Cᵢ
posterior_T ~ Beta(α_T, β_T)
```

**信号强度 S(f)** 候选：
- `S = E[θ | data] = α / (α + β)`（后验均值）—— 简单但忽略不确定性
- `S = quantile(Beta(α, β), p)`，p ∈ {0.05, 0.10, 0.25}（**推荐**）—— 后验下分位数，对小样本天然保守
- `S = E[θ] - k · σ[θ]`，k ∈ {1, 2}（mean - k·sd）—— 工程简化版

### 2.2 累加规则

**P1 累加性**：极简，O(1)

```
对每个新事件 e = (K, N, C):
  α += K
  β += (N - K - C) + γ · C
```

**Batch / Incremental 统一**（P6）：
- Batch：α += K, β += (N-K-C) + γ·C
- Incremental（N=1）：
  - 支持样本：α += 1
  - 沉默样本：β += 1
  - Counter 样本：β += γ
  - 三种情形是同一公式的特例

### 2.3 判据（rule / hypothesis / forget）

**信号强度连续值** S = quantile(Beta(α, β), 0.05) ∈ [0, 1]

**判定**（θ 阈值待校准）：

```
S ≥ θ_rule        → rule        （建议 0.4）
θ_forget < S < θ_rule → hypothesis （建议 0.05）
S ≤ θ_forget      → forget      （建议 0.05）
```

**为什么用 0.05 分位数而不是均值**：
- 1/1 → Beta(1.5, 0.5)，mean ≈ 0.75，P5 ≈ 0.50 → 仍属 hypothesis
- 10/10 → Beta(10.5, 0.5)，mean ≈ 0.95，P5 ≈ 0.83 → rule
- 2/10 → Beta(2.5, 8.5)，mean ≈ 0.23，P5 ≈ 0.08 → hypothesis（弱）
- 100/200 → Beta(100.5, 100.5)，mean ≈ 0.50，P5 ≈ 0.44 → rule（强证据）

下分位数对样本量敏感：100/200 即使均值同 1/2 也比 1/2 强，正是我们想要的。

### 2.4 敏感性分析（数值例子）

**实验 1：固定 K，变化 N（验证 P2）**

α₀ = β₀ = 0.5，γ = 1，C = 0：

| K/N | α | β | mean | P5 |
|---|---|---|---|---|
| 1/1 | 1.5 | 0.5 | 0.75 | 0.32 |
| 1/2 | 1.5 | 1.5 | 0.50 | 0.13 |
| 1/5 | 1.5 | 4.5 | 0.25 | 0.04 |
| 1/10 | 1.5 | 9.5 | 0.14 | 0.02 |
| 1/100 | 1.5 | 99.5 | 0.015 | 0.002 |

**P2 验证通过**：K=1 不变，N 增大时 P5 单调下降。1/1 高于 1/10，符合"K=1 时 N=1 优于 N=10"的直觉。

**实验 2：固定 K/N 比例，变化绝对样本量（验证证据强度）**

K/N = 0.5，C = 0：

| K/N | α | β | mean | P5 |
|---|---|---|---|---|
| 1/2 | 1.5 | 1.5 | 0.50 | 0.13 |
| 5/10 | 5.5 | 5.5 | 0.50 | 0.27 |
| 50/100 | 50.5 | 50.5 | 0.50 | 0.42 |
| 500/1000 | 500.5 | 500.5 | 0.50 | 0.476 |

**符合直觉**：均值同为 0.5，但 P5 随样本量增大而提升，反映"更确信 50% 真值"。

**实验 3：Counter 影响（验证 P3）**

K=5，N=10，γ=1：

| C | α | β | mean | P5 |
|---|---|---|---|---|
| 0 | 5.5 | 5.5 | 0.50 | 0.27 |
| 1 | 5.5 | 5.5 | 0.50 | 0.27 |（γ=1 时与沉默无差）
| 3 | 5.5 | 5.5 | 0.50 | 0.27 |（γ=1 时与沉默无差）

γ = 1 不区分 counter 与沉默。建议 **γ ≥ 2**（counter 至少抵两个沉默样本）：

| C, γ=2 | α | β | mean | P5 |
|---|---|---|---|---|
| 0 | 5.5 | 5.5 | 0.50 | 0.27 |
| 1 | 5.5 | 6.5 | 0.46 | 0.23 |
| 3 | 5.5 | 8.5 | 0.39 | 0.18 |

**P3 验证通过**（γ=2）：C 增大 → P5 下降。建议 γ ∈ [2, 5]，由用户经验校准。

**实验 4：序列累加（验证 P1 + Episode 数）**

3 次连续观察 K=2/N=5（C=0），γ=1：

| 累积事件数 | Σα | Σβ | mean | P5 |
|---|---|---|---|---|
| 0（仅先验） | 0.5 | 0.5 | 0.50 | 0.025 |
| 1 | 2.5 | 3.5 | 0.42 | 0.13 |
| 2 | 4.5 | 6.5 | 0.41 | 0.18 |
| 3 | 6.5 | 9.5 | 0.41 | 0.20 |

**Episode 数效应**：相同 K/N=0.4 重复观察，P5 缓慢提升（0.13 → 0.20），反映"重复出现 → 更确信"。

**实验 5：1/1 vs 2/10（解决用户原始痛点）**

| 情形 | α | β | mean | P5 |
|---|---|---|---|---|
| 1/1 | 1.5 | 0.5 | 0.75 | 0.32 |
| 2/10 | 2.5 | 8.5 | 0.23 | 0.08 |

**当前体系**：obs(2/10) = 2 > obs(1/1) = 1 →  错误地认为 2/10 更强。
**Beta 后验**：P5(1/1) = 0.32 > P5(2/10) = 0.08 → **正确**反映 1/1 的统计意义不弱于 2/10。

### 2.5 优缺点分析

**优点**：
- **数学严谨**：贝叶斯共轭，每一步都对应可证明的后验更新
- **极简累加**：α, β 两个标量，O(1) 状态、O(1) 增量
- **天然 batch/incremental 统一**：单一公式覆盖
- **下分位数自动惩罚低 K/N**：1/10 不会因为时间累积而误升级
- **解释力强**：α/(α+β) 直接是真实概率的后验估计；P5 直接是"我至少有 95% 把握 θ ≥ P5"
- **退化机制可叠加**：见 §6 时间衰减
- **counter 整合自然**：通过 γ 倍权直接转化为 β 增量

**缺点**：
- **依赖 prior 选择**：α₀ = β₀ = 0.5 是 Jeffreys 默认，但若用户对此特征有强先验（如已知大致出现频率）需手动指定
- **隐含 i.i.d. 假设**：每次观察事件被假设为独立同分布的伯努利试验。**用户挑剔抽样会破坏这个假设**（详见 §2.6）
- **γ 是经验参数**：counter vs 沉默的相对权重需校准
- **下分位数计算需调用 scipy.stats.beta.ppf**：但成本极低（< 1 ms）

### 2.6 用户挑剔抽样下的偏差风险

**核心问题**：用户从 dev UI 挑选样本，**不是从 BO 总体随机抽样**。这破坏了 Beta-Binomial 的 i.i.d. 假设。

具体偏差路径：
- 用户优先挑选"看上去有特征"的样本 → K/N 整体偏高（**乐观偏差**）
- 用户回避明显异常样本 → counter 偏低（**遗漏反例**）
- 不同次 skill 调用挑的样本不独立（同一用户的偏好会延续） → 事件不独立

**缓解策略**（按推荐度排序）：

1. **抽样修正系数 ρ ∈ (0, 1]**：在更新时打折
   - α 实际增量 = ρ · K
   - β 实际增量 = ρ · ((N-K-C) + γ·C)
   - 等效于"K 个支持算 ρ·K 个有效证据"
   - 推荐：用户挑选时 ρ = 0.5；cold-start 分位抽样时 ρ = 0.8；随机抽样时 ρ = 1
2. **强反例补充**：每次 skill 启动前强制随机抽样 N_neg 个反例（不在用户选择范围内），统一以 N=N_neg, K=0 计入。等效于"主动注入 negatives"对抗用户遗漏偏差
3. **下分位数代替均值**：如 §2.3 所述，P5 已对小样本天然保守，部分缓解乐观偏差

**结论**：抽样偏差**有缓解手段**，但需要在工程层面注入修正系数 ρ。Beta-Binomial 不能自动消除偏差，但提供了清晰的注入接口。

## §3 候选模型 B — 累积对数似然比（Cumulative LLR）

### 3.1 形式定义

**两个假设**：
- H₀（null）：特征只是噪声，θ = p₀（先验基线频率，如 0.10）
- H₁（alt）：特征是真规律，θ = p₁（如 0.50）

**单事件 LLR**（Bernoulli 似然比的对数）：

```
ℓ(K, N, p₀, p₁) = K · log(p₁/p₀) + (N-K) · log((1-p₁)/(1-p₀))
```

**累积 LLR**（每事件可加性）：

```
L_T = Σᵢ ℓ(Kᵢ, Nᵢ, p₀, p₁)
```

整合 counter（γ 倍权重）：

```
ℓ(K, N, C) = K · log(p₁/p₀) + (N-K-C) · log((1-p₁)/(1-p₀)) + γC · log((1-p₁)/(1-p₀))
```

**信号强度** S = L_T，或转化为后验赔率：

```
posterior_odds = prior_odds · exp(L_T)
posterior_prob = posterior_odds / (1 + posterior_odds)
```

### 3.2 累加规则

**P1 累加性**：极简，O(1)
```
L += K · log(p₁/p₀) + (N-K-C) · log((1-p₁)/(1-p₀)) + γC · log((1-p₁)/(1-p₀))
```

**Batch / Incremental 统一**（P6）：
- 同 §2，直接代入 N

### 3.3 判据

**Wald 边界**（基于错误率 α, β）：
- A = log((1-β) / α)，B = log(β / (1-α))
- L_T ≥ A → 接受 H₁ → rule
- L_T ≤ B → 接受 H₀ → forget
- B < L_T < A → 继续观察 → hypothesis

数值例（α = 0.01, β = 0.10）：
- A = log(0.99/0.01) ≈ 4.60
- B = log(0.10/0.99) ≈ -2.30

### 3.4 敏感性分析

p₀ = 0.10, p₁ = 0.50, γ = 1：

| K/N | ℓ | 累积后判定 |
|---|---|---|
| 1/1 | log(5) + 0 = 1.61 | hypothesis |
| 1/10 | log(5) + 9·log(5/9) ≈ 1.61 - 5.30 = -3.69 | forget |
| 5/10 | 5·log(5) + 5·log(5/9) ≈ 8.05 - 2.95 = 5.10 | rule |
| 5/100 | 5·log(5) + 95·log(5/9) ≈ 8.05 - 55.92 = -47.87 | forget（极强） |

**P2 验证通过**：1/10 < 1/1（被 N-K 拖入负区域）

**实验：序列累加**

3 次 K=2/N=5（C=0），p₀=0.1, p₁=0.5：

| 累积 | L | 判定 |
|---|---|---|
| 1 | 2·log(5) + 3·log(5/9) = 3.22 - 1.77 = 1.45 | hyp |
| 2 | 2.90 | hyp |
| 3 | 4.35 | hyp（接近 A=4.60） |
| 4 | 5.80 | rule |

**符合直觉**：弱信号需要 4 次重复才升级。

**Counter 影响**（K=5, N=10, γ=2, p₀=0.1, p₁=0.5）：

| C | L |
|---|---|
| 0 | 5.10 |
| 1 | 5.10 - 2·log(5/9) ≈ 6.28 |（错了，应当是减更多）

注意：log((1-p₁)/(1-p₀)) = log(0.5/0.9) ≈ -0.59 是**负数**。所以 C 增大时 L 下降，但下降幅度看 γ。

修正：γ=2 时，C=1 → +2·log(0.5/0.9) ≈ -1.18（额外减少 1.18），L = 5.10 - 1.18 ≈ 3.92。**P3 验证通过**。

### 3.5 优缺点分析

**优点**：
- **频率派经典**：Wald 的 SPRT 理论保证（详见 §5）
- **加性结构极简**：log 加法
- **直接对接假设检验**：L_T 是充分统计量
- **错误率可控**：A, B 边界基于 α, β 解析推导

**缺点**：
- **必须先指定 p₀ 和 p₁**：基线频率 p₀ 用户难以给出准确值；p₁ 是"理想 rule 的命中率"，更主观
- **p₀, p₁ 选择敏感**：换一组参数可能颠倒结论
- **解释力略弱于 Beta**：L_T 是似然比，不直接是概率
- **Wald 边界依赖 i.i.d.**：用户挑剔抽样下 Wald 错误率保证失效（同 Beta，但 Beta 至少有"修正 ρ"的清晰路径）

### 3.6 用户挑剔抽样下的偏差风险

**比 Beta-Binomial 更严重**：Wald 错误率 α, β 的保证完全依赖 i.i.d.，挑剔抽样下边界不再有意义。可以用同样的 ρ 修正系数缓解，但解释性下降——L_T 不再是真正的对数似然比，而是"加权的伪 LLR"。

## §4 候选模型 C — Wilson Score Confidence Interval Lower Bound

### 4.1 形式定义

频率派**精确**置信区间下界（Wilson 1927）：

```
W_lower(K, N, z) = (p̂ + z²/(2N) - z·√(p̂(1-p̂)/N + z²/(4N²))) / (1 + z²/N)
其中 p̂ = K/N，z = 标准正态分位数（z=1.96 → 95% CI）
```

**信号强度** S = W_lower(K, N)

### 4.2 累加规则

**P1 累加性挑战**：Wilson 公式是**非加性的**——它直接基于 K, N，不是 α, β 这种加性参数。

**累加方案**：
- 状态：`{ΣK, ΣN}`（两个标量）
- 每事件：ΣK += K, ΣN += N - C + γC（**这里要小心**：counter 必须以"反对"身份计入 N，但不计入 K）

但**整合 counter 困难**：Wilson 默认是二元（成功/失败），没有自然位置容纳"加权反对"。强行处理：
- ΣK 不变（K 的支持仍计入）
- ΣN += N - K - C + γC + K = N + (γ-1)C（即 counter 加权膨胀 N）

这相当于把 γ·C 个虚拟"失败样本"塞进 N，破坏了 Wilson 的精确推导。

### 4.3 判据

```
S ≥ 0.4 → rule
0.05 < S < 0.4 → hypothesis
S ≤ 0.05 → forget
```

### 4.4 敏感性分析（z=1.96, 即 95% CI 下界）

| K/N | p̂ | W_lower |
|---|---|---|
| 1/1 | 1.0 | 0.21 |
| 1/2 | 0.5 | 0.09 |
| 1/10 | 0.1 | 0.02 |
| 5/10 | 0.5 | 0.24 |
| 50/100 | 0.5 | 0.40 |
| 500/1000 | 0.5 | 0.47 |

**P2 验证通过**：K=1, N 增大 → W_lower 下降。

**对比 Beta P5**：

| K/N | Beta P5 (Jeffreys) | Wilson lower (95%) |
|---|---|---|
| 1/1 | 0.32 | 0.21 |
| 5/10 | 0.27 | 0.24 |
| 50/100 | 0.42 | 0.40 |

**两者数值高度相似**（Beta P5 略高一点），符合贝叶斯-频率渐近一致性。

### 4.5 优缺点分析

**优点**：
- **频率派经典**：含义清楚（"95% 把握 θ ≥ W_lower"）
- **解释力极强**：直接是概率下界
- **数值上与 Beta P5 接近**：可作为 Beta 的 sanity check

**缺点**：
- **非加性，状态需保留 ΣK, ΣN**（虽然加性 OK，但比 Beta 多一个状态变量似乎不本质，实际上和 Beta 一样需保留两个标量）
- **Counter 整合不自然**：必须人为膨胀 N
- **不支持先验**（除非通过 prior_K, prior_N 注入，等效于 Beta 但更绕）
- **不支持时间衰减**：衰减需要重新设计（K, N 都按 λ 衰减？解释力损失）
- **挑剔抽样下精确性失效**：同 Beta/LLR

### 4.6 与 Beta-Binomial 的关系

**关键洞察**：Wilson 区间是 Jeffreys-prior Beta 后验区间的频率派"姊妹"，二者数值上高度相似。**它们的差别仅在哲学（贝叶斯 vs 频率）和参数化（α/β vs ΣK/ΣN）**，工程上几乎等价。

由于 Beta-Binomial 在以下方面更优：
- **时间衰减更自然**：α, β 各自乘以衰减因子
- **counter 整合更自然**：直接加到 β
- **先验注入更自然**：α₀, β₀ 直接体现
- **退化更自然**：可以让 α, β 同时按 λ 收缩

我们**选择 Beta 而非 Wilson**。Wilson 作为 Beta P5 的 sanity check 保留。

## §5 候选模型 D — Sequential Probability Ratio Test (SPRT)

### 5.1 形式定义

SPRT 是 Wald 的累积 LLR + Wald 边界 + **停时停止** 的完整决策框架：

```
1. 设定 H₀: θ = p₀, H₁: θ = p₁
2. 设定可接受错误率 α, β
3. 累积 L_T = Σℓᵢ
4. 停时规则：
   - L_T ≥ A → 接受 H₁，停止（→ rule）
   - L_T ≤ B → 接受 H₀，停止（→ forget）
   - B < L_T < A → 继续（→ hypothesis）
```

数学上 SPRT 在 i.i.d. 下**最优**：在固定错误率 α, β 下，**期望样本数最小**（Wald-Wolfowitz 1948）。

### 5.2 累加规则

同 §3.2（LLR）。

### 5.3 判据

A, B 边界由 α, β 决定：
- A = log((1-β)/α)
- B = log(β/(1-α))

α=0.05, β=0.10：A ≈ 2.94, B ≈ -2.25

### 5.4 优缺点分析

**优点**：
- **同 LLR 的所有优点**
- **最优性**：固定错误率下样本数最少
- **决策导向**：天然适配"何时停止"的 stopping rule

**缺点**：
- **核心问题：SPRT 设计为"做完决策就停止"**——决策做完后再不更新统计量。**不适合"长期累积、可退化"的场景**。
- **退化能力弱**：一旦决策为 rule，SPRT 不会自动退回 hypothesis；需要在外层包装"重新启动 SPRT"逻辑
- **解释力中等**：A, B 是阈值，没有直接的概率含义

### 5.5 为什么不推荐

我们的场景是 **"持续累积、可逆判定"**——rule 可以降级为 hypothesis，hypothesis 可以遗忘。SPRT 的"决策即终止"哲学根本不匹配。**LLR（候选 B）保留 SPRT 的累加能力但没有停时强制**，更合适。

## §6 候选模型 E — Pointwise Mutual Information (PMI)

### 6.1 形式定义

```
PMI(feature, breakout) = log(P(feature ∩ breakout) / (P(feature) · P(breakout)))
```

需要全局边际概率 P(feature) 和 P(breakout) ——**这是其他候选不需要的全局信息**。

### 6.2 优缺点分析

**优点**：
- **信息论严谨**：PMI 是非加性，但是从联合分布出发可解释
- **能区分"特征+突破共现"和"特征独立出现"**：直接刻画"关联性"

**缺点**：
- **致命：需要全局边际概率**——P(feature) 需要在所有 BO 样本（含未挑选样本）上统计；P(breakout) 也需要全样本
- **不适合 incremental 累加**：每次更新需要回填全局统计
- **不能直接整合 counter**

**结论**：PMI 是**离线统计指标**，不适合 Librarian 的 incremental 累积场景。**不推荐**。

## §7 共通增强机制（适用于 A, B, C 三个候选）

下面三个机制可以**正交叠加到任意候选**上，独立选型。

### 7.1 时间衰减（实现 P4 退化）

**衰减系数 λ ∈ (0, 1]**（每天衰减一次）。

**Beta-Binomial 衰减**（推荐）：
```
每日：α ← max(α₀, α · λ);  β ← max(β₀, β · λ)
```
- α, β **同时**按 λ 收缩 → 相对比 (α/β) 不变 → 后验均值不变，但**不确定性增大** → P5 下降
- 触底兜底（max with prior）防止数值消失
- λ 选择：每日衰减 0.99 → 半衰期 ≈ 69 天；λ = 0.995 → 半衰期 ≈ 138 天

**LLR 衰减**：
```
每日：L ← λ · L
```
- 简单，但解释力下降——L 不再是严格的累积 LLR

**Wilson 衰减**：人为缩 ΣK, ΣN，破坏频率派精确性

**结论**：Beta-Binomial 的衰减最自然。

### 7.2 Counter 加权 γ

定义 γ ≥ 1 为 counter 相对沉默样本的权重。

| 直觉 | γ |
|---|---|
| Counter 同沉默 | 1（不推荐） |
| Counter 等于 2 个沉默 | 2 |
| Counter 等于 5 个沉默（强反对）| 5 |
| Counter 直接证伪 hypothesis | ∞（特殊处理：触发立即遗忘） |

**推荐起点**：γ = 3，由用户根据 DeepSeek L1 精判的可靠度调整。

### 7.3 抽样偏差修正系数 ρ

定义 ρ ∈ (0, 1] 为"有效证据缩放系数"。

| 抽样方式 | ρ |
|---|---|
| 用户主动挑选 | 0.4 |
| Cold-start 分位抽样 | 0.7 |
| 随机抽样 | 1.0 |

**应用方式**：
- α 实际增量 = ρ · K
- β 实际增量 = ρ · ((N-K-C) + γ·C)

注意：ρ 同时缩放支持和反对——这保留了 K/N 比例信息，只缩小"等效观察规模"。这正是我们想要的：把"用户挑了 10 个支持"等价于"随机抽样了 4 个支持"。

## §9 推荐方案：Beta-Binomial + 三大增强

### 9.1 完整公式

**状态**：每个 feature `f` 维护 `(α_f, β_f, t_f)`，其中 t_f 是上次更新时间戳。

**先验**：α₀ = β₀ = 0.5（Jeffreys）

**事件更新**（接收 e = (K, N, C, t)）：

```python
# 步骤 1：时间衰减
days_elapsed = (t - t_f).days
decay = lambda ** days_elapsed   # lambda = 0.995, 半衰期 ~138 天
alpha_f = max(0.5, alpha_f * decay)
beta_f  = max(0.5, beta_f  * decay)

# 步骤 2：抽样修正后注入
rho = sampling_rho(source)       # 用户挑选 0.4 / cold-start 0.7 / 随机 1.0
alpha_f += rho * K
beta_f  += rho * ((N - K - C) + gamma * C)   # gamma = 3
t_f = t
```

**信号查询**：

```python
S = scipy.stats.beta.ppf(0.05, alpha_f, beta_f)   # P5
status = (
    "rule"       if S >= 0.40 else
    "hypothesis" if S >= 0.05 else
    "forget"
)
```

### 9.2 参数推荐起点

| 参数 | 推荐值 | 理由 |
|---|---|---|
| α₀, β₀ | 0.5 | Jeffreys prior，无信息但不退化 |
| γ（counter 权重） | 3 | DeepSeek 判 no = 3 个沉默样本 |
| ρ（用户挑选） | 0.4 | 强乐观偏差缓解 |
| ρ（cold-start） | 0.7 | 中等偏差 |
| ρ（随机） | 1.0 | 无偏 |
| λ（每日衰减） | 0.995 | 半衰期 ~138 天 |
| θ_rule | 0.40 | P5 ≥ 40% 即"高置信规律" |
| θ_forget | 0.05 | P5 < 5% 即"几乎肯定不是规律" |
| 分位 p | 0.05 | 95% 把握下界 |

**所有参数应在用户使用过程中校准**——这套起点值能让"1/1 → hypothesis"、"5/10 持续累积 → rule"、"2/100 → forget" 符合直觉，但实际阈值需要根据用户主观满意度调整。

### 9.3 升级/降级路径示意

```
新建特征 (K=1, N=1):
  α=0.9, β=0.5, P5=0.32 → hypothesis

下次观察 (K=2, N=5):
  α=0.9 + 0.4·2 = 1.7
  β=0.5 + 0.4·(3+0) = 1.7
  P5=0.21 → hypothesis（K/N 下降，P5 下降）

第三次 (K=5, N=10, C=0):
  α=1.7 + 0.4·5 = 3.7
  β=1.7 + 0.4·(5) = 3.7
  P5=0.28 → hypothesis（仍未达 0.40）

第四次 (K=8, N=10, C=0):
  α=3.7 + 0.4·8 = 6.9
  β=3.7 + 0.4·(2) = 4.5
  P5=0.42 → rule（升级！）

后续 30 天无观察:
  α=6.9 · 0.995^30 = 5.95
  β=4.5 · 0.995^30 = 3.88
  P5=0.40 → 临界 rule

后续 60 天无观察:
  α=5.95 · 0.995^60 = 4.40
  β=3.88 · 0.995^60 = 2.87
  P5=0.36 → 退回 hypothesis（降级！）
```

**核心要点**：
- 升级/降级**完全由 P5 阈值驱动**，无需独立的"obs/episodes"判据
- **统一了 rule/hypothesis 二分**：阈值连续刻度
- **统一了 batch/incremental**：N=1 的情形是 N=batch_size 的特例
- **天然支持退化**：长期不更新 → 衰减 → P5 下降

### 9.4 与原始 obs/episodes 判据的对比

**当前体系**：
- `obs >= 2 AND episodes >= 2` → rule
- 缺陷：obs 累加无 N 惩罚；episodes 数 ≠ 统计强度

**新体系**：
- `Beta_P5 >= 0.40` → rule
- 优势：
  - 自动整合 K, N, C 三类证据
  - 自动惩罚低 K/N
  - 自动随时间衰减
  - 单一连续值，可在 UI 中显示渐进变化（不再是离散二分）

## §10 退而求其次：极简工程版

如果用户希望**回避 scipy 依赖**或不接受贝叶斯解释，**最简化的工程版**：

```python
S = α / (α + β) - k * sqrt(α * β / ((α + β)^2 * (α + β + 1)))
                                                                # k = 1.645 → ~5% lower
```

这是 **Beta 后验均值 - k·标准差**，对中等 α, β 时数值上接近 Beta P5（精度 ±0.05）。

| K/N | Beta P5 | mean - 1.645·sd |
|---|---|---|
| 1/1 | 0.32 | 0.40 |
| 5/10 | 0.27 | 0.27 |
| 50/100 | 0.42 | 0.42 |

**结论**：极简版略高估 P5（在小样本时），但工程上无依赖、闭式公式。可作为 fallback。

## §12 风险与开放问题

### 12.1 数学层面的开放问题

**Q1：γ 和 ρ 的真值如何标定？**
- 我们建议起点 γ=3, ρ_user=0.4，但这些是工程经验值
- 严格做法：从历史 BO 样本中通过 cross-validation 优化
- 当前数据量（几十到几百 BO）下不足以做严格标定，建议**先用经验值跑 3-6 个月再校准**

**Q2：Jeffreys prior (0.5, 0.5) 还是 Uniform (1, 1)？**
- Jeffreys：1/1 时 P5 = 0.32（更宽容）
- Uniform：1/1 时 P5 = 0.13（更保守）
- 推荐 **Jeffreys**——给单次强信号合理的初始置信，避免"过度保守"导致 1/1 永远进不了 hypothesis

**Q3：衰减是否应同时 α, β 等比例缩放？**
- 等比例缩放：均值不变，方差增大（推荐）
- 只缩 α：均值下降，方差变化复杂（不推荐）
- 推荐**等比例**——退化 = 不确定性增大，符合直觉

### 12.2 工程层面的开放问题

**Q4：每日衰减 vs 事件触发衰减？**
- 每日衰减：cron job，所有特征同步衰减（推荐）
- 事件触发：每次访问时按 elapsed days 衰减（lazy，更轻量）
- 推荐 **lazy 衰减**——避免每天扫全库

**Q5：Beta P5 计算是否需要缓存？**
- 每次查询调用 `scipy.stats.beta.ppf` < 1 ms
- 千个特征查询总成本 < 1 秒
- 不需要缓存

**Q6：升级/降级是否需要 hysteresis（迟滞带）？**
- 当前设计：S=0.40 是 rule/hypothesis 边界
- 风险：S 在 0.40 附近震荡时频繁切换状态
- 建议：rule → hypothesis 用 0.35（迟滞 0.05），hypothesis → rule 用 0.40
- 这是工程优化，可在使用中决定

### 12.3 抽样偏差的根本性风险

**最严肃的风险**：Beta-Binomial 假设 i.i.d. 伯努利试验，**用户挑剔抽样会产生系统性偏差**，ρ 修正只是**部分**缓解。

**这个偏差不能被 §9 的方案完全消除**。可叠加的额外缓解（按推荐度）：

1. **强制反例注入**：每次 skill 启动前**自动**抽 N_neg 个 cold-start 反例（不在用户挑选范围），强制以 K=0 计入。这是对"用户遗漏反例"的硬抗。
2. **多次独立抽样平均**：让用户在不同时段独立挑选，多次的样本视为独立 episodes
3. **仅在主要决策（rule 升级）时检查**：rule 升级时强制 hold-out 验证（同 `feature_mining_philosophy_decision.md` §4.3）

这三条在 Phase 1-3 不一定全做，但**长期 Phase 4+ 必须做**。

## §13 推荐方案的一段话总结

**Beta(α, β) 共轭后验作为 Librarian 累积统计量**：

- α₀ = β₀ = 0.5（Jeffreys prior）
- 每次观察 (K, N, C)：α += ρ·K，β += ρ·((N-K-C) + γ·C)
- 每日 lazy 衰减：α, β ← max(prior, · λ)
- 信号 S = Beta_P5(α, β)，rule 阈值 0.40，forget 阈值 0.05
- 起点参数：γ=3，ρ_user=0.4，ρ_cold=0.7，ρ_random=1.0，λ=0.995

**统一了**：
- rule / hypothesis 连续刻度（无需 obs / episodes 二分）
- batch / incremental（incremental 是 batch=1 的特例）
- 支持 / 沉默 / counter 三类证据
- 升级 / 降级 / 退化（衰减驱动）

**唯一脆弱点**：用户挑剔抽样的 i.i.d. 假设破坏，需 ρ 修正系数 + 长期反例注入双重缓解。

## §14 推荐选择

**首选：Beta-Binomial 共轭后验（候选 A） + Jeffreys prior + 时间衰减 + counter 加权 + 抽样修正**

**理由（一句话）**：在七大性质评分中得 34/35（其他候选最多 31），数学严谨度与工程极简性并存——状态仅 (α, β) 两个标量，更新 O(1)，查询 O(1)，且天然支持时间衰减、counter 整合、抽样修正三个独立维度的扩展。
