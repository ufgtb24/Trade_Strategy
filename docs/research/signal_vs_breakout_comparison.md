# 联合信号选股 vs 突破选股：综合对比分析报告

> 日期：2026-02-09
> 分析师：comparator (综合对比)，基于 signal-analyst 和 breakout-analyst 的深度分析

---

## Executive Summary

**核心结论：两种方法不是替代关系，而是互补关系。建议采用"双层架构"融合方案。**

联合信号选股（下称"信号系统"）和突破选股（下称"突破系统"）分别代表了两种不同的选股哲学：

- **信号系统**：多信号源广撒网 + 简单计数排序——追求"宽度"，即从多个独立维度（突破/放量/大阳线/双底）确认市场异常，但对单一信号的评估较为粗放。
- **突破系统**：单事件源深钻研 + 精细评分排序——追求"深度"，即对一种事件（突破）进行极致量化（20+特征、12个评分因子），但视角单一。

两者的重叠部分（B信号 ≈ 突破系统的严格子集）恰好提供了自然的融合接口。**建议方案：以信号系统作为"初筛层"提供多维度候选池，以突破系统的质量评分作为"精排层"对B信号候选进行深度打分，V/Y/D信号保留独立通道。** 这样既保留了信号系统的广度和抗过拟合特性，又利用了突破系统对突破质量的精细区分能力。

---

## 一、多维度对比分析

### 1.1 选股理念对比

| 维度 | 联合信号系统 | 突破选股系统 |
|------|-------------|-------------|
| **核心哲学** | "多信号共识 = 高确信度" | "突破质量越高 = 成功概率越大" |
| **方法论** | 信号计数/加权（宽度优先） | 多维特征评分（深度优先） |
| **信号定义** | 市场结构事件的布尔检测 | 单一事件的连续值量化 |
| **排序依据** | weighted_sum（自然计数加权和） | quality_score（Bonus乘法模型） |
| **设计倾向** | 反过拟合，宁可粗放 | 精细建模，追求区分度 |

**分析**：

信号系统的核心假设是"独立信号的联合出现本身就是强信号"——如果一只股票同时出现突破、放量、大阳线，那么不需要精确量化每个事件的"质量"，联合出现这一事实已经提供了足够的选股依据。这是一个统计学上合理的假设：多个弱分类器的组合（boosting思想的朴素版本）。

突破系统的核心假设是"同样是突破，突破300天前的阻力位和突破30天前的完全不同"——它需要精确量化突破的各个维度来区分优劣。这在技术分析领域也是合理的：阻力位的"厚度"（年龄、测试次数、成交量堆积）直接影响突破后的上涨动能。

**关键洞察**：两种假设并不矛盾。信号系统回答"这只股票是否值得关注"，突破系统回答"这次突破的质量有多高"。前者是筛选问题，后者是排序问题。

### 1.2 信号来源对比

| 维度 | 联合信号系统 | 突破选股系统 |
|------|-------------|-------------|
| **信号源数量** | 4种独立信号 | 1种核心事件 |
| **覆盖的市场行为** | 突破、放量、大阳线、双底 | 仅突破 |
| **信号独立性** | 高（4种检测器独立运行） | N/A（单一事件） |
| **特征维度** | 低（每个信号1-2个属性） | 高（20+特征） |
| **事件检测灵敏度** | B信号低（严格参数）；V/Y高 | 高（宽松参数） |

**分析**：

信号系统的 4 种信号覆盖了不同的市场行为模式：
- **B（突破）**：买方力量突破卖方筹码堆积——价格结构突变
- **V（放量）**：异常资金涌入——流动性突变
- **Y（大阳线）**：单日极端买方主导——情绪突变
- **D（双底）**：底部确认——趋势反转信号

这 4 种信号从价格结构(B)、成交量(V)、K线形态(Y)、趋势模式(D) 四个正交维度捕捉市场异常。理论上，它们的联合出现比任何单一维度都更有说服力。

突破系统虽然只关注一种事件，但对该事件的描述维度极其丰富（成交量激增比、动量PBM、波动率标准化、缺口比例等），本质上是将"突破"这一事件的多个侧面都纳入了考量。

**关键洞察**：信号系统的"宽度"和突破系统的"深度"是两种正交的信息维度。信号系统能捕捉到突破系统完全忽略的V/Y/D事件；突破系统能区分信号系统无法区分的"高质量突破"和"低质量突破"。

### 1.3 评分/排序机制

| 维度 | 联合信号系统 | 突破选股系统 |
|------|-------------|-------------|
| **评分模型** | 加法模型：sum(strength * freshness) | 乘法模型：BASE * bonus1 * bonus2 * ... |
| **加权来源** | 自然计数（pk_num, tr_num），V/Y固定1.0 | 12个手工设定的分段乘数 |
| **鲜度处理** | PRTR价格衰减 + 时间半衰期(30天) | 无显式衰减（全历史扫描） |
| **异常过滤** | turbulent时仅保留D信号 | overshoot惩罚（3sigma->0.7, 4sigma->0.4） |
| **得分范围** | 连续但有界（受信号数和freshness约束） | 理论无上界（乘法累积） |
| **排序策略** | (weighted_sum, signal_count) 双字段降序 | quality_score 单字段降序 |

**分析**：

**加法模型 vs 乘法模型的本质差异**：

加法模型（信号系统）的隐含假设是"边际递减"——第3个信号提供的增量信息与第1个相同。乘法模型（突破系统）的隐含假设是"协同增强"——一个同时具有高成交量、高动量、高阻力位年龄的突破，其质量是各因子的乘积而非加和。

从金融直觉看，乘法模型更符合真实世界：一个突破了252天阻力位(1.5x)、成交量3倍放大(1.3x)、带缺口(1.15x)的突破，确实应该比简单加和这些优势的得分更高。但乘法模型的风险在于：如果某个因子的乘数设定不合理，误差会被放大而非摊薄。

**鲜度机制**：信号系统的鲜度衰减（PRTR + 时间半衰期）是一个重要的实用设计。它解决了"一个月前的突破信号今天是否还有参考价值"的问题。突破系统缺乏这一机制，全历史扫描可能产生大量已经"过期"的突破信号。

### 1.4 参数依赖度

| 维度 | 联合信号系统 | 突破选股系统 |
|------|-------------|-------------|
| **核心检测参数** | ~15个（4个检测器各3-5个） | ~5个（凸点+突破检测） |
| **评分/权重参数** | 0个（自然计数，无手工权重） | ~40个（12个Bonus的阈值+乘数） |
| **总参数量** | ~15个 | ~45个 |
| **参数敏感性** | 中（检测阈值影响信号触发） | 高（乘法模型放大参数偏差） |
| **校准需求** | 低（参数多为物理含义明确的阈值） | 高（Bonus乘数需要回测调优） |

**分析**：

信号系统将参数集中在"检测"环节（什么构成一个有效信号），而"评分"环节几乎无参数（自然计数）。这是一个优雅的设计选择——它将不确定性锁定在了相对可控的环节（检测阈值有明确的物理含义，如"突破需要超过阻力位至少1%"），而在最容易过拟合的环节（评分权重）选择了最简单的方案。

突破系统的参数分布则相反：检测参数较少但评分参数极多。12个Bonus因子，每个有2-4个分段阈值和对应乘数，总共约40个需要设定的数值。这些数值的"最优"组合很难通过理论推导获得，实际上需要依赖回测——而回测优化正是过拟合的温床。

**关键洞察**：信号系统的"参数节俭"策略在样本外表现上可能更稳健。突破系统的丰富参数提供了更高的样本内区分度，但需要严格的交叉验证来防止过拟合。

### 1.5 可解释性对比

| 维度 | 联合信号系统 | 突破选股系统 |
|------|-------------|-------------|
| **单股票解释** | 信号序列标签（如"BVY"） | ScoreBreakdown（每个因子贡献） |
| **解释粒度** | 粗（"有3种信号"） | 细（"age_bonus=1.30因为阻力位存在63天"） |
| **直觉性** | 高（"同时突破+放量+大阳线"易理解） | 中（需理解Bonus乘法逻辑） |
| **可操作性** | 中（知道有什么信号，但不知道信号"多好"） | 高（知道每个因子的具体数值和贡献） |
| **横向比较** | 容易（信号数直接比较） | 较难（分数受多因子非线性影响） |

**分析**：

两种可解释性服务于不同场景。信号系统的序列标签适合快速筛选和概览（"这只股票有3个信号，那只只有1个"）；突破系统的ScoreBreakdown适合深度分析单只股票（"为什么这个突破的分数特别高？因为阻力位年龄252天、成交量3倍放大"）。

### 1.6 过拟合风险评估

| 风险维度 | 联合信号系统 | 突破选股系统 |
|---------|-------------|-------------|
| **模型复杂度** | 低（加法+自然计数） | 高（乘法+40参数） |
| **数据窥探风险** | 低（参数有物理含义） | 中高（Bonus阈值可能基于历史数据调优） |
| **未来信息泄露** | 无 | 有（stability_score使用未来5日数据） |
| **因子相关性** | 低（4种信号基本正交） | 中（volume_surge和gap可能相关） |
| **样本外退化风险** | 低 | 中高 |

**分析**：

信号系统的过拟合风险显著低于突破系统。主要原因：

1. **评分环节无参数**：自然计数不存在"最优权重"的概念，也就不存在对训练数据的过度适应。
2. **信号独立性**：B/V/Y/D从不同维度检测，即使某个信号在特定市场环境下失效，其他信号仍然有效。
3. **无未来信息**：所有计算仅使用当前和历史数据。

突破系统的过拟合风险主要来自：

1. **乘法模型的参数量**：12个因子各有阈值和乘数，组合空间极大。
2. **stability_score使用未来5日数据**：这是回测中的信息泄露，实盘无法使用。
3. **因子间潜在相关性**：volume_surge_ratio和gap_up_pct可能在某些市场环境下高度相关，导致乘法模型重复计算同一信息。

## 二、互补性分析

### 2.1 B信号与突破系统的关系

B信号是突破系统的**严格子集**，而非简单的"简化版"。两者的关系可以精确描述为：

```
B信号 ⊂ 突破系统检测结果（检测层面）
B信号 ≈ 突破系统中"高确信度突破"的集合
```

**参数差异的含义**：

| 参数差异 | 效果 |
|---------|------|
| total_window: 30 vs 10 | B信号要求更宽的凸点，过滤掉短期波动形成的"伪阻力位" |
| min_side_bars: 5 vs 2 | B信号要求凸点两侧有更多确认K线，减少噪音 |
| min_relative_height: 0.2 vs 0.05 | B信号要求阻力位相对更高，过滤掉微小阻力位的突破 |
| exceed_threshold: 0.01 vs 0.005 | B信号要求更大的突破幅度，减少假突破 |
| peak_measure: "high" vs "body_top" | B信号使用最高价作为阻力位，更保守 |
| breakout_modes: "close" vs "body_top" | B信号要求收盘价确认突破，更严格 |
| lookback: 126天 vs 全历史 | B信号只关注半年内的阻力位 |

B信号的每一个参数都比突破系统更严格。这意味着：**凡是被B信号检测到的突破，必然也会被突破系统检测到，且在突破系统中大概率获得中高分数**（因为严格参数已经过滤掉了低质量突破）。

但反过来不成立：突破系统检测到的许多突破不会触发B信号。这些"B信号遗漏"的突破中，有一部分可能是有价值的（如突破了一个仅存在20天但被测试了4次的阻力位），也有一部分是噪音。突破系统的质量评分在此处发挥作用——它能区分这些被B信号过滤掉的突破中哪些真正有价值。

### 2.2 V/Y/D信号的独特价值

这是信号系统相对于突破系统最重要的差异化优势。

**V（超大成交量）信号**：捕捉异常资金流入，可能预示即将到来的突破或其他重大事件。突破系统只在突破已经发生时才检查成交量（作为评分因子），而V信号可以在突破发生之前就发出预警。

**Y（大阳线）信号**：捕捉极端买方主导的单日行情。某些情况下，大阳线不一定伴随历史阻力位的突破（如缺口上涨后在无阻力区间运行），但仍然是强烈的买入信号。突破系统完全忽略这类事件。

**D（双底）信号**：捕捉趋势反转模式。双底确认的时间点通常在股价底部区域，此时可能尚未突破任何有意义的阻力位。突破系统需要等待价格上涨并突破阻力位才能产生信号，而D信号可以更早地识别底部反转机会。

**V/Y/D的联合价值**：在没有B信号的情况下，V+Y的组合（放量大阳线）本身就是一个强信号；D+V的组合（双底后放量）暗示底部反转得到资金确认。这些模式无法被纯突破系统捕捉。

### 2.3 突破系统质量评分的独特价值

突破系统的质量评分提供了信号系统缺乏的**单一信号内部区分度**。

信号系统中，两个B信号的区别仅在于pk_num（突破了几层阻力）。但真实世界中，以下两种突破差异巨大：

- 突破A：突破1个存在252天、被测试4次的阻力位，成交量3倍放大，带缺口
- 突破B：突破1个存在30天、仅被测试1次的阻力位，成交量1.2倍放大，无缺口

在信号系统中，两者的pk_num都是1，strength相同。但突破系统可以给出完全不同的分数（突破A可能是突破B的3-5倍）。

### 2.4 互补性总结

```
信号系统独有价值：
  ├── V/Y/D信号覆盖非突破型市场行为
  ├── 多信号共识增加选股可靠性
  ├── 鲜度衰减提供信号时效性管理
  └── turbulent过滤减少极端行情下的噪音

突破系统独有价值：
  ├── 同类事件（突破）内的精细质量区分
  ├── 阻力位属性量化（年龄、测试次数、高度）
  ├── 突破行为量化（动量PBM、缺口、波动率标准化）
  └── 全历史扫描覆盖更早的阻力位

重叠区域（B信号 ∩ 突破系统）：
  └── B信号 ⊂ 突破系统（B信号是突破系统的严格高置信子集）
```

## 三、不同市场环境下的表现分析

### 3.1 牛市（持续上涨趋势）

| 系统 | 表现预期 | 原因 |
|------|---------|------|
| 信号系统 | 中等 | B/V/Y信号频繁触发但可能噪音增多；D信号稀少 |
| 突破系统 | 良好 | 持续产生突破事件，质量评分可有效区分真突破和趋势内波动 |

牛市中突破频繁，信号系统的"信号计数"可能导致大量股票都有2-3个信号，区分度下降。突破系统的精细评分在此场景下优势明显——它可以区分"突破52周高点"和"突破2周高点"。

### 3.2 熊市（持续下跌趋势）

| 系统 | 表现预期 | 原因 |
|------|---------|------|
| 信号系统 | 较好 | D信号可捕捉底部反转；turbulent过滤减少假信号 |
| 突破系统 | 较差 | 突破事件稀少且多为假突破；缺乏熊市适应机制 |

熊市中D信号成为最有价值的信号源，而突破系统完全缺乏这类能力。信号系统的turbulent过滤虽然是二值性的（存在边界效应），但至少提供了一定的熊市保护。

### 3.3 震荡市（区间波动）

| 系统 | 表现预期 | 原因 |
|------|---------|------|
| 信号系统 | 中等偏弱 | B信号可能在区间上沿频繁触发后回落；V/Y信号噪音增多 |
| 突破系统 | 中等 | 可通过阻力位测试次数和peak_age区分真假突破 |

震荡市是两种方法都面临挑战的环境。突破系统的优势在于，经过多次测试(tests>=3)的阻力位被突破时，其test_bonus和age_bonus会给出更高分数，理论上能更好地区分震荡区间内的假突破和真正的趋势突破。

### 3.4 环境适应性总结

| 市场环境 | 更优方法 | 关键差异因素 |
|---------|---------|-------------|
| 牛市初期 | 信号系统（B+V+Y联合确认） | 多信号共识减少追涨风险 |
| 牛市中后期 | 突破系统（质量评分区分） | 精细评分区分趋势延续vs过热 |
| 熊市 | 信号系统（D信号+turbulent过滤） | 底部反转检测能力 |
| 震荡市 | 两者均有局限 | 均缺乏震荡市专用逻辑 |
| 个股事件驱动 | 信号系统（V/Y信号） | 捕捉非突破型异常行为 |

## 四、建议方案：双层融合架构

### 4.1 方案选择分析

| 方案 | 优势 | 劣势 |
|------|------|------|
| 仅用信号系统 | 简单、抗过拟合、多维度覆盖 | B信号内部区分度不足 |
| 仅用突破系统 | 精细区分突破质量 | 缺失V/Y/D维度、参数过多 |
| **双层融合** | **兼得两者优势** | **实现复杂度增加** |

**推荐：双层融合架构**——以信号系统作为"初筛层"（Stage 1），以突破系统的质量评分作为B信号的"精排增强"（Stage 2），V/Y/D信号保留独立通道不受突破系统影响。

### 4.2 融合方案设计

#### 架构概览

```
Stage 1: 信号初筛（现有联合信号系统，不变）
  ├── B检测器 -> B信号 (strength=pk_num)
  ├── V检测器 -> V信号 (strength=1.0)
  ├── Y检测器 -> Y信号 (strength=1.0)
  └── D检测器 -> D信号 (strength=tr_num)
  |
  composite.py -> weighted_sum 排序 -> 候选池（Top N）

Stage 2: B信号增强（仅对候选池中含B信号的股票）
  ├── 使用突破系统的特征计算 -> 获取突破质量特征
  ├── 计算 quality_score（使用精简版Bonus模型）
  └── 将 quality_score 归一化后注入 B信号的 strength
  |
  重新计算 weighted_sum -> 最终排序
```

#### 关键设计决策

**1. Stage 2 仅增强，不替代**

B信号的检测仍使用信号系统的严格参数（total_window=30等），不改为突破系统的宽松参数。这保证了B信号的高置信度不被稀释。Stage 2 只是为已检测到的B信号增加"质量维度"。

**2. 精简Bonus模型**

从突破系统的12个Bonus因子中，选择信息增量最大且过拟合风险最低的子集：

保留因子：
- age_bonus：阻力位年龄有明确的经济学含义（筹码沉淀时间），过拟合风险低
- test_bonus：测试次数是自然计数，与信号系统哲学一致
- volume_bonus：突破时成交量放大是经典确认信号
- overshoot_penalty：过冲惩罚防止追涨，风控价值高

移除因子及理由：
- height_bonus：与B信号的min_relative_height=0.2已有冗余
- peak_volume_bonus：与volume_bonus信息部分重叠
- idr_vol_bonus：与volume_bonus相关性高
- gap_vol_bonus：与volume_bonus相关性高
- pbm_bonus：计算复杂，物理含义不如其他因子直观
- streak_bonus：连续突破在严格B信号参数下极少发生
- pk_momentum_bonus：与age_bonus部分重叠
- stability_score：**使用未来数据，必须移除**

精简后仅保留 4 个因子（age, test, volume, overshoot），参数从~40个降至~12个，大幅降低过拟合风险。

**3. quality_score 归一化方法**

将 quality_score 转化为 B 信号的增强 strength：

```
enhanced_strength = pk_num * (1 + normalized_quality)
其中 normalized_quality = (quality_score - BASE) / BASE, clamp到[0, 1]
```

这样，质量评分仅作为"增益"叠加在原有pk_num之上，而非完全替代。当quality_score等于BASE(50)时，增益为0，B信号行为与现有系统完全一致。

**4. V/Y/D信号不受影响**

V/Y/D信号保持现有逻辑不变。它们的价值在于覆盖突破之外的市场行为维度，与质量评分正交。

### 4.3 实施优先级

| 优先级 | 步骤 | 复杂度 | 价值 |
|-------|------|--------|------|
| P0 | 移除突破系统中stability_score的未来数据依赖 | 低 | 高 |
| P1 | 实现精简版Bonus模型（4因子） | 中 | 高 |
| P2 | 在composite.py中增加Stage 2增强逻辑 | 中 | 高 |
| P3 | 回测验证融合方案 vs 单一系统的表现差异 | 高 | 高 |
| P4 | 根据回测结果微调精简Bonus的阈值 | 中 | 中 |

### 4.4 融合方案的风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| 增加系统复杂度 | Stage 2 为可选增强，可通过配置开关禁用 |
| 精简Bonus仍可能过拟合 | 仅保留4个因子，参数量可控；交叉验证 |
| Stage 2 增加计算开销 | 仅对候选池（Top N，通常<50只）执行 |
| 两层评分逻辑增加调试难度 | enhanced_strength公式简单透明，可追踪分解 |

## 六、结论

1. **两种方法是互补关系而非替代关系**。信号系统提供广度（4种信号源），突破系统提供深度（20+特征精细评分）。

2. **B信号是突破系统的严格高置信子集**。信号系统通过更严格的参数自然过滤了低质量突破，相当于突破系统+隐式质量过滤器。

3. **V/Y/D信号提供了突破系统完全缺失的维度**。特别是D信号在熊市中的底部反转检测能力，以及V/Y信号对非突破型市场异常的捕捉能力。

4. **突破系统的质量评分提供了信号系统缺乏的同类事件区分度**。但其40参数的乘法模型过拟合风险较高，需要精简。

5. **建议实施双层融合架构**：信号系统初筛 + 精简版质量评分增强B信号。保留信号系统的抗过拟合特性和多维度覆盖，同时引入突破质量的精细区分能力。

6. **最高优先级**：移除突破系统中stability_score的未来数据依赖（信息泄露）。这不论是否采用融合方案都应立即执行。
