# Sentiment Score 与组合模板的集成策略

> 2026-03-27 | Agent Team 研究报告

## 结论

**sentiment_score 应作为独立风控层存在，不注入技术因子组合模板。**

三位分析师（架构、方案对比、金融原理）独立得出一致结论。

---

## 推荐架构

```
扫描突破 → 技术因子评分(组合模板筛选) → [Sentiment 风控门] → 入池
                                              │
                                        score < -0.40 → 强否决(直接排除)
                                        score < -0.15 → 排除(可保留观察)
                                        score ∈ [-0.15, +0.30] → 放行
                                        score > +0.30 → 放行 + 标记(可选排序加分)
```

**关键设计决策**：
- 技术模板负责"选入"（哪些突破值得关注）
- sentiment 负责"风控"（过滤有负面消息的股票）
- 两者职责分离，互不干扰

---

## 为什么不注入模板（作为第 13 个因子）

### 1. 回测致命缺陷

mining 训练集有约 9800 个历史突破事件，需要为每个事件补采历史新闻并运行情感分析。成本极高，且历史新闻的可得性和完整性无法保证。小盘股新闻覆盖不均匀会引入系统性偏差，污染 Optuna 优化结果。

### 2. 粒度不匹配

技术因子是**逐突破**评分（每个 breakout 有独立因子值），sentiment 是**逐股票**评分（整个 ticker 一个 score）。同一只股票的所有突破会共享同一个 sentiment multiplier，概念混乱。

### 3. 时间尺度不兼容

技术因子基于历史形态（天~周级，慢变化），sentiment 基于实时新闻（小时~天级，快衰减 half_life=3 天）。Optuna 优化假设因子值在决策窗口内稳定，sentiment 不满足这个假设。

### 4. 一票否决被稀释

模板评分是乘法模型（values: 1.2 奖励 / 0.8 惩罚），一个因子的 0.8 惩罚可被其他因子的 1.2 奖励抵消。这意味着"技术面好但新闻面差"的股票仍可能通过筛选——恰恰是否决权要防止的场景。

### 5. API 依赖风险

sentiment 依赖外部 API（Finnhub/AlphaVantage + LLM），API 故障不应阻塞核心扫描流程。独立层降级策略简单：跳过 sentiment 检查即可。注入模板则需要设计 fallback 值，增加复杂性。

## 为什么独立风控层是最优解

### 信息异质性

技术因子测量"价格行为中已实现的信息"（过去发生了什么），sentiment 测量"尚未完全定价的信息"（市场叙事和预期）。两者几乎正交，组合有理论增益（IR 改善约 41%），但组合方式应尊重各自的信息本质。

### 风控 vs 因子的本质区别

| 特征 | 因子逻辑 | 风控逻辑 |
|------|---------|---------|
| 作用方式 | 连续评分，渐进调节 | 二值门控，通过/拒绝 |
| 可补偿性 | 某因子弱可被其他因子补偿 | 不可补偿——安全阈值是硬底线 |
| 失败成本 | 遗漏好机会（机会成本） | 买入暴雷股（本金损失） |

sentiment 的非对称阈值设计（-0.15 vs +0.30，"排除坏股比选入好股更重要"）天然是风控语义。

### 实操时序自然匹配

系统工作流是"先突破后查新闻"。技术扫描是 CPU 密集（本地计算），sentiment 是 IO 密集（API 调用 5-15s/股）。"先粗筛后精筛"的漏斗模型是信息检索的最优策略——低成本高召回率的筛选器在前，高成本高精度的验证器在后。

### 业界共识

量化基金主流做法：Alpha 叠加 + 独立风控双层架构。Alpha 信号之间是加法关系（各自贡献增量信息），风控是乘法关系（任何红线都可以否决）。Tetlock (2007) 研究表明负面新闻对股价的预测能力是正面新闻的 2-3 倍，支持将 sentiment 主要用于风控排除。

## 关于正面信号（score > +0.30）

建议作为**可选项（默认关闭）**：

- 正面 sentiment 的选股增量价值尚未经过回测验证
- 学术研究表明正面新闻预测力远弱于负面
- 如果开启，实现为入池后的**排序因子**（同等技术评分时优先正面 sentiment），而非模板内因子
- 未来如果验证了正面 sentiment 有显著增量 alpha，可升级为 alpha 叠加模式

---

## 实现方案

### 复杂度：最低

在模板匹配结果后增加 sentiment 风控过滤器，改动调用端 1-2 处。不改 mining/factor_registry 等核心模块。

### 降级策略

sentiment API 不可用时，自动退化为纯技术模板选股，系统不中断。

### 所有方案对比

| 维度 | 独立风控层 | 注入模板 | 前置过滤 | 混合模式 |
|------|-----------|---------|---------|---------|
| 实现复杂度 | **最低** | 高 | 低 | 中 |
| mining 改动 | **无** | 深度 | 无 | 无 |
| 可回测性 | **不需要** | 极差 | 差 | 部分差 |
| 系统可用性 | **降级优雅** | API 成硬依赖 | API 成硬依赖 | 中 |
| API 开销 | **候选列表** | 全量 | 全量 | 候选列表 |
| 否决语义 | **不可补偿** | 被稀释 | 不可补偿 | 不可补偿 |
| 架构耦合度 | **最低** | 高 | 低 | 中 |
| 正面信号利用 | 弱(排序加分) | 强 | 否 | 部分 |
| 上线风险 | **最低** | 高 | 中 | 中 |

## 对"技术面好但新闻面差"的处理策略

1. **score < -0.40（强否决）**：无条件排除。当公司面临 SEC 调查、破产传闻时，技术突破可能是"逃顶"而非"起飞"
2. **score < -0.15（温和负面）**：排除，但可保留在观察池中等待 sentiment 转好
3. **score ∈ [-0.15, +0.30]（中性）**：放行，完全由技术因子决定。大多数股票落在此区间
4. **score > +0.30（正面）**：放行 + 可选排序加分
