# mining_mode 审核报告：lte 因子的设计意图 vs 数据推断

## 背景

`outputs/statistics/best/all_factor.yaml` 中有 5 个因子被设为 `mode: lte`。其中 `overshoot` 已在注册表中硬编码 `mining_mode='lte'`（已确认合理，详见 overshoot_factor_analysis.md）。本报告审核另外 4 个由 Spearman 自动推断为 lte 的因子。

## 审核方法

对每个因子检查三个维度：
1. **计算逻辑**：features.py 中的实现，值的物理含义
2. **注册表信号**：default_values > 1.0（奖励型/gte 意图）还是 < 1.0（惩罚型/lte 意图）
3. **数据信号**：Spearman 相关系数方向和强度

## 审核结果

### 1. streak（连续突破）— 设计意图与数据一致

| 维度 | 信号 |
|------|------|
| 计算 | 窗口期内突破次数，值越大 = 连续突破越多 |
| default_values | **(0.9, 0.75)** — 惩罚型，连续突破是坏事 |
| Spearman | r = -0.0506 → lte |

**设计意图**：lte。连续突破 = 动量耗尽/震荡突破，非有效趋势启动。

**结论**：设计意图与数据一致。**建议锁定 `mining_mode='lte'`**，防止未来数据漂移翻转方向。

---

### 2. pk_mom（峰值动量/凹陷深度）— 设计意图与数据矛盾

| 维度 | 信号 |
|------|------|
| 计算 | `1 + log(1 + D_atr)`，D_atr = (peak - trough) / ATR。凹陷越深值越大 |
| default_values | **(1.2, 1.5)** — 奖励型，深蹲起跳是好事 |
| Spearman | r = -0.0218 → lte |
| docstring | 明确写"奖励深蹲起跳" |

**设计意图**：gte。深凹陷 = 回调充分消化卖压 + 弹簧效应 = 高质量突破。

**矛盾分析**：
- Spearman r = -0.0218，仅略超 weak_threshold (0.015)，信号极弱
- `has_nan_group=True`：超出时间窗口的突破返回 0.0，这些 0.0 值参与 Spearman 计算可能扭曲方向
- 自动推断将一个设计为**奖励**的因子翻转为**惩罚**

**结论**：**强烈建议锁定 `mining_mode='gte'`**。设计意图明确，Spearman 信号极弱且可能被 NaN 组污染。

---

### 3. age（突破位龄）— 设计意图 gte，数据弱 lte

| 维度 | 信号 |
|------|------|
| 计算 | 被突破的最老峰值距当前的交易日数，值越大 = 阻力位存在越久 |
| default_values | **(1.02, 1.03, 1.05)** — 奖励型，老阻力位突破是好事 |
| Spearman | r = -0.0561 → lte |
| YAML 优化结果 | threshold=24, value=0.8, mode=lte（age ≤ 24 被惩罚） |

**设计意图**：gte。传统技术分析认为越老的阻力位被突破，信号越强。

**微妙之处**：YAML 中 lte 模式 + 低阈值惩罚的效果是"年轻阻力位不好"，这与 gte 意图**方向一致**但实现路径不同。lte 模式下 TPE 搜索的是"哪个上限以下不好"，gte 模式下搜索的是"哪个下限以上好"——会得到不同的最优阈值。

**结论**：**建议不锁定**。r = -0.0561 虽然弱但在 4 个因子中最强，且 default_values 的奖励幅度极小 (1.02~1.05)，说明设计者对 age 的正向效应信心本就不强。让数据驱动。

---

### 4. test（测试次数）— 设计意图 gte，数据弱 lte，优化结果支持 gte

| 维度 | 信号 |
|------|------|
| 计算 | 阻力位最大簇的峰值数，值越大 = 同一位置被测试越多次 |
| default_values | **(1.1, 1.25, 1.4)** — 奖励型，多次测试后突破是好事 |
| Spearman | r = -0.0522 → lte |
| YAML 优化结果 | threshold=2, value=0.8, mode=lte（test ≤ 2 被惩罚） |

**设计意图**：gte。多次测试 = 供给充分消化 = 突破后上涨动力更大。

**关键观察**：YAML 优化结果（test ≤ 2 惩罚）本质上在说"测试次数少是坏事"，这与 gte 设计意图完全一致。Spearman 弱负相关可能因为 test 是离散值（值域 1~N），线性单调假设不适用。

**结论**：**建议锁定 `mining_mode='gte'`**。设计意图明确，YAML 优化结果方向也支持 gte，Spearman 信号弱。

---

## 总结

| 因子 | 设计意图 | default_values | Spearman r | 一致性 | 建议 |
|------|---------|---------------|-----------|--------|------|
| streak | lte | 0.9, 0.75 (惩罚) | -0.0506 | 一致 | **锁定 lte** |
| pk_mom | gte | 1.2, 1.5 (奖励) | -0.0218 | **矛盾** | **锁定 gte** |
| test | gte | 1.1, 1.25, 1.4 (奖励) | -0.0522 | 弱矛盾 | **锁定 gte** |
| age | gte | 1.02, 1.03, 1.05 (奖励) | -0.0561 | 弱矛盾 | 不锁定，让数据驱动 |
| overshoot | lte (风控) | 0.80, 0.60 (惩罚) | +0.0331 | 特殊 | 已锁定 lte（风控意图） |
| peak_vol | gte | — | -0.0062 | 弱/非单调 | **建议锁定 gte** |

### 决策原则

锁定 mining_mode 的条件：
1. 设计意图明确（default_values 方向清晰 + 代码语义明确）
2. Spearman 信号弱（|r| < 0.03）或可能被 NaN 组/离散值域污染
3. 锁定方向后，TPE 会在正确的搜索空间中自行找到最优阈值

不锁定的条件：
1. 设计意图与数据存在较强矛盾（|r| > 0.05）
2. 设计者对效应方向信心不强（default_values 接近 1.0）
