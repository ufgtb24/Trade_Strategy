## 一、旧缺口复核

### 1.1 [P0] Streak bonus → penalty

**旧结论**：Streak 方向完全错误，应从加分（x1.20/x1.40）改为惩罚（x0.85/x0.70）。

**新数据验证**：

| 指标 | 旧报告 (Height 禁用) | 新报告 (Height 启用) | 变化 |
|------|---------------------|---------------------|------|
| Spearman r | -0.1988 | -0.1936 | 微弱缩小，仍为强负相关 |
| RF 重要性 | 0.2178（第 2） | 0.0712（第 3） | 大幅下降 — 被 Height 吸收 |
| level 0 → 2 median | 0.1956 → 0.1120 (-43%) | 0.1956 → 0.1128 (-42%) | 基本一致 |
| PK-Mom + Streak 交互 | -0.0327 | -0.0325 | 一致，仍为最大负交互 |

**复核结论**：**缺口完全成立，证据强度不变**。Spearman r 从 -0.1988 微调至 -0.1936，差异可忽略。RF 重要性大幅下降是因为 Height 进入模型后"抢走"了部分解释力，但这不影响 Streak 方向错误的事实 — level 0/1/2 的 median 递减模式完全不变。

**当前系统状态**：**未修复**。YAML 中 streak_bonus values 仍为 [1.20, 1.40]，正向加分逻辑不变。

**优先级**：维持 **P0**。这仍是整个系统中**唯一一个方向完全反转的因子**，且影响范围巨大：level 1 覆盖 3741 个样本（38.2%），level 2 覆盖 2742 个样本（28.0%），合计 66.2% 的突破事件被错误加分。

### 1.5 [P2] Overshoot penalty 暂时禁用或设为中性

**旧结论**：Overshoot 设为惩罚但统计上正向（r=+0.0794），存在角色矛盾。建议禁用或设为中性。

**新数据验证**：

| 指标 | 旧 | 新 |
|------|-----|-----|
| Spearman r | +0.0794*** | +0.0736*** |
| level 0/1/2 median | 0.1593/0.2771/0.2923 | 0.1596/0.2737/0.2720 |

**深入分析（旧报告未涉及的新角度）**：

Overshoot 的正相关需要更审慎的解读。在交互矩阵中，Overshoot 与几乎所有因子的交互都是强正值：

| 交互对 | 效应 |
|--------|------|
| Age + Overshoot | +0.1820 |
| Streak + Overshoot | +0.1647 |
| PK-Mom + Overshoot | +0.1559 |
| PBM + Overshoot | +0.1535 |
| Tests + Overshoot | +0.1425 |

但 Volume + Overshoot 的交互为 **-0.0105**（负值）。这揭示了一个关键细节：当放量突破同时触发 Overshoot 时，效果反而不如单独出现。可能的解释是：放量 + 超涨 = 一日游行情的特征（大资金在突破日集中出货），而非放量的超涨更可能是供给不足驱动的缺口上涨（更健康）。

此外，Overshoot 的样本量极小：level 1 仅 120（1.2%），level 2 仅 106（1.1%）。当前的惩罚乘数（0.80/0.60）实际影响的样本不到 2.3%，系统级影响有限。

**复核结论**：**缺口仍然成立，但需修正结论**。旧报告建议"禁用或设为中性"过于简单化。新数据显示：
1. Overshoot 的正相关是真实的（r=+0.0736，在 Height 引入后仅微降）
2. Volume + Overshoot 负交互是一个重要信号 — 说明超涨惩罚在放量场景下可能有合理性
3. 样本量太小（226 个），不足以做精细区分

**修正建议**：从"禁用"调整为**设为中性（values=[1.0, 1.0]）并保留数据收集**。不应因样本不足而仓促改为加分项，也不应继续用 0.60x 的重惩罚对待仅 106 个样本的 level 2。

**优先级**：维持 **P2**，但处置方案需更新。

## 二、新增缺口（旧报告未识别的问题）

### 2.1 [新] Height 权重严重低估（GAP-NEW-01）

**问题描述**：Height 是统计上最强的因子（Spearman r=0.3044，RF=0.5810），但其当前乘数配置 [1.1, 1.2, 1.4] 在 11 个因子中排名偏低，远不匹配其统计重要性。

**量化对比**：

| 因子 | Spearman r | RF 重要性 | 最大乘数 | 乘数 / r 比 |
|------|-----------|----------|---------|------------|
| **Height** | 0.3044 | 0.5810 | **1.4x** | **4.60** |
| Volume | 0.1320 | 0.1469 | 2.0x | 15.15 |
| Age | -0.0245 | 0.0119 | 1.5x | -61.22 (反向!) |
| Tests | 0.0008 | 0.0233 | 1.4x | 1750.00 |
| Streak | -0.1936 | 0.0712 | 1.4x | -7.23 (反向!) |
| DayStr | 0.1590 | 0.0515 | 1.2x | 7.55 |
| PBM | 0.0396 | 0.0245 | 1.3x | 32.83 |
| Drought | 0.0918 | 0.0561 | 1.5x | 16.34 |

Height 的"乘数/r 比"仅 4.60，是所有正相关因子中**最低的**。与之对比，同为正向因子的 Volume（最大乘数 2.0x，r=0.1320）获得了更高的绝对乘数，尽管其 r 值不到 Height 的一半。

**level 分布的收益提升与乘数不匹配**：

| Height level | median | 较 level 0 提升 | 当前乘数 | 应有乘数（按比例） |
|-------------|--------|----------------|---------|-------------------|
| 0 | 0.1261 | 基线 | 1.0x | 1.0x |
| 1 | 0.2146 | +70.2% | 1.1x | ~1.30x |
| 2 | 0.2647 | +109.9% | 1.2x | ~1.50x |
| 3 | 0.3394 | +169.2% | 1.4x | ~1.75x |

从 level 0 到 level 1，median 收益提升 70.2%，但乘数仅增加 10%。这个 7:1 的失衡意味着 Height 的信号价值被严重稀释。

**根本原因**：Height 是后来才启用的因子，其乘数可能沿用了保守的初始值，未根据实际统计结果校准。

### 2.2 [新] Height 未参与 Pattern 分类（GAP-NEW-02）

**问题描述**：`_classify_pattern()` 方法使用 5 个因子进行模式判定：PK-Mom、Age、Tests、Volume、Streak。**Height — 系统中预测力最强的因子 — 完全未被纳入。**

当前 Pattern 分类的输入因子与其统计有效性：

| 因子 | 参与 Pattern 分类 | Spearman r | 有效性 |
|------|------------------|-----------|--------|
| PK-Mom | 是 | +0.0962 | 有效 |
| Age | 是（核心角色） | -0.0245 | 无效 |
| Tests | 是 | +0.0008 | 无效 |
| Volume | 是 | +0.1320 | 有效 |
| Streak | 是 | -0.1936 | 反向 |
| **Height** | **否** | **+0.3044** | **最强** |
| Drought | 是（兜底） | +0.0918 | 有效 |
| DayStr | 否 | +0.1590 | 有效 |

这产生了一个结构性问题：Pattern 分类的核心输入是一组统计上无效或反向的因子（Age、Tests、Streak），而最有效的因子（Height、DayStr）不参与分类。这解释了为什么 Pattern 的收益区分度并不理想 — 例如 `momentum`（5714 个样本，58.4%）作为最大类别，其 median（0.1721）仅略高于基线（0.1613）。

**具体影响**：
- `deep_rebound`（pk_mom>=1 AND age>=2）：依赖 Age，而 Age 无预测力。513 个样本的 median 仅 0.1791，不算突出
- `power_historical`（vol>=1 AND age>=2）：同样依赖 Age，但 median 0.2742 较好 — 这主要是 Volume 的功劳，Age 是搭便车
- `grind_through`（age>=2 AND tests>=1）：同时依赖两个无效因子，median 仅 0.1340，表现差
- `historical`（age>=2）：纯 Age 驱动，median 0.1478，低于基线

如果将 Height 纳入分类，可以构建真正有区分力的模式。例如：
- `height_surge`：Height level >= 2，直接对应 median >= 0.2647 的高收益群体
- `height_volume`：Height level >= 1 AND Volume level >= 1，应对应交互效应 +0.1647 的组合增益

### 2.3 [新] 代码与 YAML 的 Height 默认值不同步（GAP-NEW-03）

**问题描述**：Height bonus 在三个位置有不同的默认值：

| 位置 | 文件 | thresholds 默认值 | values 默认值 |
|------|------|-------------------|--------------|
| 核心评分器 | `breakout_scorer.py:135` | `[0.10, 0.20]` | `[1.15, 1.30]` |
| 参数加载器 | `param_loader.py:313` | `[0.10, 0.20]` | `[1.15, 1.30]` |
| YAML 配置 | `scan_params.yaml:52-59` | `[0.2, 0.4, 0.7]` | `[1.1, 1.2, 1.4]` |

代码硬编码的默认阈值 `[0.10, 0.20]` 仅支持 2 个 level（10%、20%），而 YAML 配置了 3 个 level（20%、40%、70%）且阈值完全不同。统计分析使用的是 YAML 的值（因为分析脚本从 scan_metadata 读取），所以当前统计结论是基于 `[0.2, 0.4, 0.7]` 阈值的。

**风险**：如果 YAML 文件缺失或损坏，系统会回退到代码默认值 `[0.10, 0.20]`，此时 Height 的行为完全不同：
- 原本 level 0 的 5429 个样本中，大量（height 在 0.10~0.20 之间的）会变成 level 1
- 原本的 3 级阈值变成 2 级
- level 分布和统计结论将不再适用

**影响范围**：目前实际运行以 YAML 为准，不是紧急问题，但属于代码卫生（code hygiene）缺陷。

### 2.4 [新] 乘法模型整体权重分配失衡（GAP-NEW-04）

**问题描述**：将所有因子的最大可能乘数与其统计贡献对比，可以量化系统的整体失衡程度。

**极端情景分析**：假设一个突破事件所有因子均触发最高 level：

```
总乘数 = age(1.5) x tests(1.4) x height(1.4) x peak_vol(1.2) x volume(2.0)
        x daystr(1.2) x pbm(1.3) x streak(1.4) x drought(1.5) x pk_mom(1.5)
        x overshoot(0.6)
      = 1.5 x 1.4 x 1.4 x 1.2 x 2.0 x 1.2 x 1.3 x 1.4 x 1.5 x 1.5 x 0.6
      = 约 17.1x
```

分解这 17.1x 总乘数中各因子的贡献权重（以 ln(乘数) 计算，因为乘法模型中对数转化为加法权重）：

| 因子 | 最大乘数 | ln(乘数) | 权重占比 | Spearman r | r 占正向总 r | 权重 vs r 偏差 |
|------|---------|---------|---------|-----------|-------------|--------------|
| Volume | 2.0 | 0.693 | 21.7% | +0.1320 | 12.4% | +9.3pp 过高 |
| **Age** | **1.5** | **0.405** | **12.7%** | **-0.0245** | **0%** (负) | **+12.7pp 严重过高** |
| **Drought** | **1.5** | **0.405** | **12.7%** | +0.0918 | 8.6% | +4.1pp 偏高 |
| **PK-Mom** | **1.5** | **0.405** | **12.7%** | +0.0962 | 9.0% | +3.7pp 偏高 |
| **Height** | **1.4** | **0.336** | **10.5%** | **+0.3044** | **28.5%** | **-18.0pp 严重偏低** |
| **Tests** | **1.4** | **0.336** | **10.5%** | +0.0008 | 0.1% | **+10.4pp 严重过高** |
| **Streak** | **1.4** | **0.336** | **10.5%** | **-0.1936** | **0%** (负) | **+10.5pp 严重过高+反向** |
| PBM | 1.3 | 0.262 | 8.2% | +0.0396 | 3.7% | +4.5pp 偏高 |
| DayStr | 1.2 | 0.182 | 5.7% | +0.1590 | 14.9% | -9.2pp 偏低 |
| PeakVol | 1.2 | 0.182 | 5.7% | +0.0239 | 2.2% | +3.5pp 偏高 |
| Overshoot | 0.6 | -0.511 | (惩罚) | +0.0736 | 6.9% | 方向矛盾 |

**核心发现**：
- **Height 的权重仅占 10.5%，但其统计贡献占正向因子总 r 的 28.5%** — 被低估了近 3 倍
- **Age（12.7%）+ Tests（10.5%）+ Streak（10.5%）合计占 33.7% 的权重，但它们的 r 值总和为负数**（-0.0245 + 0.0008 + (-0.1936) = -0.2173）— 超过三分之一的模型权重被分配给了零贡献或负贡献因子
- **DayStr 被低估**：仅 5.7% 权重，但 14.9% 的统计贡献

**具体荒谬场景**：

场景 A：Height level 3 + Age level 3 同时触发
- 当前：height(1.4) x age(1.5) = **2.1x**
- 实际：Height level 3 median = 0.3394（极好），Age level 3 median = 0.1614（基线水平）
- 结论：Age 的 1.5x 乘数"白吃"了 Height 的成果

场景 B：Streak level 2 + Height level 3 同时触发
- 当前：streak(1.4) x height(1.4) = **1.96x**
- 实际：Streak level 2 median = 0.1128（差），Height level 3 median = 0.3394（好）
- 结论：Height 的正面信号被 Streak 的错误加分进一步放大，应该是 height(1.4) x streak(0.70) = **0.98x** 才合理

### 2.5 [新] PK-Mom 的 level 1 异常（GAP-NEW-05）

**问题描述**：PK-Mom level 分布中，level 1 仅有 **1 个样本**（median 0.0345）。

| level | count | median |
|-------|-------|--------|
| 0 | 3564 | 0.1433 |
| 1 | **1** | 0.0345 |
| 2 | 6226 | 0.1731 |

这暗示 PK-Mom 的阈值设置存在问题。YAML 配置为 `thresholds: [1.2, 1.5]`，而 PK-Mom 的定义是 `1 + log(1 + D_atr)`。在数学上：
- PK-Mom = 0：无近期 peak（归入 level 0）
- PK-Mom 在 (0, 1.2)：有近期 peak 但凹陷小（归入 level 0）
- PK-Mom 在 [1.2, 1.5)：这个区间**仅 1 个样本**（level 1）
- PK-Mom >= 1.5：占大多数的 6226 个样本（level 2）

这意味着 **level 1 的阈值区间 [1.2, 1.5) 几乎是空集**。PK-Mom 的值分布可能是双峰的：要么接近 0（无近期 peak），要么直接跳到 >= 1.5（只要有凹陷，D_atr >= 0.65 ATR 就够了）。

**影响**：PK-Mom 事实上是一个二值因子（0 或 2），level 1 的乘数 1.2x 形同虚设。这也影响了 Pattern 分类 — `_classify_pattern` 使用 `pk_mom_level >= 1`，但实际上几乎等价于 `pk_mom_level >= 2`。不过，由于判定逻辑是 `>=`，level 2 同样满足 `>= 1`，所以功能上不受影响，仅存在语义模糊。

### 2.6 [新] Overshoot 惩罚方向与数据矛盾的深层分析（GAP-NEW-06）

**旧报告**建议"禁用或设为中性"，这过于粗略。新数据提供了更丰富的视角。

**新证据**：

Overshoot level 1/2 的 median 分别为 0.2737 和 0.2720，远高于 level 0 的 0.1596。但注意：

1. **当前惩罚实际在"反向筛选"优质样本**：被 Overshoot 惩罚的突破事件，其未惩罚前的原始分数可能本来就很高（因为它们触发了其他正面因子）。Overshoot 的 0.80x/0.60x 惩罚在打压这些高分突破的排名。

2. **交互矩阵中的关键线索**：Overshoot 与 Volume 的交互为 **-0.0105**（负），但与其他所有因子的交互均为正（+0.11 ~ +0.18）。这意味着：
   - 非放量场景下的超涨 → 后续收益好（正交互，可能是供给不足驱动）
   - 放量场景下的超涨 → 后续收益差（负交互，可能是大资金出货）

3. **label_10_40 的测量窗口**：label 是第 10~40 天的最大收益，而 Overshoot 基于前 5 天的涨幅。如果前 5 天已涨了很多，第 10~40 天的最大收益可能**仍然在前 5 天建立的高位附近**，而不是新增涨幅。这造成统计上的正相关假象。

**结论**：Overshoot 的处置需要更细粒度的分析（区分放量/非放量场景），当前最安全的做法是设为中性而非反转为加分。

## 三、权重失衡量化分析

### 3.1 理想权重 vs 当前权重

采用 Spearman |r| 作为"理想权重"的代理指标（因为 |r| 直接衡量因子对收益的单调关联强度），计算各因子的理想 ln(乘数) 权重，然后反推理想乘数：

**正向因子理想权重分配**（排除 Streak 和 Overshoot 这两个特殊因子）：

| 因子 | |r| | 理想权重 | 建议最大乘数 | 当前最大乘数 | 偏差 |
|------|-----|---------|-----------|-----------|------|
| Height | 0.3044 | 38.3% | ~1.70x | 1.4x | -0.30 |
| DayStr | 0.1590 | 20.0% | ~1.40x | 1.2x | -0.20 |
| Volume | 0.1320 | 16.6% | ~1.35x | 2.0x | +0.65 |
| PK-Mom | 0.0962 | 12.1% | ~1.25x | 1.5x | +0.25 |
| Drought | 0.0918 | 11.6% | ~1.24x | 1.5x | +0.26 |
| PBM | 0.0396 | 5.0% | ~1.10x | 1.3x | +0.20 |
| PeakVol | 0.0239 | 3.0% | ~1.06x | 1.2x | +0.14 |
| Tests | 0.0008 | 0.1% | ~1.00x | 1.4x | +0.40 |
| Age | -0.0245 | 负 | ~1.00x 或惩罚 | 1.5x | **+0.50 (方向错)** |

注意：Volume 的当前 2.0x 最大乘数看起来偏高，但考虑到 Volume 在交互矩阵中几乎与所有因子正交互（top 5 正交互中占 4 席），且 Volume level >= 1 的样本仅 7.4%（稀缺信号），其高乘数有一定合理性。但即便如此，Volume 2.0x vs Height 1.4x 的配比仍然不合理 — Height 的 r 值是 Volume 的 2.3 倍。

### 3.2 最严重的失衡点

**失衡 1：Height vs Age — "功劳与奖赏的倒挂"**

```
Height: r = +0.3044, 最大乘数 1.4x → 有功劳、少奖赏
Age:    r = -0.0245, 最大乘数 1.5x → 无功劳、多奖赏
```

Height level 3 (median 0.3394) + Age level 3 (median 0.1614) 同时触发时：
- 当前总乘数：1.4 x 1.5 = 2.1x
- 合理总乘数：~1.7 x ~1.0 = 1.7x
- **Age "白吃" 了 0.4x 的乘数，约占总乘数的 19%**

**失衡 2：Streak — "给敌人发武器"**

```
Streak: r = -0.1936, 最大乘数 1.4x → 负相关、正奖赏
```

Streak level 2 的 median（0.1128）比全样本基线（0.1613）低 30%。系统却给这些低质量突破额外 40% 的加分。在 9791 个样本中，2742 个（28.0%）的 Streak level 为 2，它们每一个都被错误地加了 1.4x 乘数。

**量化影响**：假设修正 Streak 为 penalty（level 2 → 0.70x），仅这一项的修正就会使 2742 个样本的分数降低 50%（从 1.4x 变为 0.7x），相对排名将发生剧烈变化。

**失衡 3：Height vs DayStr — "沉默的劳模"**

```
Height: r = +0.3044, RF = 0.5810, 最大乘数 1.4x
DayStr: r = +0.1590, RF = 0.0515, 最大乘数 1.2x
```

DayStr 的 Spearman r 排名第二，但其最大乘数仅 1.2x，在所有正向因子中倒数第二（仅高于禁用前的 PeakVol 1.2x）。DayStr level 2 的 median（0.2282）较 level 0（0.1493）提升了 52.8%，但乘数仅给了 20% 的奖励。

## 四、Pattern 分类系统缺口

### 4.1 核心结构问题

当前 Pattern 分类使用的因子、统计有效性与覆盖的样本量：

| 模式 | 判定条件 | 所用因子有效性 | 样本量 | median | 评价 |
|------|---------|--------------|--------|--------|------|
| deep_rebound | pk_mom>=1 AND age>=2 | PK-Mom 有效, **Age 无效** | 513 | 0.1791 | 中等 |
| power_historical | vol>=1 AND age>=2 | Volume 有效, **Age 无效** | 164 | 0.2742 | 好（Volume 的功劳） |
| grind_through | age>=2 AND tests>=1 | **两者均无效** | 445 | 0.1340 | 差 |
| momentum | pk_mom>=1 | PK-Mom 有效 | 5714 | 0.1721 | 中等，太大类 |
| historical | age>=2 | **Age 无效** | 943 | 0.1478 | 差 |
| volume_surge | vol>=1 | Volume 有效 | 81 | 0.2755 | 好，但样本太少 |
| dense_test | tests>=2 | **Tests 无效** | 8 | 0.2074 | 样本不足 |
| trend_continuation | streak>=1 | **Streak 反向** | 1580 | 0.1283 | 差（符合预期） |
| dormant_breakout | drought>=1 | Drought 有效 | 88 | 0.1734 | 中等 |
| basic | 兜底 | - | 255 | 0.1371 | 差 |

**关键问题**：

1. **最强因子缺席**：Height（r=0.3044）和 DayStr（r=0.1590）均不参与分类。如果一个 Height level 3、DayStr level 2 的突破事件没有触发 PK-Mom 或 Age，它会被归入 `basic` 或 `trend_continuation` — 完全浪费了最具预测力的信息。

2. **58.4% 的样本被归入单一大类**：`momentum`（pk_mom>=1）覆盖了 5714 个样本，其内部收益分布极为分散（std=0.3545）。这个分类缺乏区分度 — Height level 3 的 momentum 和 Height level 0 的 momentum 被混为一谈。

3. **负面因子产生正面标签**：`trend_continuation`（Streak>=1）在语义上暗示"趋势在延续，是好事"，但实际上这是最差的模式之一（median 0.1283），因为 Streak 是负向因子。标签的语义与收益的实际方向矛盾，会误导用户。

4. **`grind_through` 是一个伪模式**：其判定条件为 age>=2 AND tests>=1，两个均为无效因子。median 仅 0.1340，低于基线。这个模式的存在没有统计支撑。

### 4.2 Pattern 重构方向

基于统计数据，Pattern 分类应围绕真正有效的因子重新设计：

**有效因子优先级**（按 Spearman r 排序）：
1. Height（+0.3044） — 必须纳入
2. DayStr（+0.1590） — 应该纳入
3. Volume（+0.1320） — 已纳入
4. PK-Mom（+0.0962） — 已纳入
5. Drought（+0.0918） — 已纳入

**建议的分类逻辑框架**（仅为方向性建议，具体阈值需基于实验确定）：

```
# 第 1 层：Height 主导的高质量模式
if height_level >= 2 and vol_level >= 1:
    return "power_breakout"       # 高位放量突破（最强组合之一）
if height_level >= 2 and drought_level >= 1:
    return "dormant_power"        # 高位久旱逢甘霖
if height_level >= 2:
    return "high_resistance"      # 高位阻力突破

# 第 2 层：动量确认模式
if vol_level >= 1 and daystr_level >= 1:
    return "volume_surge"         # 放量爆发
if pk_mom_level >= 1 and height_level >= 1:
    return "momentum_height"      # 势能 + 高度
if pk_mom_level >= 1:
    return "momentum"             # 纯势能

# 第 3 层：时间维度
if drought_level >= 1:
    return "dormant_breakout"     # 沉寂突破

# 第 4 层：兜底
return "basic"
```

**关于 Age 的处置**：旧报告正确指出 Age 支撑着 `power_historical` 和 `deep_rebound` 等模式，不能简单移除。但在新框架中，Height 可以自然替代 Age 的角色：
- `power_historical`（vol>=1 AND age>=2）→ 改为 `power_breakout`（vol>=1 AND height>=2）
- `deep_rebound`（pk_mom>=1 AND age>=2）→ 改为 `momentum_height`（pk_mom>=1 AND height>=1）

这样 Age 就可以安全地从分类逻辑中移除，且分类效果会更好。

## 六、核心结论

### 结论 1：Height 的加入从根本上改变了系统的信息结构

Height 以 Spearman r=0.3044（第一）和 RF 重要性 0.5810（占据超过一半的模型解释力）确立了绝对主导地位。它不仅是最强的单因子，还通过其存在重新定义了其他因子的角色：Age 和 PeakVol 在 Height 面前几乎为零增量信息。**但当前系统完全未对此做出响应** — Height 的乘数（最大 1.4x）仅排中游，且不参与 Pattern 分类。这是从旧版（Height 禁用）到新版（Height 启用）过渡中最大的未完成工作。

### 结论 2：旧缺口报告的所有结论在新数据中完全成立

6 条旧缺口中，0 条被推翻、6 条被确认。Streak 的负相关（从 r=-0.1988 微调至 r=-0.1936）、Drought 的非线性（level 1 仍是最佳）、Age/Tests 的无效性（r 进一步趋近于零）、PeakVol 的低价值（RF 降至 0.0097），全部高度重现。**这说明这些是系统的结构性问题，而非数据噪声，需要尽快修复。**

### 结论 3：当前系统有超过三分之一的权重分配给了零贡献或负贡献因子

Age（12.7%）+ Tests（10.5%）+ Streak（10.5%）合计占乘法模型 ln 权重的 33.7%，但它们的 Spearman r 总和为 -0.2173。这意味着系统的评分排名中，有三分之一的"评价维度"不仅不帮助筛选优质突破，反而在引入噪声甚至反向信号。这是整个评分系统最大的效率损失来源。

### 结论 4：Pattern 分类系统需要围绕 Height 重构

当前 Pattern 分类的核心输入是 Age（无效）和 PK-Mom（有效但覆盖太广），而最强因子 Height 完全缺席。这导致 58.4% 的样本被归入缺乏区分度的 `momentum` 大类，而 `grind_through`（两个无效因子的交集）和 `trend_continuation`（负向因子驱动）则产生误导性的模式标签。Height 纳入 Pattern 分类后，可以自然替代 Age 的角色，解除"Age 不能移除因为 Pattern 依赖它"的限制。

### 结论 5：Phase 1 的纯 YAML 修改可以解决约 80% 的权重失衡问题

不需要任何代码变更，仅通过调整 `scan_params.yaml` 中 7 个因子的 values/enabled 配置，就可以：
- 修正 Streak 的方向错误（影响 66.2% 样本）
- 将 Height 的权重提升到匹配其统计贡献
- 消除 Age/Tests/PeakVol 的无效权重（占总权重 28.9%）
- 修正 Drought 的非线性失真
- 提升 DayStr 的权重

这是风险最低、回报最高的改进路径。
