# 突破行为因子审核报告

## Executive Summary

本报告对突破检测系统中 7 个突破行为因子（volume, overshoot, day_str, pbm, streak, drought, pk_mom）的逻辑合理性、参数设置、金融学依据、因子间关系及潜在问题进行了深度审核。

**核心发现**：

1. **volume 因子**：阈值偏高（5x/10x），可能遗漏大量有意义的放量突破，建议下调至 (2.5, 5.0)。
2. **day_str 与 overshoot 的关系**：两者并不矛盾，分别捕获"突破日瞬时爆发力"和"多日累积涨幅风险"，设计合理。但存在 day_str 极端值推高 overshoot 的间接联动，需关注边界。
3. **pk_mom 因子**：注册表中 `mining_mode=None`（Spearman 自动推断），但 values=(1.2, 1.5) 搭配 `>=` 评分语义为"凹陷越深奖励越大"，设计意图与代码实现一致，无语义矛盾。
4. **drought 奖励值递减**：逻辑合理，但收益递减的斜率过陡（1.25 → 1.05），可简化为两级。
5. **pbm lookback=5 天过短**：可能捕获噪音而非趋势性动量，建议考虑 10-15 天。

## 1. Volume（突破量能）

### 1.1 计算逻辑审核

```
volume_ratio = 突破日成交量 / mean(过去63日成交量)
```

**逻辑正确性**：计算逻辑清晰、标准。63 个交易日（约 3 个月）作为均量基准窗口是行业惯例，能平滑短期量能波动，同时不至于包含过远的历史导致均值失真。

**代码细节**：`_calculate_volume_ratio()` 中 `window_start = max(0, index - VOLUME_LOOKBACK)`，当历史数据不足 63 天时会使用较短窗口，这可能导致 IPO 后不久或数据截断的股票产生偏高的基准均量（如果早期有异常放量），从而压低 volume_ratio。不过这属于边界情况，影响有限。

### 1.2 参数审核

| 参数 | 当前值 | 评估 |
|------|--------|------|
| 阈值 | (5.0, 10.0) | **偏高** |
| 奖励值 | (1.5, 2.0) | 合理 |

**分析**：5 倍放量作为第一级门槛过高。从实际市场经验来看：
- **2-3 倍放量**已经是有意义的量能确认，许多成功的突破发生在 2-4 倍放量区间
- **5 倍放量**属于显著放量事件，通常伴随重大消息面驱动
- **10 倍放量**属于极端放量，在日常突破中非常罕见

当前设置的问题是：**大量 2-4 倍的有效放量突破不会获得任何奖励**，而这些突破中有相当比例后续表现良好。5x 门槛可能导致该因子的实际触发率过低，区分力不足。

### 1.3 金融学依据

成交量是价格突破最重要的确认指标之一（Richard Wyckoff 理论、Volume Spread Analysis）。放量突破意味着：
- 大量买方愿意在突破价位以上成交，反映真实需求
- 突破价位附近的卖方供应被充分吸收
- 机构参与的可能性更高（机构交易通常贡献更大成交量）

该因子的方向和原理完全正确。

### 1.4 改进建议

- **建议将阈值调整为 (2.5, 5.0)**：第一级捕获"明确放量"，第二级捕获"显著放量"
- **奖励值可适度调整为 (1.2, 1.5)**：降低第一级奖励避免对常见放量事件过度奖励
- 或者保持当前奖励值但下调阈值，取决于数据挖掘结果中该因子的实际分布

## 2. Overshoot（超涨比）

### 2.1 计算逻辑审核

```
five_day_vol = annual_volatility / sqrt(50.4)   # 5日期望波动率（252/5=50.4）
overshoot = gain_5d / five_day_vol               # σ 单位
```

**逻辑正确性**：将 5 日涨幅标准化为波动率的倍数，使得不同波动率特征的股票可以在同一尺度上比较"涨幅是否过度"。sqrt(50.4) 的换算因子正确（年化波动率按 sqrt(T) 缩放到 T 天波动率）。

**设计意图**：作为惩罚因子（values < 1.0），overshoot 用于识别短期涨幅过大的风险信号——这类突破更可能是短期超买后面临回调压力。

### 2.2 参数审核

| 参数 | 当前值 | 评估 |
|------|--------|------|
| gain_window | 5 天 | 合理 |
| 阈值 | (4.0, 5.0) σ | 合理 |
| 惩罚值 | (0.80, 0.60) | 合理 |
| mining_mode | 'lte' | 正确 |

**分析**：
- **4σ 和 5σ** 作为惩罚阈值是合理的。正态分布下 4σ 事件的单尾概率约 0.003%，但股票收益率具有厚尾特征，实际触发率会高于此。在厚尾分布下，4σ 事件并不罕见，大约 2-5% 的突破可能触发。
- **gain_window=5 天**合理，既不太短（避免单日噪音），也不太长（保持对短期超涨的敏感性）。
- **惩罚力度 0.80/0.60** 恰当。0.60 的惩罚（40% 降分）对 5σ 超涨事件来说足够严厉，反映了极端涨幅后回调的高概率。

### 2.3 mining_mode='lte' 的含义

`mining_mode='lte'` 仅影响挖掘管道：在搜索最优阈值时，挖掘器会选择 `overshoot <= threshold` 的子集（即过滤掉超涨过大的样本）。评分系统始终使用 `>=` 比较：当 `overshoot >= 4.0` 时触发惩罚（multiplier=0.80 < 1.0）。

**两者语义一致**：挖掘管道找"低 overshoot 的子集更好"，评分系统"高 overshoot 时惩罚"——都是在表达"超涨越大，越不好"。

### 2.4 金融学依据

- **均值回复效应**：短期涨幅过大的股票倾向于回调（De Bondt & Thaler 1985，短期反转效应）
- **RSI 超买**：5 天内 4-5σ 的涨幅必然对应极端 RSI 读数，技术分析中这是超买警告
- **做空盘对手方**：大涨后空头套利者介入的概率增加，形成下行压力
- **获利了结**：前期低位买入的持仓者面临巨大浮盈，抛售冲动增强

**结论**：overshoot 因子设计合理，金融学依据充分。

## 3. Day_Str（突破日强度）

### 3.1 计算逻辑审核

```
daily_vol = annual_volatility / sqrt(252)
idr_ratio = intraday_change_pct / daily_vol   # 日内涨幅 / 日波动率
gap_ratio = gap_up_pct / daily_vol             # 跳空幅度 / 日波动率
day_str = max(idr_ratio, gap_ratio)            # 取较大者
```

**逻辑正确性**：取日内涨幅和跳空幅度中较大者的 σ 倍数，能捕获两种不同的突破形态：
1. **日内强势突破**（开盘在阻力位下方，日内拉升突破）
2. **跳空突破**（直接跳空越过阻力位）

**代码细节注意**：
- `intraday_change_pct = (close - open) / open` —— 仅当 close > open 时为正
- `gap_up_pct = (open - prev_close) / prev_close` —— 仅当 open > prev_close 时为正
- 两者都使用 `> 0` 的条件判断，负值会被置为 0，这意味着 day_str **只关注上涨方向**，这对于突破检测是正确的

### 3.2 参数审核

| 参数 | 当前值 | 评估 |
|------|--------|------|
| 阈值 | (1.5, 2.5) σ | 合理 |
| 奖励值 | (1.2, 1.35) | 合理 |

**分析**：
- **1.5σ 日波动**：意味着突破日的价格变动是平均日波动的 1.5 倍。对于日波动率 2% 的股票，这对应约 3% 的日涨幅，属于"明显强势日"。合理。
- **2.5σ**：对应约 5% 的日涨幅（2% 日波动率股票），属于"极端强势日"。合理。
- **奖励值 1.2/1.35**：适中，不过度放大单日表现的影响。

### 3.3 金融学依据

突破日的价格行为是突破有效性的最直接信号：
- **强势K线**（大阳线、跳空）表明买方在突破价位有坚定信念
- **Pivot Point 突破理论**：William O'Neil 的 CAN SLIM 方法论强调突破时的"Power Move"——突破日涨幅越大，后续成功概率越高
- **市场微观结构**：强势突破日通常伴随限价卖单被快速扫掉、做空者被迫回补等连锁反应，这些都是买方控制盘面的信号

**结论**：day_str 设计正确，参数合理。

## 4. PBM（突破前动量）

### 4.1 计算逻辑审核

```
# Step 1: 计算原始动量
displacement = closes[-1] - closes[0]           # 净位移
path_length = Σ|closes[i] - closes[i-1]|        # 路径长度
efficiency = |displacement| / path_length        # 路径效率 [0,1]
raw_momentum = (displacement / closes[0]) * efficiency / n_bars

# Step 2: 波动率标准化
daily_vol = annual_volatility / sqrt(252)
pbm = raw_momentum * sqrt(n_bars) / daily_vol
```

**逻辑正确性**：这个因子设计精巧，融合了三个维度：
1. **收益率**（displacement / closes[0]）：净涨幅
2. **路径效率**（|displacement| / path_length）：涨势的"直线度"，0 表示完全震荡无方向，1 表示单向运动
3. **时间标准化**（/ n_bars × sqrt(n_bars)）：最终 = displacement / (closes[0] × path_length × sqrt(n_bars)) × |displacement| / daily_vol

标准化后的 PBM 值表示"经效率调整的动量相对于随机游走期望值的偏离程度"。

**数学推导验证**：
```
pbm = raw_momentum * sqrt(n) / daily_vol
    = [(d/P₀) * (|d|/L) / n] * sqrt(n) / σ_d
    = d * |d| / (P₀ * L * sqrt(n) * σ_d)
```

当路径是完全单向（efficiency=1, L=|d|）时：
```
pbm = d / (P₀ * sqrt(n) * σ_d)
```
这就是标准的 t-统计量形式（收益率 / 标准误），统计意义清晰。

当路径包含震荡（efficiency < 1）时，PBM 被进一步压缩，惩罚"虽然净涨但过程颠簸"的情况。这是合理的——颠簸上涨说明买卖双方分歧大，突破的可持续性存疑。

### 4.2 参数审核

| 参数 | 当前值 | 评估 |
|------|--------|------|
| lookback | 5 天 | **偏短** |
| 阈值 | (0.7, 1.45) σN | 合理 |
| 奖励值 | (1.15, 1.3) | 合理 |

**关于 lookback=5 天的问题**：

5 天回看窗口仅覆盖突破前一周的价格行为，这个窗口的优缺点：

**优点**：
- 聚焦最近期的动量，与即将发生的突破最相关
- 减少远期噪音的干扰

**缺点**：
- **样本量太小**：5 个数据点计算路径效率，统计稳健性差。一天的异常波动可以极大影响结果
- **容易与 day_str 重叠**：如果突破前 4 天是窄幅横盘，第 5 天（突破前一天）大涨，PBM 会很高，但这更像是 day_str 捕获的信号而非"突破前动量"
- **无法捕获中期趋势**：经典的技术分析中，突破前的"蓄力上涨"通常是 2-4 周（10-20 天）的过程

### 4.3 金融学依据

- **趋势延续性**：突破前有持续上升动量的股票，突破后延续趋势的概率更高（动量效应，Jegadeesh & Titman 1993）
- **买方力量积累**：PBM > 0 表明突破前已有买方在逐步建仓推高价格，突破是自然延伸而非孤立事件
- **路径效率的独特价值**：纯动量因子（如 N 日收益率）无法区分"平稳上涨"和"大幅震荡后的净涨"，路径效率填补了这个空白

### 4.4 改进建议

- **考虑将 lookback 扩展到 10-15 天**：覆盖 2-3 周的动量积累，统计稳健性更好
- 或者**提供两个 lookback 选择**（如 5 天和 15 天），作为不同的因子或 sub_params 选项
- 如果保持 5 天，需要意识到与 day_str 在短窗口下的相关性可能较高

## 5. Streak（连续突破）

### 5.1 计算逻辑审核

```
streak = detector.get_recent_breakout_count(idx)  # 近 window 天内的突破次数
```

计算逻辑依赖 detector 的历史突破记录。window=20 天意味着统计过去 4 周内该股票发生了多少次突破。

### 5.2 参数审核

| 参数 | 当前值 | 评估 |
|------|--------|------|
| window | 20 天 | 合理 |
| 阈值 | (2, 4) 次 | 合理 |
| 惩罚值 | (0.9, 0.75) | 合理 |

**分析**：
- **2 次/20 天**：4 周内第二次突破 → 轻微惩罚（0.9），合理。说明该股票在阻力位附近反复来回，突破的"稀缺性"和信号强度下降。
- **4 次/20 天**：几乎每周一次突破 → 严重惩罚（0.75），合理。频繁突破往往意味着：
  - 峰值检测阈值过低，产生大量伪峰值
  - 股票处于高波动无方向的震荡区间
  - 每次"突破"都缺乏后续跟进

### 5.3 金融学依据

- **信号稀缺性原则**：真正有意义的突破应该是相对稀少的事件。过于频繁的突破信号更可能是噪音
- **支撑/阻力的消耗**：每次突破后回落都会"消耗"该价位的阻力含义，后续突破的技术意义递减
- **Whipsaw 风险**：频繁突破-回落模式是经典的"假突破"信号环境

### 5.4 潜在问题

- **与 drought 的互斥关系**：streak >= 2 意味着 drought 不可能太大（最近 20 天内有前次突破，drought 最多 20 天），两个因子在逻辑上有互斥区域。这不是设计缺陷，但说明它们各自覆盖不同的区域：streak 关注"过密"，drought 关注"稀疏"，中间状态（适度间隔的非频繁突破）两者都不触发。

## 6. Drought（突破干旱期）

### 6.1 计算逻辑审核

```
drought = detector.get_days_since_last_breakout(idx)  # 距上次突破的交易日数
# 首次突破返回 None
```

逻辑简单清晰。`nullable=True` 处理首次突破的情况（无前次突破记录），评分时 None 被视为不触发任何奖惩。

### 6.2 参数审核

| 参数 | 当前值 | 评估 |
|------|--------|------|
| 阈值 | (60, 80, 120) 天 | **三级过密** |
| 奖励值 | (1.25, 1.1, 1.05) | **递减模式有问题** |

**奖励值递减问题的深入分析**：

评分系统使用 `>=` 阶梯判断：
```
drought >= 60  → multiplier = 1.25
drought >= 80  → multiplier = 1.10 （覆盖上一级）
drought >= 120 → multiplier = 1.05 （覆盖上一级）
```

这意味着：
- drought = 60 → 1.25（最高奖励）
- drought = 80 → 1.10
- drought = 120 → 1.05（最低奖励）

**这是一个值得探讨的设计选择**。它表达的是："60-79 天的间隔是最佳甜蜜点，间隔越长反而奖励越小"。

**从金融学角度分析这种递减是否合理**：

**支持递减的理由**：
1. **过长的干旱期可能意味着股票处于长期下跌/横盘趋势**，突破更可能是从低位反弹，而非趋势性上涨的起点
2. **300 天未突破的股票突然突破**，可能缺乏机构关注和交易量支持，突破的可持续性存疑
3. **60-80 天（约 3-4 个月）的间隔**恰好覆盖一个完整的季度/盈利周期，这个时间尺度上的突破往往有基本面驱动

**反对递减的理由**：
1. **供需理论**：阻力位越久未被测试，其上方积累的供应量越大，一旦被消化（突破），意味着更强的买方力量
2. **技术分析经典理论**：长期的盘整/下跌后的突破（如 1 年以上横盘后突破）往往标志着大级别趋势的开始
3. 这与 **age 因子**（阻力位年龄越大越好）的逻辑有一定矛盾

**综合判断**：递减设计体现了一种实用主义——超长干旱期的突破虽然有理论上的强意义，但实际成功率可能被其他因素（如低关注度、流动性不足）所拖累。这种取舍是合理的，但 **三级递减增加了参数复杂度而收益有限**。

### 6.3 改进建议

- **简化为两级**：(60, 120) → (1.20, 1.10)，减少一级能降低参数维度，中间的 80 天级别与 60 和 120 差异不大
- 或者**改为平台式**：所有 >= 60 天统一给 1.15-1.20 奖励，不再递减。这更符合"长时间未突破 → 突破有价值"的简单逻辑
- 首次突破（None）的处理值得考虑：当前是不奖不惩（multiplier=1.0），但首次突破往往是 IPO 后或长期下跌后的第一个技术突破信号，可能值得给予适度奖励

## 7. PK_Mom（峰值动量）

### 7.1 计算逻辑审核

```python
# 限定时间窗口内的近期 peak
delta_t = breakout_idx - peak_idx
if delta_t > pk_lookback(30) or delta_t <= 0:
    return 0.0  # 超出窗口，不计算

# 找到 peak 和 breakout 之间的 trough（最低价）
trough_price = min(df["low"][peak_idx : breakout_idx + 1])

# 凹陷深度标准化
dip_depth = peak_price - trough_price
d_atr = dip_depth / ATR_at_peak_time

# 最终值
pk_mom = 1 + log(1 + d_atr)
```

**逻辑正确性**：

这个因子衡量的是"从峰值到突破之间的 V 型回调深度"。pk_mom 值越大，说明凹陷越深——即突破前经历了更深的回调再反弹。

- `pk_mom = 1.0`：无凹陷（价格从未跌破峰值，直接突破）
- `pk_mom = 1.2`：浅凹陷
- `pk_mom = 1.5`：深凹陷
- `pk_mom > 2.0`：极深凹陷（d_atr >= e - 1 ≈ 1.72，即回调幅度接近 2 个 ATR）

**使用 peak 时刻 ATR 作为分母的设计**：代码注释解释了原因——如果使用 breakout 前一日的 ATR，V 型反弹段会膨胀 ATR 值，导致 d_atr 反而降低。使用 peak 时刻的 ATR 消除了这种污染。这是一个细致且正确的设计决策。

### 7.2 关于 mining_mode 的澄清

**重要更正**：在因子注册表中，pk_mom 的 `mining_mode` 为 `None`（默认值），**而非 'lte'**。这意味着挖掘管道会通过 Spearman 相关系数自动推断方向。

注册表定义：
```python
FactorInfo('pk_mom', 'Peak Momentum', '峰值动量',
           (1.2, 1.5), (1.2, 1.5),
           has_nan_group=True, category='breakout', ...)
# 注意: 没有 mining_mode 参数，使用默认值 None
```

评分系统行为（`_get_factor_value()` 始终用 `>=`）：
- pk_mom >= 1.2 → multiplier = 1.2（奖励）
- pk_mom >= 1.5 → multiplier = 1.5（更大奖励）

**这意味着：凹陷越深的 V 型反弹突破获得越大的奖励**。

### 7.3 参数审核

| 参数 | 当前值 | 评估 |
|------|--------|------|
| pk_lookback | 30 天 | 合理 |
| 阈值 | (1.2, 1.5) | 合理 |
| 奖励值 | (1.2, 1.5) | **偏高** |

**pk_lookback=30 天**：近期 peak（约 6 周内）才计算，远期 peak 的 V 型模式交给 height 因子覆盖。窗口大小合理。

**奖励值分析**：
- pk_mom = 1.2 时 multiplier = 1.2（20% 加成），对应 `d_atr = e^0.2 - 1 ≈ 0.22`，即回调幅度约 0.22 个 ATR。对于 ATR = 3% 的股票，这只是 0.66% 的回调——非常浅的凹陷就获得 20% 奖励，可能偏大。
- pk_mom = 1.5 时 multiplier = 1.5（50% 加成），这是所有因子中**最大的单因子奖励之一**。需要验证如此大的奖励是否有数据支撑。

### 7.4 金融学依据

pk_mom 因子捕获的是"深蹲起跳"模式——价格从峰值回落后强力反弹并突破。这个模式在技术分析中有坚实的理论基础：

**支持"深凹陷更好"的理由**：
1. **弹簧效应（Wyckoff Spring）**：价格跌破支撑后快速反弹突破，是经典的蓄力突破模式。回调越深说明洗盘越彻底
2. **止损触发后的空头回补**：深回调触发更多止损单，空方力量被消耗殆尽，为后续上涨减少了卖方压力
3. **技术面的确认度**：经历深 V 后仍能突破新高，说明买方力量确实强劲

**但需要注意的风险**：
1. 深凹陷也可能意味着基本面恶化后的虚假反弹（死猫跳）
2. 回调幅度过深时（如 3-5 个 ATR），反弹突破更可能是对回调的技术性反弹而非趋势反转

### 7.5 改进建议

- **考虑设置上限**：当 pk_mom > 2.0（极深凹陷，d_atr > 1.72）时，奖励不再增加或转为惩罚，避免"死猫跳"获得高分
- **降低奖励值**：建议 (1.1, 1.25)，50% 的奖励过于激进
- `pk_mom = 0.0`（超出 30 天窗口的 peak）在评分时 multiplier = 1.0（zero_guard 未启用，但 0 < 1.2 不触发），这是正确行为

## 8. 因子间关系分析

### 8.1 Day_Str 与 Overshoot：是否冲突？

**结论：不冲突，互补性良好。**

| 维度 | day_str | overshoot |
|------|---------|-----------|
| 时间尺度 | 单日（日内涨幅/跳空） | 5 日累积涨幅 |
| 方向 | 奖励（> 1.0） | 惩罚（< 1.0） |
| 含义 | 突破当天的爆发力 | 连续上涨的回调风险 |

两者可以共存的场景：
1. **day_str 高 + overshoot 低**：突破日大涨，但前 4 天平稳 → 理想的突破模式（高分）
2. **day_str 高 + overshoot 高**：突破日大涨且前 4 天也连续大涨 → 强势但有超买风险（day_str 奖励被 overshoot 惩罚部分抵消，总分适中）
3. **day_str 低 + overshoot 高**：突破日温和但过去 5 天涨幅惊人 → 缓慢攀升后的弱突破（被惩罚，合理）

**唯一需要注意的边界**：当 day_str 极端高（如 4σ 日涨幅）时，单日贡献就可能将 5 日 overshoot 推到惩罚区间。此时 day_str 奖励（1.35）和 overshoot 惩罚（0.80）的净效应为 1.35 × 0.80 = 1.08，仅有微弱正向影响。这种"自我对冲"是否是预期行为取决于设计意图——如果认为"即使是单日暴涨如果推高了 5 日涨幅也应谨慎"，那么当前设计合理。

### 8.2 PBM 与 Streak 的关系

**间接关联，不冲突**。

- **高 streak（频繁突破）+ 高 pbm（强前动量）**：可能出现在趋势强劲的股票中，但 streak 惩罚会抵消 pbm 奖励，净效应倾向保守。这合理——即使前动量强，频繁突破仍然是噪音信号。
- **低 streak（稀少突破）+ 高 pbm**：理想组合，两者不冲突。
- 两个因子测量的维度不同：pbm 是价格行为的质量，streak 是信号频率的过滤。

### 8.3 PK_Mom 与其他因子

- **pk_mom 与 day_str**：pk_mom 衡量突破前的 V 型深度，day_str 衡量突破日本身的强度。两者可以独立触发，捕获不同维度的信息。一个深 V 反弹后的强势突破日（两者都高）确实是高质量突破信号。
- **pk_mom 与 pbm**：存在一定的正相关——深 V 反弹过程中，反弹段的 PBM 往往较高（路径效率高、方向一致）。但 PBM 的 lookback=5 天可能只覆盖反弹的最后阶段，而 pk_mom 覆盖整个 V 型过程。当 lookback 较短时两者的冗余度较低。
- **pk_mom 与 height**（阻力属性因子）：pk_mom 聚焦近期 peak（30 天内），height 聚焦所有被突破 peak 的相对高度。虽然近期 peak 的回调深度与 height 有关联，但两者不重复：height 是阻力位本身的属性，pk_mom 是价格到达阻力位的路径特征。

### 8.4 Drought 与 Streak 的逻辑互斥

如前所述，两者存在天然的互斥区域：
- streak >= 2（20 天内有 2+ 次突破）→ drought 最多 20 天 → drought 不触发奖励（需 >= 60 天）
- drought >= 60 → 意味着最近 60 天没有突破 → streak 必然为 1（仅当前这次）

这不是设计缺陷，而是因子自然覆盖了频谱的两端：
- **高频端**：streak 惩罚过密的突破
- **低频端**：drought 奖励稀疏的突破
- **中间段**（如 20-59 天间隔的突破）：两者都不触发，multiplier = 1.0

## 9. 系统性问题与边界条件

### 9.1 乘法模型的累积效应

当多个因子同时触发时，乘法效应可能导致极端分数：
- 最极端奖励场景：volume(2.0) × day_str(1.35) × pbm(1.3) × drought(1.25) × pk_mom(1.5) = 2.0 × 1.35 × 1.3 × 1.25 × 1.5 = **6.57 倍**
- 基准分 50 × 6.57 = **328.5 分**

这种极端分数的股票是否真的值得如此高的评价？还需要加上阻力属性因子的潜在乘积效应。如果总乘数经常超过 4-5 倍，可能需要考虑对总分加上软上限或对数压缩。

### 9.2 波动率为零的边界情况

多个因子（day_str, overshoot, pbm）依赖 `annual_volatility > 0`。当波动率计算返回 0（历史数据不足 20 天）时，这些因子全部返回 0.0，不触发任何等级。

**影响**：新上市股票（IPO 不足 20 个交易日）或数据截断的股票将无法获得这些因子的评分。这是合理的降级行为——数据不足时不做判断优于做错误判断。

### 9.3 gain_5d 的窗口边界

`_calculate_gain_5d()` 要求 `idx >= gain_window`，否则返回 0.0。这意味着数据前 5 天的突破 overshoot 为 0，不会触发惩罚。在实际使用中这个边界影响极小（极少有在数据前 5 天就突破的情况）。

### 9.4 pk_mom 对首次突破的处理

当股票是首次检测到的突破（没有历史突破记录）时：
- `breakout_info.broken_peaks` 仍然有值（被突破的峰值来自历史高点检测，不是历史突破记录）
- `nearest_peak` 基于所有被突破峰值中索引最大的那个
- 只要 `delta_t <= 30` 且 ATR 有效，pk_mom 就会正常计算

这个行为是正确的——pk_mom 不依赖历史突破记录，只依赖被突破峰值与当前突破的关系。

## 10. 综合改进建议

### 优先级 P0（强烈建议）

1. **Volume 阈值下调**：(5.0, 10.0) → (2.5, 5.0)。当前阈值可能导致因子实际触发率过低，无法有效区分突破质量。这是最需要数据验证的一项。

### 优先级 P1（建议）

2. **PK_Mom 奖励值下调**：(1.2, 1.5) → (1.1, 1.25)。50% 的单因子奖励过于激进，在乘法模型中会被其他因子放大。

3. **Drought 简化为两级**：(60, 80, 120) / (1.25, 1.1, 1.05) → (60, 120) / (1.20, 1.10)。三级递减增加的精度有限，两级足以捕获"适度干旱"和"长期干旱"的区别。

### 优先级 P2（可选优化）

4. **PBM lookback 扩展**：5 → 10-15 天，提高统计稳健性和对中期动量的捕获能力。但需要注意与 day_str 的去相关。

5. **PK_Mom 上限保护**：当 pk_mom > 2.0 时不再增加奖励（或微弱惩罚），防止极端深 V 的"死猫跳"获得高分。

6. **总分压缩机制**：如果乘法模型的总乘数分布右偏严重（极端高分过多），考虑对总分施加 `log` 或 `sqrt` 压缩。

### 不建议修改

- **Day_str 与 Overshoot 的关系**：当前设计互补性良好，不需要调整。
- **Streak 参数**：(2, 4) 的阈值和 (0.9, 0.75) 的惩罚都是合理的。
- **Overshoot 整体设计**：阈值、惩罚值、mining_mode 设置都合理。
